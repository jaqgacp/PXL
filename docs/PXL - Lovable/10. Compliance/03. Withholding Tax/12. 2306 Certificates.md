# PXL ERP Blueprint: BIR Form 2306 — Certificate of Final Tax Withheld at Source

## Module Overview
BIR Form 2306 is the Certificate of Final Tax Withheld at Source. It is issued by a withholding agent to a payee when **Final Withholding Tax (FWT)** is deducted. FWT differs fundamentally from EWT (Expanded Withholding Tax, covered by Form 2307) — FWT is a full and final settlement of tax liability on specific income types and is not creditable against income tax. Common FWT events in Philippine corporate accounting:

| Income Type | FWT Rate | BIR Reference |
| :--- | :--- | :--- |
| Interest income on bank deposits (PHP accounts) | 20% | NIRC Sec. 27(D)(1) |
| Interest income on FCDU accounts | 15% | NIRC Sec. 27(D)(3) |
| Cash dividends to domestic corporations | 0% | Exempt under NIRC |
| Royalties | 20% | NIRC Sec. 27(D)(4) |
| Prizes exceeding ₱10,000 | 20% | NIRC Sec. 24(B)(1) |
| FWT on capital gains (real property, shares) | Various | NIRC Sec. 27(D)(5) |

**Issuance Obligation:** A company that pays income subject to FWT must issue BIR Form 2306 to the payee within the prescribed period (within 20 days after end of the quarter, or upon payment if earlier). Failure to issue 2306 is a BIR CAS compliance failure.

**Who Issues 2306:** The company (withholding agent) issues 2306 to payees. Banks issue 2306 to the company for interest income (captured in Bank Adjustments module). This module tracks both scenarios.

## Dashboard UI

### The Action Bar
* **Search Bar:** Find by Certificate No., Payee TIN, or Period.
* **Filter:** By Year, Quarter, FWT Type (Bank Interest, Royalties, etc.), Status (Pending, Issued).
* **Generate Certificates Button:** Auto-creates 2306 records from posted FWT transactions for the period.
* **Print / Export PDF Button:** Generates BIR-formatted Form 2306 PDF for issuance to payees.
* **Mark as Issued Button:** Updates `status` to `issued` and records the date issued.
* **Export Batch Button:** Exports a batch of 2306 PDFs for distribution.

### The Data List
* Certificate No.
* Payee Name
* Payee TIN
* Quarter Covered
* FWT Type
* Gross Income
* FWT Rate
* Tax Withheld
* Status (Pending / Issued)

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 50 records.

## Supabase Database Architecture

### Table 1: `form_2306_certificates`
Unlike Form 2307 which has a header/lines split, Form 2306 is a single-level certificate (one income type, one tax rate, one payee per certificate).

**Critical Database Rules:**
1. `UNIQUE(company_id, certificate_no)` — sequential numbering per tenant.
2. `issued_at` and `issued_by` are system-set when status changes to `issued`. Never editable.
3. The certificate is linked to its source document via `source_document_type` and `source_document_id` for full traceability (Principle 7).

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `certificate_no` | Text | Required | `UNIQUE(company_id, certificate_no)`. Sequential number. |
| `period_year` | Integer | Required | Year the FWT was withheld. |
| `period_quarter` | Integer | Required, CHECK (1–4) | Quarter the FWT was withheld. |
| `payee_name` | Text | Required | Registered name of the income recipient. |
| `payee_tin` | Text | Required | BIR TIN of the payee. |
| `payee_address` | Text | Required | Registered address of the payee. |
| `fwt_type` | Text | Required | ENUM: `bank_interest_php`, `bank_interest_fcdu`, `royalties`, `prizes`, `dividends`, `capital_gains`, `other`. |
| `atc_code` | Text | Required | BIR Alphanumeric Tax Code for the specific FWT (e.g., `WI080` for bank interest). |
| `gross_income` | Numeric(15,2) | Required | Total income before FWT deduction. |
| `fwt_rate` | Numeric(5,2) | Required | Rate applied (e.g., 20.00 for 20%). |
| `fwt_amount` | Numeric(15,2) | Required | `gross_income × (fwt_rate / 100)`. |
| `net_income_paid` | Numeric(15,2) | Required | `gross_income - fwt_amount`. |
| `source_document_type` | Text | Required | Originating transaction type: `bank_adjustment`, `journal_entry`, `check_voucher`, etc. |
| `source_document_id` | UUID | Required | FK to the originating document (Principle 7 traceability). |
| `source_reference` | Text | Required | The document number from the source (e.g., Bank DM reference). |
| `file_url` | Text | Nullable | Supabase Storage URL if a signed PDF copy is archived. |
| `issued_at` | Timestamp | Nullable | Timestamp when the certificate was issued to payee. |
| `issued_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who marked the certificate as issued. |
| `status` | Text | Required, Default `pending` | ENUM: `pending` (not yet issued to payee), `issued` (certificate delivered). |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, certificate_no)`

## Row-Level Security

```sql
-- Block UPDATE on issued certificates
CREATE POLICY "block_update_on_issued_2306"
ON form_2306_certificates
FOR UPDATE
USING (status = 'pending');

-- Block DELETE — BIR certificates are permanent records
CREATE POLICY "block_delete_2306"
ON form_2306_certificates
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_2306"
ON form_2306_certificates
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```

## Integration with Bank Adjustments Module

When a Bank Adjustment is posted with `is_interest_income = true`, the system **automatically creates** a `form_2306_certificates` record with:
- `fwt_type = 'bank_interest_php'`
- `source_document_type = 'bank_adjustment'`
- `source_document_id = bank_adjustments.id`
- `gross_income` = full interest amount
- `fwt_amount` = 20% of gross
- `status = 'pending'` (the bank has already withheld — the certificate is awaiting receipt from the bank)

This ensures the FWT credit is immediately tracked and can be claimed on BIR Form 1702Q (Quarterly Income Tax Return).
