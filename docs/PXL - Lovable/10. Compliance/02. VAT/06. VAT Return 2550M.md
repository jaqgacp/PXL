# PXL ERP Blueprint: VAT Return — BIR Form 2550M (Monthly VAT Declaration)

## Module Overview
The VAT Return 2550M is the Monthly Value-Added Tax Declaration filed by VAT-registered entities with the BIR on or before the 20th day following the close of the month (or per eFPS group deadline). It reports total Output VAT on sales and total Input VAT on purchases for the month. The system auto-populates all amounts from posted Sales Invoices and Purchase Invoices within the period — no manual re-encoding. The schema maps directly to the actual BIR Form 2550M boxes to ensure the generated return can be filed without modification.

**Activation Condition:** This module is visible only when `compliance_profiles.vat_registered = true` AND `compliance_profiles.vat_filing_frequency = 'monthly'`.

## Dashboard UI

### The Action Bar
* **Search Bar:** Find by period or reference number.
* **Filter:** By Year, Month, Status (Draft, Final, Filed, Amended).
* **Generate Return Button:** Pulls data from posted SIs and PIs for the selected month. Idempotent — re-generating overwrites the draft.
* **Export DAT Button:** Downloads the BIR-formatted `.dat` file for eFPS/eBIR submission.
* **Export PDF Button:** Generates a print-ready Form 2550M.
* **Mark as Filed Button:** Updates status to `filed` and records the BIR confirmation number.

### The Data List
* Return Period (Month-Year)
* Total Output VAT
* Total Input VAT
* Net VAT Payable / Excess Input
* Status
* Date Filed
* BIR Confirmation No.

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 24 records (two years visible).

## BIR Form 2550M — Box Mapping

The following schedule maps each BIR Form 2550M box to its database source. The system auto-computes all amounts from posted transaction data.

| BIR Box | Form Label | Database Source |
| :--- | :--- | :--- |
| Box 12 | Taxable Sales / Receipts | `SUM(sales_invoice_lines.net_amount) WHERE vat_classification = 'regular'` for the period |
| Box 13 | Output Tax (12% of Box 12) | `SUM(sales_invoices.total_vat_amount)` for regular-rated lines |
| Box 14 | Zero-Rated Sales / Receipts | `SUM(sales_invoice_lines.net_amount) WHERE vat_classification = 'zero_rated'` |
| Box 15 | Exempt Sales / Receipts | `SUM(sales_invoice_lines.net_amount) WHERE vat_classification = 'exempt'` |
| Box 16 | Total Sales (12+14+15) | Sum of Boxes 12, 14, 15 |
| Box 17 | Total Output Tax (= Box 13) | Carried from Box 13 |
| Box 18A | Input Tax Carried Over from Prior Month | `vat_return_2550m_headers.input_vat_carried_forward` from previous period |
| Box 18B | Input Tax on Domestic Purchases (Capital Goods ≤ ₱1M) | `SUM(purchase_invoice_lines.input_vat_amount) WHERE item category = capital_goods AND unit_cost ≤ 1,000,000` |
| Box 18C | Input Tax on Domestic Purchases (Capital Goods > ₱1M) | Per RMC 42-2003 amortization schedule; separate computation |
| Box 18D | Input Tax on Domestic Purchases (Non-Capital) | `SUM(purchase_invoice_lines.input_vat_amount) WHERE NOT capital_goods` |
| Box 18E | Input Tax on Importations | Reserved for import transactions |
| Box 19 | Total Available Input Tax (18A+18B+18C+18D+18E) | Sum of all Box 18 components |
| Box 20A | Input Tax Allocable to Exempt Sales | Computed: `(Exempt Sales / Total Sales) × Total Input Tax` |
| Box 20B | Input Tax Allocable to Zero-Rated Sales | Computed: `(Zero-Rated Sales / Total Sales) × Total Input Tax` |
| Box 21 | Total Input Tax Not Allocable (19 - 20A - 20B) | `Box 19 - Box 20A - Box 20B` |
| Box 22 | Net VAT Payable (Box 17 - Box 21) | Positive = VAT due; Negative = excess input to carry over |
| Box 23 | Less: VAT Withheld by Customers (2307 received) | `SUM(form_2307_tracking.ewt_amount) WHERE period = current month` |
| Box 24 | Tax Still Due / Excess Input to Carry Over | `Box 22 - Box 23` |

## Supabase Database Architecture

### Table 1: `compliance_vat_return_2550m_headers`

**Critical Database Rules:**
1. `UNIQUE(company_id, period_year, period_month)` — one 2550M per company per month.
2. Status `filed` locks the record. A new record with `status = 'amended'` is created for amendments.
3. `input_vat_carried_forward` from a filed period automatically populates Box 18A of the next period.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `period_year` | Integer | Required | The year covered (e.g., 2026). |
| `period_month` | Integer | Required, CHECK (1–12) | The month covered (1 = January, 12 = December). |
| `taxpayer_tin` | Text | Required | Snapshot from `companies.tin` at time of generation. |
| `box_12_taxable_sales` | Numeric(15,2) | Required, Default 0 | Regular-rated net sales. |
| `box_13_output_tax` | Numeric(15,2) | Required, Default 0 | 12% Output VAT on taxable sales. |
| `box_14_zero_rated_sales` | Numeric(15,2) | Required, Default 0 | Zero-rated net sales. |
| `box_15_exempt_sales` | Numeric(15,2) | Required, Default 0 | Exempt net sales. |
| `box_16_total_sales` | Numeric(15,2) | Required, Default 0 | Box 12 + 14 + 15. |
| `box_18a_input_vat_carried_forward` | Numeric(15,2) | Required, Default 0 | Excess input VAT from prior month. Auto-populated from prior period's Box 24 when negative. |
| `box_18b_input_capital_small` | Numeric(15,2) | Required, Default 0 | Input VAT on capital goods ≤ ₱1M. |
| `box_18c_input_capital_large` | Numeric(15,2) | Required, Default 0 | Input VAT on capital goods > ₱1M (amortized). |
| `box_18d_input_noncapital` | Numeric(15,2) | Required, Default 0 | Input VAT on non-capital purchases. |
| `box_18e_input_importation` | Numeric(15,2) | Required, Default 0 | Input VAT on importations. |
| `box_19_total_available_input` | Numeric(15,2) | Required, Default 0 | Sum of all Box 18 components. |
| `box_20a_input_allocable_exempt` | Numeric(15,2) | Required, Default 0 | Input VAT apportioned to exempt sales. |
| `box_20b_input_allocable_zero` | Numeric(15,2) | Required, Default 0 | Input VAT apportioned to zero-rated sales. |
| `box_21_net_input_vat` | Numeric(15,2) | Required, Default 0 | Box 19 - Box 20A - Box 20B. |
| `box_22_net_vat_payable` | Numeric(15,2) | Required, Default 0 | Box 17 - Box 21. Positive = due. Negative = excess. |
| `box_23_cwt_from_2307` | Numeric(15,2) | Required, Default 0 | VAT withheld by customers (2307 received this period). |
| `box_24_tax_still_due` | Numeric(15,2) | Required, Default 0 | Box 22 - Box 23. Final tax due or excess to carry forward. |
| `input_vat_carried_forward` | Numeric(15,2) | Required, Default 0 | Amount to carry to next period's Box 18A. Computed: max(0, -Box_24). |
| `efps_confirmation_no` | Text | Nullable | BIR eFPS confirmation number upon filing. |
| `date_filed` | Date | Nullable | Actual filing date. |
| `date_paid` | Date | Nullable | Date tax was remitted. |
| `amendment_number` | Integer | Required, Default 0 | 0 = original. 1, 2, 3... = amendment sequence. |
| `amends_id` | UUID | Nullable, Foreign Key | Self-reference: links to the original return being amended. |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `final`, `filed`, `amended`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, period_year, period_month, amendment_number)`

### Table 2: `compliance_vat_return_2550m_lines`
Detail lines linking each 2550M to the specific posted transactions that compose it. Enables drill-down from the tax return to the source invoice.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Denormalized for RLS. |
| `header_id` | UUID | Required, Foreign Key | Links to `compliance_vat_return_2550m_headers.id`. |
| `source_document_type` | Text | Required | ENUM: `sales_invoice`, `purchase_invoice`. |
| `source_document_id` | UUID | Required | FK to `sales_invoices.id` or `purchase_invoices.id`. |
| `document_number` | Text | Required | Snapshot of the SI or PI number. |
| `document_date` | Date | Required | Snapshot of the transaction date. |
| `tin` | Text | Required | Snapshot of customer or supplier TIN. |
| `registered_name` | Text | Required | Snapshot of customer or supplier name. |
| `vat_classification` | Text | Required | ENUM: `regular`, `zero_rated`, `exempt`. |
| `net_amount` | Numeric(15,2) | Required | Sale or purchase amount exclusive of VAT. |
| `vat_amount` | Numeric(15,2) | Required, Default 0 | VAT amount on this transaction. |
| `box_reference` | Text | Required | Which BIR box this line contributes to (e.g., `box_12`, `box_18d`). |
| `created_at` | Timestamp | Auto | Creation timestamp. |

## Row-Level Security

```sql
-- Filed returns are immutable
CREATE POLICY "block_update_on_filed_2550m"
ON compliance_vat_return_2550m_headers
FOR UPDATE
USING (status IN ('draft', 'final'));

-- No deletes allowed on any 2550M record
CREATE POLICY "block_delete_2550m"
ON compliance_vat_return_2550m_headers
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_2550m"
ON compliance_vat_return_2550m_headers
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
