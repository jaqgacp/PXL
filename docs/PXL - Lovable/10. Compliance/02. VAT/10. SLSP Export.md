# PXL ERP Blueprint: SLSP Export (Summary List of Sales and Purchases)

## Module Overview
The Summary List of Sales and Purchases (SLSP) is a BIR-required attachment submitted with the Quarterly VAT Return (2550Q). It is a line-by-line disclosure of every VAT-registered transaction in the quarter, identifying each counterparty by TIN and Registered Name. The BIR uses the SLSP to cross-reference the sales declared by the company against the purchases claimed by its counterparties — discrepancies trigger audit flags.

The SLSP is submitted as a DAT file to the BIR eFPS/eBIR system. Every column in the DAT file specification must map to an actual database field. A generic `amount` column is not a BIR-compliant SLSP schema.

**Activation Condition:** Visible when `compliance_profiles.slsp_required = true`.

**Filing Frequency:** Quarterly, attached to BIR Form 2550Q.

## Dashboard UI

### The Action Bar
* **Search Bar:** Find by period or TIN.
* **Filter:** By Year, Quarter, List Type (Sales List or Purchases List), Status.
* **Generate SLSP Button:** Auto-pulls all posted and filed SI and PI transactions for the quarter.
* **Export DAT Button:** Generates the BIR SLSP `.dat` file in the exact format required by the BIR validation software.
* **Export Excel Button:** Human-readable version for review before submission.
* **Mark as Filed Button:** Updates status to `filed`.

### The Data List (Summary View)
* Quarter
* List Type (Sales / Purchases)
* Total Transaction Count
* Total Gross Sales/Purchases
* Total VAT Amount
* Status

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 50. Line-item drill-down uses separate paginated view.

## BIR SLSP DAT File Format — Required Columns

The BIR SLSP validation software requires the following exact fields. Each corresponds to a column in the `compliance_slsp_export_lines` table.

### Sales List (SLS) Fields:
| DAT Column | Description | Source |
| :--- | :--- | :--- |
| Buyer's TIN | 9-digit or 12-digit TIN of the buyer | `customers.tin` snapshot from `sales_invoice.customer_tin_snapshot` |
| Buyer's Name | Registered Name with BIR | `customers.registered_name` snapshot |
| Buyer's Address | Registered address | `customers.registered_address` snapshot |
| Amount of Gross Sales | Invoice amount exclusive of VAT | `sales_invoice_lines.net_amount` sum per invoice |
| Amount of Output VAT | VAT charged on the invoice | `sales_invoices.total_vat_amount` |
| Invoice Number | BIR-registered SI/OR number | `sales_invoices.si_number` |
| Invoice Date | Date of SI/OR | `sales_invoices.date` |

### Purchases List (PLS) Fields:
| DAT Column | Description | Source |
| :--- | :--- | :--- |
| Seller's TIN | TIN of the supplier | `suppliers.tin` snapshot from `purchase_invoices.supplier_tin_snapshot` |
| Seller's Name | Registered Name with BIR | `suppliers.registered_name` snapshot |
| Seller's Address | Registered address | `suppliers.registered_address` snapshot |
| Amount of Gross Purchases | Purchase amount exclusive of VAT | `purchase_invoice_lines.net_amount` sum per invoice |
| Amount of Input VAT | Input VAT claimable | `purchase_invoices.total_input_vat` |
| Invoice Number | Supplier's SI/OR number | `purchase_invoices.supplier_invoice_number` |
| Invoice Date | Date of supplier's invoice | `purchase_invoices.date` |
| Classification | Purchase classification | Mapped from `vat_classification` (e.g., `T`=Taxable, `Z`=Zero-Rated, `E`=Exempt) |

## Supabase Database Architecture

### Table 1: `compliance_slsp_export_headers`

**Critical Database Rules:**
1. `UNIQUE(company_id, period_year, period_quarter, list_type)` — separate header for Sales List and Purchases List per quarter.
2. Filed headers are immutable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `period_year` | Integer | Required | The year covered. |
| `period_quarter` | Integer | Required, CHECK (1–4) | The quarter. |
| `list_type` | Text | Required | ENUM: `sales`, `purchases`. Separate header per type. |
| `total_transaction_count` | Integer | Required, Default 0 | Number of line items in this export. |
| `total_gross_amount` | Numeric(15,2) | Required, Default 0 | Sum of all `gross_amount` values. |
| `total_vat_amount` | Numeric(15,2) | Required, Default 0 | Sum of all `vat_amount` values. |
| `efps_confirmation_no` | Text | Nullable | BIR confirmation upon submission. |
| `date_filed` | Date | Nullable | Actual submission date. |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `final`, `filed`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, period_year, period_quarter, list_type)`

---

### Table 2: `compliance_slsp_export_lines`
One row per invoice. Contains all mandatory BIR DAT file fields.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Denormalized for RLS. |
| `header_id` | UUID | Required, Foreign Key | Links to `compliance_slsp_export_headers.id`. |
| `source_document_type` | Text | Required | ENUM: `sales_invoice`, `purchase_invoice`. |
| `source_document_id` | UUID | Required | FK to the originating document. Enables drill-back from SLSP line to source document. |
| `document_number` | Text | Required | BIR-registered SI number (sales) or supplier invoice number (purchases). |
| `document_date` | Date | Required | Date of the invoice. |
| `counterparty_tin` | Text | Required | Customer TIN (Sales List) or Supplier TIN (Purchases List). BIR requires this — cannot be null. |
| `counterparty_name` | Text | Required | Customer or supplier registered name. |
| `counterparty_address` | Text | Required | Customer or supplier registered address. |
| `gross_amount` | Numeric(15,2) | Required | Invoice amount exclusive of VAT. |
| `vat_amount` | Numeric(15,2) | Required, Default 0 | VAT amount on the invoice (Output or Input). |
| `vat_classification` | Text | Required | ENUM: `taxable`, `zero_rated`, `exempt`. Maps to DAT classification code. |
| `dat_classification_code` | Text | Required | BIR DAT file classification: `T`=Taxable, `Z`=Zero-Rated, `E`=Exempt. Derived from `vat_classification`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |

## Row-Level Security

```sql
-- Filed headers are immutable
CREATE POLICY "block_update_on_filed_slsp"
ON compliance_slsp_export_headers
FOR UPDATE
USING (status IN ('draft', 'final'));

-- No deletes
CREATE POLICY "block_delete_slsp"
ON compliance_slsp_export_headers
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_slsp"
ON compliance_slsp_export_headers
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
