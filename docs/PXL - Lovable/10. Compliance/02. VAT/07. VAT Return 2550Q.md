# PXL ERP Blueprint: VAT Return — BIR Form 2550Q (Quarterly VAT Return)

## Module Overview
The VAT Return 2550Q is the Quarterly Value-Added Tax Return filed by VAT-registered entities within 25 days following the close of each taxable quarter. It consolidates the three monthly Output and Input VAT figures, computes the net VAT payable for the quarter, and credits any advance payments made via the 2550M returns for months 1 and 2 of the quarter. The system auto-populates all boxes from posted transactions and previously filed 2550M data — no manual re-encoding.

**Activation Condition:** This module is visible only when `compliance_profiles.vat_registered = true`. It is filed regardless of whether the company also files 2550M — 2550Q is always required for VAT registrants.

**Critical Distinction from 2550M:** The 2550Q spans three months. It aggregates Box 12–23 from each of the three months, then deducts taxes already paid via the two monthly 2550M remittances (Box 22A and 22B).

## Dashboard UI

### The Action Bar
* **Search Bar:** Find by quarter/year or confirmation number.
* **Filter:** By Year, Quarter (Q1–Q4), Status (Draft, Final, Filed, Amended).
* **Generate Return Button:** Aggregates all posted SIs and PIs for the quarter. Idempotent.
* **Export DAT Button:** BIR-formatted `.dat` file for eFPS/eBIR submission.
* **Export PDF Button:** Print-ready Form 2550Q.
* **Mark as Filed Button:** Sets status to `filed` and records BIR confirmation number.

### The Data List
* Quarter (e.g., Q3-2026)
* Total Taxable Sales
* Total Output VAT
* Total Input VAT
* Net VAT Payable
* Tax Already Paid (2550M months 1 & 2)
* Still Due
* Status

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 12 records (three years visible).

## BIR Form 2550Q — Box Mapping

| BIR Box | Form Label | Database Source |
| :--- | :--- | :--- |
| Box 19A | Taxable Sales — Month 1 | Aggregate of `box_12_taxable_sales` from 2550M of Month 1 |
| Box 19B | Taxable Sales — Month 2 | Aggregate of `box_12_taxable_sales` from 2550M of Month 2 |
| Box 19C | Taxable Sales — Month 3 (current) | Direct computation from posted SIs in Month 3 |
| Box 19D | Total Taxable Sales (19A+19B+19C) | Sum |
| Box 20A | Output Tax — Month 1 | `box_13_output_tax` from 2550M of Month 1 |
| Box 20B | Output Tax — Month 2 | `box_13_output_tax` from 2550M of Month 2 |
| Box 20C | Output Tax — Month 3 | Direct computation |
| Box 20D | Total Output Tax (20A+20B+20C) | Sum |
| Box 21A | Zero-Rated Sales — Quarter Total | `SUM(sales_invoice_lines.net_amount) WHERE vat_classification = 'zero_rated'` for all 3 months |
| Box 21B | Exempt Sales — Quarter Total | `SUM(sales_invoice_lines.net_amount) WHERE vat_classification = 'exempt'` for all 3 months |
| Box 23A | Input VAT Carried Over from Prior Quarter | From prior quarter's `box_27_excess_input_carried_forward` |
| Box 23B | Input VAT on Capital Goods (≤ ₱1M) | Quarter aggregate |
| Box 23C | Input VAT on Capital Goods (> ₱1M, amortized) | Per RMC 42-2003 amortization |
| Box 23D | Input VAT on Non-Capital Domestic Purchases | Quarter aggregate |
| Box 23E | Input VAT on Importations | Quarter aggregate |
| Box 24 | Total Available Input Tax (23A+23B+23C+23D+23E) | Sum |
| Box 25A | Input Tax Allocable to Exempt Sales | `(Exempt Sales / Total Sales) × Total Input Tax` |
| Box 25B | Input Tax Allocable to Zero-Rated Sales | `(Zero-Rated Sales / Total Sales) × Total Input Tax` |
| Box 26 | Total Input Tax Attributable to Taxable Sales (24-25A-25B) | Net input VAT claimable |
| Box 27 | Net VAT Payable / Excess Input (Box 20D - Box 26) | Positive = due. Negative = excess to carry forward. |
| Box 27A | Excess Input VAT Carried Forward to Next Quarter | `max(0, -Box_27)` |
| Box 28A | VAT Paid in Month 1 (from 2550M filing) | `compliance_vat_return_2550m_headers.box_24_tax_still_due` for Month 1 |
| Box 28B | VAT Paid in Month 2 (from 2550M filing) | `compliance_vat_return_2550m_headers.box_24_tax_still_due` for Month 2 |
| Box 28C | CWT from 2307 Received (quarter total) | `SUM(form_2307_tracking.ewt_amount)` for the quarter |
| Box 29 | Tax Still Due (Box 27 - Box 28A - Box 28B - Box 28C) | Final payment due with the 2550Q |

## Supabase Database Architecture

### Table 1: `compliance_vat_return_2550q_headers`

**Critical Database Rules:**
1. `UNIQUE(company_id, period_year, period_quarter)` — one 2550Q per company per quarter.
2. Status `filed` locks the record. Amendments create new rows with `amendment_number > 0`.
3. On filing, `box_27a_excess_input_carried_forward` is automatically pre-loaded into the next quarter's `box_23a`.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `period_year` | Integer | Required | The year covered (e.g., 2026). |
| `period_quarter` | Integer | Required, CHECK (1–4) | The quarter (1=Jan-Mar, 2=Apr-Jun, 3=Jul-Sep, 4=Oct-Dec). |
| `taxpayer_tin` | Text | Required | Snapshot from `companies.tin`. |
| `month1_2550m_id` | UUID | Nullable, Foreign Key | Links to the Month 1 `compliance_vat_return_2550m_headers.id`. |
| `month2_2550m_id` | UUID | Nullable, Foreign Key | Links to the Month 2 `compliance_vat_return_2550m_headers.id`. |
| `box_19d_total_taxable_sales` | Numeric(15,2) | Required, Default 0 | Quarter total regular-rated net sales. |
| `box_20d_total_output_tax` | Numeric(15,2) | Required, Default 0 | Quarter total Output VAT. |
| `box_21a_zero_rated_sales` | Numeric(15,2) | Required, Default 0 | Quarter zero-rated net sales. |
| `box_21b_exempt_sales` | Numeric(15,2) | Required, Default 0 | Quarter exempt net sales. |
| `box_23a_input_vat_carried_fwd` | Numeric(15,2) | Required, Default 0 | Excess Input from prior quarter (Box 27A of prior 2550Q). |
| `box_23b_input_capital_small` | Numeric(15,2) | Required, Default 0 | Input VAT on capital goods ≤ ₱1M. |
| `box_23c_input_capital_large` | Numeric(15,2) | Required, Default 0 | Input VAT on capital goods > ₱1M (amortized). |
| `box_23d_input_noncapital` | Numeric(15,2) | Required, Default 0 | Input VAT on non-capital domestic purchases. |
| `box_23e_input_importation` | Numeric(15,2) | Required, Default 0 | Input VAT on importations. |
| `box_24_total_available_input` | Numeric(15,2) | Required, Default 0 | Sum of Box 23A through 23E. |
| `box_25a_input_allocable_exempt` | Numeric(15,2) | Required, Default 0 | Input tax apportioned to exempt sales. |
| `box_25b_input_allocable_zero` | Numeric(15,2) | Required, Default 0 | Input tax apportioned to zero-rated sales. |
| `box_26_net_input_attributable` | Numeric(15,2) | Required, Default 0 | Box 24 - Box 25A - Box 25B. |
| `box_27_net_vat_payable` | Numeric(15,2) | Required, Default 0 | Box 20D - Box 26. Positive = due, negative = excess. |
| `box_27a_excess_input_carried_fwd` | Numeric(15,2) | Required, Default 0 | Amount to carry to next quarter's Box 23A. `max(0, -box_27)`. |
| `box_28a_vat_paid_month1` | Numeric(15,2) | Required, Default 0 | Tax paid via Month 1's 2550M filing. |
| `box_28b_vat_paid_month2` | Numeric(15,2) | Required, Default 0 | Tax paid via Month 2's 2550M filing. |
| `box_28c_cwt_from_2307` | Numeric(15,2) | Required, Default 0 | CWT credits from 2307s received in the quarter. |
| `box_29_tax_still_due` | Numeric(15,2) | Required, Default 0 | Box 27 - Box 28A - Box 28B - Box 28C. Final payment with 2550Q. |
| `efps_confirmation_no` | Text | Nullable | BIR eFPS confirmation number upon filing. |
| `date_filed` | Date | Nullable | Actual filing date. |
| `date_paid` | Date | Nullable | Date tax was remitted. |
| `amendment_number` | Integer | Required, Default 0 | 0 = original. 1, 2, 3... = amendment sequence. |
| `amends_id` | UUID | Nullable, Foreign Key | Self-reference: the original return being amended. |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `final`, `filed`, `amended`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, period_year, period_quarter, amendment_number)`

## Row-Level Security

```sql
-- Filed returns are immutable
CREATE POLICY "block_update_on_filed_2550q"
ON compliance_vat_return_2550q_headers
FOR UPDATE
USING (status IN ('draft', 'final'));

-- No deletes
CREATE POLICY "block_delete_2550q"
ON compliance_vat_return_2550q_headers
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_2550q"
ON compliance_vat_return_2550q_headers
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
