# PXL ERP Blueprint: Inventory Valuation

## Module Overview
The Inventory Valuation module calculates the total monetary worth of all stock on hand at a specific point in time. For Philippine businesses, this valuation is strictly required for the Balance Sheet (Current Assets) and directly impacts the Cost of Goods Sold (COGS) calculation for the Income Statement. This module enforces the chosen cost method (e.g., FIFO or Moving Average) as defined in the Item Master and leverages the `inventory_movements` ledger to determine exact historical costs.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **As Of Date Picker:** Generate valuation up to a specific date (crucial for year-end or month-end closing).
* **Warehouse Filter:** Filter by specific branches or warehouses.
* **Category Filter:** View valuation by item grouping.
* **Export Button:** Download the valuation report to Excel, PDF, or CSV for the accountant/auditor.
* **Print Button:** Formatted print view.

### The Data List (The main report table):
Columns:
* Item Code
* Item Name
* Category
* Valuation Method (FIFO/Weighted Avg)
* Quantity on Hand (As of selected date)
* Average/Computed Unit Cost
* Total Value

## The Supabase Database Architecture (For Lovable)
*No standalone table is created for Valuation.*
The system will use robust SQL functions or views to calculate this on the fly:
1. **Quantity Calculation:** SUM of `quantity` from `inventory_movements` where `posting_date` <= "As Of Date".
2. **Cost Calculation:** Depends on the item's `costing_method` in the `items` table.
   - For **Weighted Average**: A cumulative rolling average calculation across historical purchase receipts in `inventory_movements`.
   - For **FIFO**: Trailing matching of remaining stock against the most recent purchase receipts in `inventory_movements`.
3. **Total Value:** `Computed Quantity` × `Computed Cost`.

The calculations strictly inherit audit validity by relying entirely on the `inventory_movements` table, which is tightly tied to approved Sales and Purchasing transactions, thus satisfying strict Philippine BIR audit requirements.


**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
