# PXL ERP Blueprint: Goods Issue

## Module Overview
The Goods Issue module is used for issuing inventory out of the warehouse for non-sales purposes. This could include internal consumption (office supplies), sampling, project use, or production materials. In Philippine accounting, these movements must be properly tracked as they do not generate Revenue (no Sales Invoice) but do affect Cost of Goods Sold or Operating Expenses. Approved Goods Issues automatically credit the inventory balance via the `inventory_movements` ledger.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **Search Bar:** Search by Issue Number or Project/Department.
* **Filter:** Filter by Status (Draft, Issued, Cancelled) and Date.
* **Customize View:** Choose columns to display.
* **Export Button:** Download list to Excel or CSV.
* **Create New Goods Issue Button:** Opens the entry form.

### The Data List (The main table - must be customizable):
Columns:
* Issue Number
* Date
* Target Department / Project
* Total Value Issued
* Status
* Created By

## The Supabase Database Architecture (For Lovable)

### Table 1: `goods_issues` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `issue_number` | Text | Required, Unique | E.g., "GI-2026-001". |
| `issue_date` | Date | Required | Date released. |
| `warehouse_id` | UUID | Required, Foreign Key | Links to `warehouses.id`. |
| `issue_type` | Text | Required | E.g., `internal_use`, `project`, `sample`. |
| `target_entity`| Text | Nullable | Name/ID of department or project. |
| `remarks` | Text | Nullable | Explanatory notes. |
| `status` | Text | Required | `draft`, `issued`, `cancelled`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `goods_issue_items` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `goods_issue_id`| UUID | Required, Foreign Key | Links to `goods_issues.id`. |
| `item_id` | UUID | Required, Foreign Key | Links to `items.id`. |
| `issue_qty` | Numeric(15,4) | Required | Amount being issued out. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

**Crucial Logic:** Upon approval (Status = `issued`), the system MUST automatically insert records into `inventory_movements` as a negative quantity.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
