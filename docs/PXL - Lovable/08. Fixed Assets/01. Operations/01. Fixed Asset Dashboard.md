# PXL ERP Blueprint: Fixed Asset Dashboard

## Module Overview
The Fixed Asset Dashboard is the command center for the entire property, plant, and equipment portfolio. It provides a real-time, consolidated view of total carrying amounts, upcoming depreciation runs, recent acquisitions, and impending asset retirements. By tracking overall asset values and threshold analytics, it guarantees visibility into the company's capital expenditures and readiness for BIR tax compliance and Audited Financial Statement reporting.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **Date Filter:** Adjust the analytics timeline (e.g., Year-to-Date, Specific Quarter).
* **Location/Branch Filter:** View assets specific to one branch or consolidated.
* **Refresh Button:** Sync real-time data.
* **Export Report:** Generate a consolidated PDF/Excel summary.

### The Dashboard Widgets (Data Views):
* **Total Asset Cost:** Sum of all capitalized costs.
* **Total Accumulated Depreciation:** Aggregated wear-and-tear.
* **Total Net Book Value:** Current carrying amount of the entire portfolio.
* **Acquisitions by Category (Chart):** A pie chart showing where capital is being invested.
* **Upcoming Retirements:** A list of assets nearing the end of their useful life or approaching their salvage value limit based on retirement rules.

## The Supabase Database Architecture

### Table 1: `fixed_asset_dashboard_configs` (Header)
Stores user-specific or system-wide layout preferences for the fixed asset analytics hub.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `view_name` | Text | Required | Label for the dashboard configuration. |
| `user_id` | UUID | Required, Foreign Key | Links to `users.id` to customize per user. |
| `is_default` | Boolean | Default: `false` | Indicates if it's the primary load view. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

### Table 2: `fixed_asset_dashboard_widgets` (Line Items)
Details the specific analytical blocks active on the dashboard.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `config_id` | UUID | Required, Foreign Key | Links to `fixed_asset_dashboard_configs.id`. |
| `widget_type` | Text | Required | ENUM: `kpi`, `chart`, `list`. |
| `data_source` | Text | Required | What module the widget pulls from. |
| `sort_order` | Integer | Required | Display order on the UI. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

*(Note: The actual transactional data displayed in these widgets is dynamically pulled from the `asset_register`, `asset_categories`, and `depreciation_runs` tables based on these configurations.)*
