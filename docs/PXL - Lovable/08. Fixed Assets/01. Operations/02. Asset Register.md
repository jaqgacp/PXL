# PXL ERP Blueprint: Asset Register

## Module Overview
The Asset Register is the master database of all capitalized Property, Plant, and Equipment (PPE) within the company. It serves as the single source of truth for tracking asset location, custodian, carrying amounts, and lifespan. The register enforces the strict capitalization threshold (ensuring non-qualifying assets are excluded) and applies specific depreciation schedules and asset retirement rules (such as estimated salvage value retention) to guarantee accurate Audited Financial Statements and BIR compliance.

## The Supabase Database Architecture

### Table 1: `asset_register` (Header)
The core table containing all fixed asset master records.

**Critical Database Rules:**
1. `UNIQUE(company_id, asset_tag)` — prevents cross-tenant asset tag duplication.
2. `net_book_value` is **NOT a writable column**. It is a `GENERATED ALWAYS AS` computed column calculated as `acquisition_cost - accumulated_depreciation - total_impairment_loss`. No application code may write to this column — it updates automatically whenever `accumulated_depreciation` or `total_impairment_loss` is updated by the Depreciation Run or Impairment modules.
3. `company_id` is required on this table and all associated tables for multi-tenant RLS.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `asset_tag` | Text | Required | `UNIQUE(company_id, asset_tag)`. The tracking barcode/tag number. |
| `asset_name` | Text | Required | The asset's descriptive name. |
| `category_id` | UUID | Required, Foreign Key | Links to `asset_categories.id`. |
| `serial_number` | Text | Nullable | OEM serial number. |
| `location_id` | UUID | Required, Foreign Key | Links to `locations.id` or `branches.id`. |
| `custodian_id` | UUID | Nullable, Foreign Key | Links to `employees.id`. |
| `acquisition_date` | Date | Required | Date acquired/ready for use. |
| `acquisition_cost` | Numeric(15,2) | Required | Total capitalized cost (checked against threshold). |
| `salvage_value` | Numeric(15,2) | Required | Estimated residual value upon asset retirement. |
| `useful_life_months` | Integer | Required | Duration of the depreciation schedule. |
| `depreciation_profile_id` | UUID | Required, Foreign Key | Links to `depreciation_profiles.id`. |
| `accumulated_depreciation` | Numeric(15,2) | Required, Default 0 | Cumulative depreciation posted by Depreciation Runs. Updated by system, never by users. |
| `total_impairment_loss` | Numeric(15,2) | Required, Default 0 | Cumulative impairment losses posted. Updated by system, never by users. |
| `net_book_value` | Numeric(15,2) | **GENERATED ALWAYS AS** `(acquisition_cost - accumulated_depreciation - total_impairment_loss) STORED` | The current carrying amount. Read-only — computed by the database. Any code that tries to INSERT or UPDATE this column will be rejected by PostgreSQL. |
| `asset_gl_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. The asset's cost GL account. |
| `accum_dep_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. The accumulated depreciation GL account. |
| `status` | Text | Required | ENUM: `active`, `fully_depreciated`, `disposed`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**PostgreSQL DDL for Computed NBV Column:**
```sql
ALTER TABLE asset_register
ADD COLUMN net_book_value NUMERIC(15,2)
  GENERATED ALWAYS AS (acquisition_cost - accumulated_depreciation - total_impairment_loss) STORED;
```

### Table 2: `asset_lifecycle_history` (Line Items)
An audit log capturing every location transfer, custodian change, and lifecycle status shift.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `asset_id` | UUID | Required, Foreign Key | Links to `asset_register.id`. |
| `event_type` | Text | Required | ENUM: `creation`, `transfer`, `revaluation`, `disposal`. |
| `event_date` | Date | Required | When the event occurred. |
| `description` | Text | Required | Details of the lifecycle event. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |
