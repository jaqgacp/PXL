# PXL ERP Blueprint: Asset Transfer

## Module Overview
The Asset Transfer module tracks the movement of fixed assets between branches, departments, or cost centers. For intra-company transfers (same legal entity, different locations), the module updates the asset's location and custodian in the Asset Register and posts an optional inter-branch cost center reallocation journal entry where applicable. For inter-entity transfers (different legal entities within the group), a "Due To / Due From" intercompany GL entry is required and mandatory.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **Search Bar:** Search by Transfer No. or Asset Tag.
* **Filter:** Toggle by Status (Draft, In-Transit, Completed).
* **Customize View:** Select which columns to display.
* **Export Button:** Download transfer logs to Excel/CSV.
* **Create New Transfer Button:** Initiate an asset relocation request.

### The Data List:
Default columns displayed:
* Transfer No.
* Date Requested
* Origin Branch
* Destination Branch
* Asset Count
* Transfer Type
* Status

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 50 records.

## GL Posting Logic

### Intra-Branch Transfer (Same Legal Entity)
No financial GL posting is required — the asset value does not change between branches of the same company. On completion, the system only:
1. Updates `asset_register.location_id` to the destination branch.
2. Updates `asset_register.custodian_id` to the new custodian.
3. Writes a record to `asset_lifecycle_history` with `event_type = 'transfer'`.

*Note: If the origin and destination branches have separate cost centers, the depreciation allocation for subsequent periods will automatically shift to the destination cost center. No manual JE is needed because depreciation is recalculated per asset register location.*

### Inter-Entity Transfer (Different Legal Entities)
A double-entry GL posting is mandatory. The asset is transferred at its current Net Book Value.

**On Source Entity:**
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Accumulated Depreciation | Accumulated Dep. Balance | | Asset category account |
| 2 | Due To Destination Entity | | Asset NBV at Transfer | User-selected `due_to_account_id` |
| 3 | Asset Account | | Original Cost | Asset category account |

**On Destination Entity (requires a mirroring JE in the destination company):**
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Asset Account (at NBV) | Asset NBV at Transfer | | Asset category account |
| 2 | Due From Source Entity | | Asset NBV at Transfer | User-selected `due_from_account_id` |

## The Supabase Database Architecture

### Table 1: `asset_transfers` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, transfer_no)` — prevents cross-tenant number duplication.
2. `journal_entry_id` is populated on completion for Inter-Entity transfers. Null for Intra-Branch.
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `transfer_no` | Text | Required | `UNIQUE(company_id, transfer_no)`. |
| `request_date` | Date | Required | Date the transfer was initiated. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `transfer_type` | Text | Required | ENUM: `intra_branch`, `inter_entity`. |
| `origin_location_id` | UUID | Required, Foreign Key | Links to `locations.id` or `branches.id`. |
| `destination_location_id` | UUID | Required, Foreign Key | Links to `locations.id` or `branches.id`. |
| `destination_company_id` | UUID | Nullable, Foreign Key | Links to `companies.id`. Required if `transfer_type = 'inter_entity'`. |
| `due_to_account_id` | UUID | Nullable, Foreign Key | Due To GL account. Required if `transfer_type = 'inter_entity'`. |
| `due_from_account_id` | UUID | Nullable, Foreign Key | Due From GL account. Required if `transfer_type = 'inter_entity'`. |
| `remarks` | Text | Nullable | Reason for transfer. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated for Inter-Entity transfers on completion. |
| `posted_at` | Timestamp | Nullable | System-set on completion of Inter-Entity transfer. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required | ENUM: `draft`, `in_transit`, `completed`, `cancelled`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, transfer_no)`

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on completed or cancelled transfers
CREATE POLICY "block_update_on_completed_transfer"
ON asset_transfers
FOR UPDATE
USING (status IN ('draft', 'in_transit'));

-- Block DELETE on all transfers
CREATE POLICY "block_delete_all_transfers"
ON asset_transfers
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_transfers"
ON asset_transfers
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: tenant isolation
CREATE POLICY "tenant_isolation_transfer_lines"
ON asset_transfer_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
