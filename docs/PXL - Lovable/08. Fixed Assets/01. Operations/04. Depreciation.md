# PXL ERP Blueprint: Depreciation

## Module Overview
The Depreciation module automates the monthly or annual calculation of wear-and-tear for all active assets. Using the pre-defined depreciation schedules (Straight-Line, Declining Balance) configured in the Asset Register, this module executes batch calculations. It ensures that the Net Book Value never drops below the specified salvage value as dictated by asset retirement rules. Approved depreciation runs automatically generate the necessary journal entries in the General Ledger.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **Search Bar:** Find a run by Document Number or Period.
* **Filter:** Toggle by Status (Draft, Posted).
* **Customize View:** Adjust visible columns.
* **Export Button:** Download the depreciation run schedule for the period.
* **Generate Run Button:** Initiates the background calculation for a specific month/period.

### The Data List:
Default columns displayed:
* Run No.
* Period (e.g., June 2026)
* Total Assets Processed
* Total Depreciation Expense
* Status (Draft / Posted)

## The Supabase Database Architecture

### Table 1: `depreciation_runs` (Header)
Captures the execution log for batch depreciation calculations.

**Critical Database Rules:**
1. Composite unique constraint on `(company_id, run_no)` — prevents cross-tenant duplication.
2. Idempotency guard: `UNIQUE(company_id, fiscal_period_id)` prevents running depreciation twice for the same period in the same company.
3. `company_id` is mandatory for multi-tenant isolation.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. The period being depreciated. |
| `run_no` | Text | Required | Document tracking number (e.g., `DR-2026-06`). |
| `period_date` | Date | Required | The accounting month being depreciated (first day of month). |
| `run_date` | Timestamp | Required | Exact date and time of calculation. |
| `total_expense` | Numeric(15,2) | Required | The aggregated depreciation expense to be posted. |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `posted`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraints:**
- `UNIQUE(company_id, run_no)`
- `UNIQUE(company_id, fiscal_period_id)` — idempotency guard against double-runs.

### Table 2: `depreciation_run_lines` (Line Items)
Stores the precise calculated impact on every individual asset during the run.

**Critical Database Rule:** Composite unique constraint on `(run_id, asset_id)` — prevents the same asset from appearing twice within a single depreciation run (double-click / race condition guard).

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `run_id` | UUID | Required, Foreign Key | Links to `depreciation_runs.id` (ON DELETE CASCADE). |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Denormalized for RLS efficiency. |
| `asset_id` | UUID | Required, Foreign Key | Links to `asset_register.id`. |
| `prior_nbv` | Numeric(15,2) | Required | Net book value before this deduction. |
| `depreciation_amount` | Numeric(15,2) | Required | The expense allocated for this period. |
| `new_nbv` | Numeric(15,2) | Required | The resulting Net Book Value (`prior_nbv - depreciation_amount`). |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(run_id, asset_id)`
