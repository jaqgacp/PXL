# PXL ERP Blueprint: Posting Controls

## Module Overview
Posting Controls dictate how operational transactions translate into financial entries in the General Ledger. In compliance with BIR CAS requirements, posted entries must not be easily deleted. This module specifies which documents generate GL entries, whether they post automatically or manually, and what happens when validation fails.

## What You Will See on the Screen (The Dashboard)
The interface shows posting rules mapped to different modules.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Module or Document Type.
* **Filter:** Filter by Posting Mode (Auto/Manual).
* **Export Button:** Export controls to CSV/Excel.
* **Edit Controls Button:** Modify global posting behaviors.

### The Data List (The main table):
* Document Type
* GL Impact (Yes/No)
* Posting Mode (Auto upon Approval, Manual Batch)
* Allow Background Posting (Yes/No)

---

## What Information We Need to Capture (The Data Fields)

### Section 1: Posting Rule Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Document Type | Purchase Invoice, Sales Invoice, etc. | Yes |
| Has GL Impact | Toggle if this document generates JVs. | Yes |
| Posting Mode | Auto (real-time) or Batch (EOD). | Yes |
| Block on Warning | Stop posting if soft validation warnings occur. | Yes |

## The Supabase Database Architecture (For Lovable)

**Critical Database Rule for Lovable:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `posting_controls`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `document_type` | Text | Required | The transaction type (e.g., `sales_invoice`, `purchase_bill`, `payment_voucher`). |
| `has_gl_impact` | Boolean | Default: `true` | Does this document type generate GL entries? |
| `posting_mode` | Text | Required | `auto` (real-time on approval), `batch` (EOD run), `manual`. |
| `block_on_warning` | Boolean | Default: `false` | Halt posting if soft validation warnings are triggered. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

**Composite Unique Constraint:** `UNIQUE(company_id, document_type)`
