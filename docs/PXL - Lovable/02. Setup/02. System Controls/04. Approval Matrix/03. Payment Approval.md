# PXL ERP Blueprint: Payment Approval Matrix

> **ARCHITECTURE NOTE:** This module does NOT have its own database table.
>
> Payment approval workflows are managed by the **Unified Approval Workflow** system defined in `01. Unified Approval Workflow.md`. Use the `approval_workflows` table with `module_type = 'payment'`.

## Module Overview
The Payment Approval Matrix is an essential internal control for cash disbursements. In Philippine accounting practice, the release of funds (via Check Vouchers or Online Bank Transfers) requires multiple signatories based on the amount. This module standardizes the review of Payment Vouchers before treasury execution.

---

## Applicable Document Types

| Document Type Value | What It Controls |
| :--- | :--- |
| `check_voucher` | Supplier payments via check |
| `petty_cash_voucher` | Petty cash expense disbursements |
| `fund_transfer` | Inter-bank or inter-branch fund transfers |
| `ap_payment` | General accounts payable payments |

---

## Common Trigger Conditions for Payments

| Trigger | When to Use |
| :--- | :--- |
| `amount_exceeds` + 50000 | Require second signatory for mid-range payments |
| `amount_exceeds` + 500000 | Require CFO/CEO sign-off for large disbursements |
| `always` | Require all check vouchers to have two approvers |

---

## Database Implementation

```sql
SELECT * FROM approval_workflows
WHERE company_id = :company_id
  AND module_type = 'payment'
  AND is_active = true;
```
