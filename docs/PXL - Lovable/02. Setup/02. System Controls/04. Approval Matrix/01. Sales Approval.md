# PXL ERP Blueprint: Sales Approval Matrix

> **ARCHITECTURE NOTE:** This module does NOT have its own database table.
>
> Sales approval workflows are managed by the **Unified Approval Workflow** system defined in `01. Unified Approval Workflow.md`. When configuring a Sales approval rule, use the `approval_workflows` table with `module_type = 'sales'`.
>
> **Example record:**
> - `module_type`: `sales`
> - `document_type`: `sales_order` | `quotation` | `credit_note`
> - `trigger_condition_type`: `amount_exceeds` | `discount_pct_exceeds`
> - `threshold_value`: 50000

## Module Overview
The Sales Approval Matrix controls authorization for high-value or high-discount sales transactions. Quotations, Sales Orders, and Credit Notes that meet the configured thresholds are routed to designated approvers before they can be confirmed or invoiced.

---

## Dashboard UI

### The Action Bar
* **Filter:** By Document Type (Quotation, Sales Order, Credit Note), Status (Active/Inactive).
* **Create New Workflow Button:** Opens the unified workflow builder pre-filtered to `module_type = 'sales'`.

### The Data List
Displays all `approval_workflows` records WHERE `module_type = 'sales'` for the current company.

* Workflow Name
* Document Type (Quotation / Sales Order / Credit Note)
* Trigger Condition (Amount > X or Discount > X%)
* Number of Steps
* Status (Active / Inactive)

---

## Applicable Document Types

| Document Type Value | What It Controls |
| :--- | :--- |
| `quotation` | Quotations / Price Offers |
| `sales_order` | Sales Order confirmations |
| `credit_note` | Credit Memos issued to customers |
| `sales_return` | Return of goods from customers |

---

## Common Trigger Conditions for Sales

| Trigger | When to Use |
| :--- | :--- |
| `amount_exceeds` + threshold | Route large orders to management review |
| `discount_pct_exceeds` + threshold | Prevent unauthorized discounting below margin |
| `always` | Require approval on every Sales Order regardless of value |

## Database Implementation

All data is stored in the unified tables. No separate `sales_approval_workflows` table should be created.

**Query to retrieve all Sales workflows:**
```sql
SELECT * FROM approval_workflows
WHERE company_id = :company_id
  AND module_type = 'sales'
  AND is_active = true;
```

**Query to get pending Sales approvals for a user:**
```sql
SELECT ai.*, aw.document_type, aw.workflow_name
FROM approval_instances ai
JOIN approval_workflows aw ON aw.id = ai.workflow_id
WHERE ai.company_id = :company_id
  AND ai.required_approver_id = :user_id
  AND ai.status = 'pending'
  AND aw.module_type = 'sales';
```

See `01. Unified Approval Workflow.md` for the full schema of `approval_workflows`, `approval_workflow_steps`, and `approval_instances`.
