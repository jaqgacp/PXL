# PXL ERP Blueprint: ATP Monitoring

## Module Overview
The ATP (Available-to-Promise) Monitoring module provides real-time visibility into inventory availability against sales commitments. In Philippine distribution and retail businesses, ensuring that promised inventory can actually be delivered is critical to avoiding stockouts and maintaining customer satisfaction. This module cross-references on-hand inventory, pending purchase orders, and unfulfilled sales orders to compute the true available quantity, helping sales teams make accurate commitments.

## What You Will See on the Screen (The Dashboard)
When you open the "ATP Monitoring" page, you will see a comprehensive view of item availability across warehouses.

### The Action Bar (Top of the page):
* **Search Bar:** Quickly find an item by Item Code or Description.
* **Filter:** Filter by Warehouse, Item Category, or ATP Status (Available, Low Stock, Negative).
* **Customize View:** Show or hide specific columns like Reserved Quantity or Incoming POs.
* **Export Button:** Download the ATP report to Excel or CSV for offline analysis.
* **Refresh Button:** Force a real-time recalculation of ATP values.

### The Data List (The main table):
* Item Code
* Item Description
* Warehouse
* On-Hand Quantity
* Reserved Quantity (from Sales Orders)
* Incoming Quantity (from Purchase Orders)
* ATP Quantity (Available to Promise)
* Status (Color-coded: Green for Available, Red for Critical)

## What Information We Need to Capture (The Data Fields)
This is typically a system-calculated view rather than a data entry form. However, users can click an item to view the breakdown of its ATP.

### Section 1: Item Summary (Header)
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Item Code | The unique identifier of the item. | Yes |
| Description | Full name of the item. | Yes |
| Warehouse | The specific location being queried. | Yes |
| Calculation Date | Timestamp of the last ATP calculation. | Yes |

### Section 2: ATP Breakdown (Line Items)
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Transaction Type | "Sales Order", "Purchase Order", "Transfer", etc. | Yes |
| Document Number | Reference to the specific transaction affecting ATP. | Yes |
| Transaction Date | Date of the transaction. | Yes |
| Quantity Effect | Positive (incoming) or Negative (outgoing) impact on inventory. | Yes |

## The Supabase Database Architecture (For Lovable)

**Critical Database Rule for Lovable:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `atp_snapshots` (Header)
Stores the cached ATP calculations for items to ensure fast dashboard loading.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the snapshot. |
| `item_id` | UUID | Required, Foreign Key | Links to master `items.id`. |
| `warehouse_id` | UUID | Required, Foreign Key | Links to master `warehouses.id`. |
| `on_hand_qty` | Numeric | Required | Current physical stock. |
| `reserved_qty` | Numeric | Required | Stock promised to SOs. |
| `incoming_qty` | Numeric | Required | Stock expected from POs. |
| `atp_qty` | Numeric | Required | Calculated available stock. |
| `status` | Text | Required | `available`, `low_stock`, `critical`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `atp_transactions` (Line Items)
Stores the individual document lines that make up the reserved or incoming quantities.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the transaction line. |
| `atp_snapshot_id` | UUID | Required, Foreign Key | Links to `atp_snapshots.id`. |
| `document_type` | Text | Required | E.g., `sales_order`, `purchase_order`. |
| `document_id` | UUID | Required | Generic ID to the source document. |
| `document_no` | Text | Required | Display reference number. |
| `expected_date` | Date | Required | Delivery or Receipt date. |
| `qty_effect` | Numeric | Required | Positive or negative quantity. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
