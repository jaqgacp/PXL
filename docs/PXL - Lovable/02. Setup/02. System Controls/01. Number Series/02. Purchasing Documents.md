# PXL ERP Blueprint: Number Series — Purchasing Documents

## Module Overview
This page manages the auto-numbering sequences for all Purchasing-related transaction documents in PXL ERP. A correctly configured purchasing number series ensures that every procurement document is uniquely identifiable, traceable in the audit log, and consistent across all branches. Unlike Sales documents, most purchasing documents do not require BIR-registered serial numbers (as they are internal documents), giving the company more flexibility in defining its own numbering conventions.

The Number Series for Purchasing Documents covers the following transaction types:
* Purchase Orders
* Receiving Reports
* Vendor Bills
* Cash Purchases
* Payment Vouchers
* Vendor Credits
* Debit Memos to Suppliers
* Purchase Returns

> **Reference:** This module shares the same `number_series` Supabase table defined in the **Sales Documents** blueprint. Please refer to that document for the complete database schema. This page documents only the Purchasing-specific configuration and business rules.

## What You Will See on the Screen (The Dashboard)
When you open the "Number Series — Purchasing Documents" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a series by Document Type, Prefix, or Branch.
* **Company Filter:** A dropdown to filter series by parent company.
* **Branch Filter:** A dropdown to filter by branch, making it easy to audit all purchasing series per location.
* **Document Type Filter:** A dropdown to show the series for only one specific document type (e.g., show only Purchase Order series).
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of purchasing number series records by uploading a pre-formatted CSV or Excel file. Useful for batch setup across multiple branches during system go-live.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for review or documentation.
* **Print Button:** Generates a print-ready view of all configured purchasing series for hardcopy reference.
* **Create New Series Button:** A prominent button to add a new number series record.

### The Data List (The main table - must be customizable):
You will see a list of all configured purchasing document series. The default columns displayed are:
* Document Type
* Company
* Branch
* Prefix
* Current / Next Number (e.g., PO-000023)
* Reset Frequency
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that number series.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Series" or "Edit," a form will open. Here is the exact data the system will save.

### Section 1: Series Scope & Assignment
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. The entity this number series belongs to. | Yes |
| Branch | Dropdown linked to active branches. Purchasing series should be assigned per branch to allow each location to independently track its own procurement documents. | Yes |
| Document Type | Dropdown: Purchase Order, Receiving Report, Vendor Bill, Cash Purchase, Payment Voucher, Vendor Credit, Debit Memo to Supplier, Purchase Return. | Yes |

### Section 2: Number Format Configuration
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Prefix | The text prepended to the number (e.g., "PO-", "RR-", "PV-", "VB-"). Can include the year or branch code (e.g., "PO-2026-"). | Optional |
| Suffix | The text appended after the number. | Optional |
| Number Length (Padding) | The total number of digits in the numeric portion, zero-padded (e.g., 6 produces "000001"). | Yes |
| Starting Number | The first number in the sequence. Defaults to 1. | Yes |
| Next Available Number | The next number the system will assign. Auto-increments after each confirmed document. Can be manually corrected by an admin. | Yes |

### Section 3: Reset & Control Settings
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Reset Frequency | Dropdown: Never (continuous), Reset Yearly, Reset Monthly. Yearly resets are common for Payment Vouchers (e.g., PV-2026-000001). | Yes |
| Allow Manual Override | Toggle (Yes / No). If YES, users with sufficient permissions can manually enter a document number (useful for encoding supplier-issued document numbers on Vendor Bills). | Yes |

### Section 4: Format Preview
This is a display-only section that shows a live preview of the generated document number based on the configuration above.

| Field Name | Description |
| :--- | :--- |
| Format Preview | Auto-rendered display (e.g., **PO-000023** or **PV-2026-000001**). Updates in real-time as the user configures the fields above. |

## The Supabase Database Architecture (For Lovable)

### Shared Table: `number_series`
The Purchasing Documents number series uses the same `number_series` table defined in the **Sales Documents** blueprint. No additional tables are required for this module.

The `document_type_id` field links to the `ref_document_types` table. The records pre-seeded for Purchasing are:

| `document_code` | `document_name` | `category` | `is_bir_registered` |
| :--- | :--- | :--- | :--- |
| `PO` | Purchase Order | `purchasing` | `false` |
| `RR` | Receiving Report | `purchasing` | `false` |
| `VB` | Vendor Bill | `purchasing` | `false` |
| `CP` | Cash Purchase | `purchasing` | `false` |
| `PV` | Payment Voucher | `purchasing` | `false` |
| `VC` | Vendor Credit | `purchasing` | `false` |
| `DMS` | Debit Memo to Supplier | `purchasing` | `false` |
| `PR` | Purchase Return | `purchasing` | `false` |
