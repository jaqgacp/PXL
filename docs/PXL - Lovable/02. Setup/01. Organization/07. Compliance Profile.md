# PXL ERP Blueprint: Compliance Profile

## Module Overview
The Compliance Profile is the central configuration record that defines a company's tax obligations and filing responsibilities with the Bureau of Internal Revenue (BIR). It captures which tax types a company is registered for, the applicable filing frequencies, the relevant BIR forms, due dates, and EFPS enrollment status. This profile is the engine that drives the entire Compliance module — it determines which tax returns, working papers, and BIR books are generated, which dashboards are shown, and which compliance alerts are triggered throughout the system. Every company must have a completed Compliance Profile before any compliance-related features are activated.

## What You Will See on the Screen (The Dashboard)
When you open the "Compliance Profile" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a compliance profile by Company Name.
* **Filter:** A dropdown to show either "Active" or "Inactive" profiles.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of compliance profiles by uploading a pre-formatted CSV or Excel file. Useful for efficiently onboarding multiple companies or subsidiaries during initial system setup.
* **Export Button:** Downloads the current list to an Excel or CSV file for compliance review or external auditors.
* **Print Button:** Generates a print-ready summary of all compliance profiles for hardcopy BIR documentation.
* **Create New Profile Button:** A prominent button to add a compliance profile for a new company.

### The Data List (The main table - must be customizable):
You will see a list of all compliance profiles. The default columns displayed are:
* Company
* Tax Registration Type (VAT / Non-VAT / Exempt)
* Income Tax Regime
* TWA Status (Badge if Top Withholding Agent)
* EFPS Enrolled
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete compliance profile for that company.

## The Supabase Database Architecture (For Lovable)
This module requires one main table. It is a one-to-one record per company and serves as the configuration backbone for the entire Compliance module.

### Table 1: `compliance_profiles` (The Main Table)
This is the primary configuration table for all tax obligations of each company. There is only one record per company.

**Critical Database Rule for Lovable:** Apply a UNIQUE constraint on the `company_id` column to ensure a company cannot have more than one compliance profile.

## TWA Automatic EWT Enforcement — Workflow Specification

**This section is mandatory reading for frontend developers building the Purchase Invoice module.**

When `compliance_profiles.is_twa = true` AND `compliance_profiles.twa_auto_ewt_enabled = true` for the active company, the following behavior is enforced system-wide on every Purchase Invoice:

### Trigger Point
The EWT computation is triggered when a user adds or modifies a line item on a Purchase Invoice.

### EWT Code Resolution (Priority Order)
1. Check `purchase_invoice_lines.item_id` → `items.default_ewt_code_id`. If set, use this.
2. Fall back to `purchase_invoices.supplier_id` → `suppliers.default_ewt_code_id`. If set, use this.
3. If `suppliers.ewt_exempt = true` → Apply zero EWT. Log the exemption. Require exemption certificate URL to be on file.
4. If no EWT code can be resolved and supplier is not exempt → **BLOCK the line save.** Alert: "EWT code is not assigned to this supplier. Cannot proceed without EWT assignment for TWA compliance."

### Computation
```
EWT Amount per line = net_amount × (ewt_rate / 100)
```

### User Override
The user **cannot** manually zero out or reduce the auto-computed EWT amount. Any override requires:
1. A supervisor-role user to approve the override request within the system.
2. A mandatory text reason logged to an audit table (`ewt_override_log`).
3. The override is flagged on the MAP/QAP DAT file for BIR scrutiny.

### On Posting
1. EWT amount is credited to the EWT Payable GL account (from `ewt_codes` → `tax_codes.gl_account_id`).
2. A record is created in `form_2307_issued` linking this Purchase Invoice to the BIR 2307 obligation.
3. The EWT is included in the current quarter's MAP/QAP computation automatically.

### Standard TWA Rates (if ATC cannot be resolved)
| Purchase Type | Default ATC | Default Rate |
| :--- | :--- | :--- |
| Goods | WC158 | 1% |
| Services | WC158 | 2% |
