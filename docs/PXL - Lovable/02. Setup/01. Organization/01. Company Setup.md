# PXL ERP Blueprint: Company Setup

## Module Overview
The Company Setup is the foundation of the ERP. It stores the primary legal, operational, and tax details of the business entity. Because the system is designed for Philippine compliance, this page captures everything needed to automatically populate the headers of Financial Statements, CAS (Computerized Accounting System) reports, and all BIR tax returns and certificates. It also supports parent-subsidiary hierarchies, establishing the structural foundation needed for seamless consolidated financial reporting across multiple corporate entities.

## What You Will See on the Screen (The Dashboard)
When you open the "Company Setup" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to easily find a company by typing its Name or TIN.
* **Filter:** A dropdown to show either "Active" or "Inactive" companies.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for reporting or offline review.
* **Print Button:** Generates a print-ready view of the company list for hardcopy documentation.
* **Create New Company Button:** A prominent button to add a new business entity.
* **Import Button:** Allows bulk creation of Companies by uploading a pre-formatted CSV or Excel file. Useful for onboarding accounting clients with multiple existing branches or subsidiaries at once.

### The Data List (The main table - must be customizable):
You will see a list of all your created companies. The default columns displayed are:
* Logo (Small image)
* Registered Name
* Parent Company (If applicable)
* TIN
* RDO Code
* Taxpayer Type
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that company.

## The Supabase Database Architecture (For Lovable)
This is the exact configuration we will provide to Lovable to build the database. We will create two tables.

### Table 1: `ref_rdo_codes` (Reference Table)
This is the master list of all BIR Revenue District Offices (RDOs) in the Philippines. We will pre-load this once to provide a dropdown selection for the user.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `rdo_code` | Text | Required, Unique | E.g., "043A", "053B". |
| `rdo_name` | Text | Required | E.g., "East Pasig", "Muntinlupa City". |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `companies` (The Main Table)
This is the core table that will store all the entity records inputted via the Company Setup dashboard. It includes a self-referencing relationship for building holding-subsidiary structures.

**Critical Database Rule for Lovable:** Ensure a unique constraint is placed on `tin` to prevent duplicate companies from being registered.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the company. |
| `parent_company_id` | UUID | Nullable, Foreign Key | Self-references `companies.id` to map subsidiaries to holding companies. |
| `entity_type` | Text | Required | `sole_proprietor`, `opc`, `corporation`, `partnership`, `cooperative`. |
| `registered_name` | Text | Required | Legal Name. |
| `trade_name` | Text | Nullable | Trade Name. |
| `line_of_business` | Text | Required | Main business activity. |
| `psic_code` | Text | Nullable | PSIC Code. |
| `tin` | Text | Required, Unique | 13-15 digit string (e.g., "123-456-789-00000"). |
| `tax_registration` | Text | Required | `vat`, `non_vat`, `exempt`. |
| `rdo_id` | UUID | Required, Foreign Key | Links to `ref_rdo_codes` table. |
| `registration_number`| Text | Nullable | Holds the SEC, DTI, or CDA number. |
| `bir_reg_date` | Date | Nullable | BIR Registration date. |
| `sec_dti_reg_date` | Date | Nullable | SEC/DTI Registration date. |
| `lgu_reg_date` | Date | Nullable | Mayor's Permit date. |
| `accounting_period` | Text | Required | `calendar` or `fiscal`. |
| `fiscal_start_month`| Integer | Nullable | Number from 1 to 12. |
| `cas_permit_no` | Text | Nullable | CAS or PTU Number for receipts. |
| `cas_date_issued` | Date | Nullable | Date CAS/PTU was issued. |
| `address_line_1` | Text | Required | Street, Building. |
| `address_line_2` | Text | Required | Barangay, Subdivision. |
| `city` | Text | Required | City/Municipality. |
| `province` | Text | Required | Province. |
| `zip_code` | Text | Required | 4-digit code. |
| `email` | Text | Required | Company email. |
| `phone_number` | Text | Nullable | Landline. |
| `mobile_number` | Text | Nullable | Mobile. |
| `signatory_name` | Text | Required | Name of authorized person. |
| `signatory_position` | Text | Required | Job title of authorized person. |
| `signatory_tin` | Text | Required | TIN of authorized person. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
