# PXL ERP Blueprint: GL Posting Configuration

## Module Overview
GL Posting Configuration (also known as Account Determination or Posting Matrix) connects operational modules to specific General Ledger accounts automatically. When a Sales Invoice is saved, this configuration determines which Accounts Receivable and Sales Revenue accounts to debit and credit, eliminating manual bookkeeping errors.

## What You Will See on the Screen (The Dashboard)
A mapping grid categorized by business module.

### The Action Bar (Top of the page):
* **Module Selector:** Sales, Purchasing, Inventory, Tax.
* **Export Button:** Export configurations.
* **Edit Matrix Button:** Modify the default accounts.

### The Data List (The main table):
* Transaction Event (e.g., "Sales Invoice Posted")
* Debit Account (e.g., Accounts Receivable - Trade)
* Credit Account (e.g., Sales Revenue)

## The Supabase Database Architecture (For Lovable)

**Critical Database Rule for Lovable:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `gl_posting_configurations`

**Precedence Rule for Account Resolution (highest to lowest):**
1. Item-level default GL account (stored on the item/service master record).
2. Category-level override (`category_id` on this table).
3. Module-level default (this table, `category_id IS NULL`).
4. If no mapping found at any level → Block the posting and alert the user. Never silently skip.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `module_name` | Text | Required | E.g., `sales`, `purchasing`, `inventory`, `tax`. |
| `transaction_event` | Text | Required | E.g., `sales_invoice_posted`, `purchase_invoice_posted`. |
| `posting_type` | Text | Required | `debit` or `credit`. |
| `account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. The target GL account. |
| `category_id` | UUID | Nullable, Foreign Key | Links to `item_categories.id`. If populated, this row is a category-level override. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

**Composite Unique Constraint:** `UNIQUE(company_id, module_name, transaction_event, posting_type, category_id)`
*(`category_id` is NULLS NOT DISTINCT in this constraint — PostgreSQL 15+ syntax required.)*
