# PXL ERP Blueprint: Recurring Journal Templates

## Module Overview
The Recurring Journal Templates module streamlines the accounting process by automating repetitive period-end entries, such as standard monthly prepayments, regular payroll allocations, or fixed rent accruals. In Philippine enterprise accounting, efficiency in the month-end close is critical. This module eliminates manual data entry for predictable entries by allowing users to define a template, its frequency, and execution rules, minimizing human error and ensuring consistency across financial periods.

## Dashboard UI
The interface provides a clear view of all active and inactive recurring templates.

### The Action Bar
* **Search Bar:** Search templates by Name, Description, or Account Name.
* **Filter:** Filter by Status (Active, Paused, Expired) and Frequency (Monthly, Quarterly, Annually).
* **Customize View:** Adjust the displayed columns.
* **Export Button:** Export the list of templates to Excel/CSV.
* **Print Button:** Print the current view.
* **Create Template Button:** Opens the setup form for a new recurring journal.
* **Import Button:** Bulk import templates using a standardized CSV/Excel format.

### The Data List
* **Template Code:** Auto-generated identifier.
* **Template Name:** User-defined name (e.g., "Monthly Rent Accrual").
* **Frequency:** E.g., Monthly, Quarterly.
* **Next Run Date:** The upcoming date this template will generate a JE.
* **Total Amount:** The base value of the entry.
* **Status:** Active, Paused, Expired.
* **Actions:** Edit, Pause/Resume, Run Now.

### Line Item Data
**CRITICAL VALIDATION:** Just like standard JEs, templates must have `Total Debit = Total Credit`. 
**GL Posting Behavior:** Templates do not hit the GL directly. They generate standard Journal Entries. If `Auto-Post` is true, the generated JE immediately commits to the GL. If false, the JE sits in draft status for review.
