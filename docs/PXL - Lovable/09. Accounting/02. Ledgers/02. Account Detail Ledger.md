# PXL ERP Blueprint: Account Detail Ledger

## Module Overview
The Account Detail Ledger (also known as the Account History or GL Account View) provides a deep dive into the transaction history of a single specific General Ledger account. In Philippine accounting practice, this view is essential for accountants performing monthly reconciliations, audit substantiations, and variance analysis. It filters the massive General Ledger down to one account, displaying opening balances, a chronological sequence of debit and credit movements, and the ending balance for a specified date range.

## Dashboard UI
The interface functions as a specialized reporting and inquiry screen.

### The Action Bar
* **Account Selector:** A prominent dropdown to select the specific GL Account to view.
* **Date Range Filter:** "From" and "To" date pickers to isolate the period (e.g., current month, current year).
* **Export Button:** Exports the specific account ledger to Excel/CSV for reconciliation working papers.
* **Print Button:** Generates a clean PDF of the account history.
* **Import Button:** Not applicable (Read-only view).

### The Data List
The screen typically displays a summary header followed by the transaction lines.
**Header Summary:**
* Account Name and Code
* Opening Balance (as of the Start Date)
* Total Period Debits
* Total Period Credits
* Ending Balance (as of the End Date)

**Transaction Lines:**
* **Date:** Posting date.
* **Source:** The origin document (e.g., Sales Invoice, JE).
* **Reference:** External reference.
* **Description:** Transaction memo.
* **Debit:** Transaction debit amount.
* **Credit:** Transaction credit amount.
* **Running Balance:** The progressive balance after the line is applied.

## Supabase Database Architecture

The Account Detail Ledger relies entirely on querying the `general_ledger_entries` table. There is no separate table for this module. The architecture emphasizes high-performance querying.

To support this module efficiently, the database must have indexes on the `general_ledger_entries` table:

1.  **Index on `account_id` and `posting_date`:** Essential for rapidly calculating opening balances and fetching period transactions.

```sql
-- Conceptual representation of the required index
CREATE INDEX idx_gl_account_date ON general_ledger_entries (account_id, posting_date);
```

### Data Flow & Logic
1.  **Opening Balance Calculation:** The system queries the sum of debits and credits for the `account_id` where `posting_date < Start Date`. It applies the account's normal balance rule (Debit or Credit normal) to determine the starting figure.
2.  **Transaction Fetching:** The system retrieves all records from `general_ledger_entries` for the `account_id` where `posting_date` is between `Start Date` and `End Date`, ordered by `posting_date` ascending.
3.  **Running Balance Computation:** Processed on the application side or via a database window function to append the running balance to each row before rendering.
