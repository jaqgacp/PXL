# Quotations

## Module Overview
The Quotation module is the initial step in the sales workflow. It serves as a formal cost estimate or proposal provided to prospective clients. In Philippine business practices, a quotation is non-binding and non-posting—meaning it does not affect financial statements or inventory ledgers. However, it is critical for tracking the sales pipeline, projected revenues, and client negotiations. Approved quotations seamlessly transition into Sales Orders to eliminate redundant data entry.

## Dashboard UI
When navigating to the Quotations page, users will find a clean workspace displaying all client proposals.

### The Action Bar
* **Search Bar:** Quickly find a quotation by Quotation Number, Customer Name, or TIN.
* **Filter:** Dropdown to filter by Status (Draft, Pending Approval, Approved, Rejected, Expired).
* **Customize View:** Toggle visibility of data columns.
* **Export Button:** Download the filtered list as an Excel or CSV file.
* **Print Button:** Generate a printable summary of quotations.
* **Create Quotation Button:** Primary action to draft a new proposal.
* **Import Button:** Bulk upload historical quotations via CSV/Excel.

### The Data List
Default columns displayed:
* Quotation Date
* Quotation Number
* Customer Name
* Validity Date
* Total Amount
* Status
* Created By

## Supabase Database Architecture


**Critical Database Rule for Lovable:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `sales_quotations` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the quotation. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `quotation_number` | Text | Required, Unique | Formatted document number (e.g., QT-2023-0001). |
| `quotation_date` | Date | Required | Date the document was created. |
| `validity_date` | Date | Required | Expiration date of the offer. |
| `currency_id` | UUID | Required, Foreign Key | Links to `ref_currencies.id`. |
| `reference_number` | Text | Nullable | Customer RFQ or external reference. |
| `remarks` | Text | Nullable | Internal or external notes. |
| `total_amount` | Numeric | Required | Total value of the quotation. |
| `status` | Text | Required | `draft`, `pending`, `approved`, `rejected`, `expired`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `sales_quotation_lines` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the line item. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |

| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
