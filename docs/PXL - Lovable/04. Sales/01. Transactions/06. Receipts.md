# Receipts (Collections)

## Module Overview
The Receipts module (covering Official Receipts / Collection Receipts) records payments received from customers to settle their open Sales Invoices. In the Philippines, this document reduces Accounts Receivable and recognizes Cash in Bank/On Hand. For VAT purposes, particularly under EOPT where the Invoice is the primary document, the Receipt now primarily serves as proof of payment and AR settlement rather than the basis for Output VAT. However, proper tracking is crucial for bank reconciliation and 2307 (Creditable Withholding Tax) claiming.

### Document Flow & Data Inheritance
* **Source:** Payments are applied against posted **Sales Invoices**.
* **Inheritance:** The user selects a Customer, which populates a list of unpaid Sales Invoices. The user allocates the payment amount, withholding tax (2307), and discounts to these specific invoice lines. The `invoice_id` is linked to each `receipt_line`.

## Dashboard UI
A centralized view for treasury and collection teams to monitor incoming cash flows.

### The Action Bar
* **Search Bar:** Search by Receipt Number, Customer Name, or Check Number.
* **Filter:** Dropdown for Payment Mode or Status (Cleared, Bounced, Draft).
* **Customize View:** Toggle column visibility.
* **Export Button:** Export collection reports to Excel/CSV.
* **Print Button:** Generate physical OR/CR for the customer.
* **Receive Payment Button:** Enter a new collection.

### The Data List
Default columns displayed:
* Receipt Date
* Receipt Number
* Customer Name
* Payment Mode
* Total Amount
* Applied CWT (2307)
* Status

## Supabase Database Architecture


**Critical Database Rule for Lovable:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `receipts` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the receipt. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `receipt_number` | Text | Required, Unique | Formatted document number (e.g., OR-2023-0001). |
| `receipt_date` | Date | Required | Date payment was received. |
| `payment_mode_id` | UUID | Required, Foreign Key | Links to `ref_payment_modes.id`. |
| `reference_number` | Text | Nullable | Check/Bank Trace No. |
| `bank_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id` (Bank). |
| `total_amount` | Numeric | Required | Total cash received. |
| `total_cwt` | Numeric | Default 0 | Total creditable withholding tax. |
| `status` | Text | Required | `draft`, `posted`, `cancelled`, `bounced`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
