# PXL ERP Blueprint: Cash Purchases

## Module Overview
The Cash Purchases module records immediate expenses or purchases where payment is made upfront without going through the standard Accounts Payable (AP) cycle (i.e., no Vendor Bill is created). This is commonly used for petty cash replenishments, direct debit expenses, or over-the-counter purchases. In the Philippines, this module must still capture tax details (Input VAT and EWT) if an Official Receipt or Cash Invoice is provided, ensuring full compliance for BIR reporting.

## What You Will See on the Screen (The Dashboard)
When you open the "Cash Purchases" page, you will see a clean workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Quickly find a transaction by Payee/Supplier Name or Reference Number.
* **Filter:** Dropdowns to filter by Date Range, Payment Account (Bank/Petty Cash), and Status.
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the list to Excel or CSV.
* **Print Button:** Generates a print-ready view of the Cash Purchases list.
* **Create New Cash Purchase Button:** Prominent button to record a new direct expense.
* **Import Button:** Bulk upload expenses from an Excel/CSV file.

### The Data List (The main table):
* Transaction Date
* Transaction Number
* Payee/Supplier Name
* Payment Account (Bank or Cash)
* Reference No. (OR/SI/Check)
* Total Amount
* Status (Draft, Posted, Cancelled)

## What Information We Need to Capture (The Data Fields)
The form is divided into Header and Line Items. 

**Data Inheritance:** A Cash Purchase is typically a standalone document. It does not inherit from POs or RRs. It generates an immediate accounting entry hitting the expense/asset account and crediting cash/bank.

## The Supabase Database Architecture (For Lovable)

**Critical Database Rule for Lovable:**
- Ensure a composite unique constraint on `company_id` and `transaction_number` if a `company_id` is used, otherwise `transaction_number` must be unique.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.

### Table 1: `cash_purchases` (Header Table)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the transaction. |
| `transaction_number`| Text | Required, Unique | Auto-generated number (e.g., "CP-2023-0001"). |
| `transaction_date`| Date | Required | Date of expense. |
| `payee_id` | UUID | Nullable, Foreign Key | Links to `suppliers.id` or `employees.id`. |
| `payee_name` | Text | Nullable | Manual name if not in masterlist. |
| `payment_account_id`| UUID | Required, Foreign Key | Links to `chart_of_accounts.id` (Cash/Bank). |
| `payment_method` | Text | Required | `cash`, `check`, `transfer`. |
| `reference_number`| Text | Nullable | Source document number. |
| `status` | Text | Required | `draft`, `posted`, `cancelled`. |
| `total_amount` | Numeric | Required | Total amount paid. |
| `remarks` | Text | Nullable | Reason for purchase. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Record creation timestamp. |
| `updated_at` | Timestamp | Auto | Record update timestamp. |

### Table 2: `cash_purchase_lines` (Line Items Table)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the line item. |
| `cash_purchase_id`| UUID | Required, Foreign Key | Links to `cash_purchases.id`. |
| `account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id` (Expense). |
| `item_id` | UUID | Nullable, Foreign Key | Links to `items.id` (if inventory/supplies). |
| `description` | Text | Nullable | Line detail. |
| `quantity` | Numeric | Nullable | Applicable for items. |
| `unit_price` | Numeric | Nullable | Applicable for items. |
| `gross_amount` | Numeric | Required | Pre-tax amount. |
| `vat_code_id` | UUID | Required, Foreign Key | Links to `tax_codes.id`. |
| `ewt_code_id` | UUID | Nullable, Foreign Key | Links to `tax_codes.id`. |
| `net_amount` | Numeric | Required | Final expense amount. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Record creation timestamp. |
| `updated_at` | Timestamp | Auto | Record update timestamp. |
