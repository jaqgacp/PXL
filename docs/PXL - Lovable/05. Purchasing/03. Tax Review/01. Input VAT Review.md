# PXL ERP Blueprint: Input VAT Review

## Module Overview
The Input VAT Review module is designed specifically for Philippine BIR compliance. It consolidates all valid Input Value-Added Tax recorded through Vendor Bills and Cash Purchases. This dashboard acts as the final verification step for accountants before generating the Summary List of Purchases (SLP) and filing the Monthly/Quarterly VAT Returns (BIR Form 2550M/2550Q). It ensures that only properly documented, TIN-bearing invoices are claimed as tax credits.

## What You Will See on the Screen (The Dashboard)
When you open the "Input VAT Review" page, you will see a detailed tax audit interface.

### The Action Bar (Top of the page):
* **Period Filter:** Dropdowns for Year and Month/Quarter. This is the primary filter.
* **Supplier Search:** Quickly isolate transactions for a specific vendor.
* **Tax Code Filter:** View transactions by type (e.g., Capital Goods, Services, Regular Goods, Exempt).
* **Export Button:** Downloads the data into an Excel format matching BIR Relief requirements.
* **Print Button:** Generates a summary report.
* **Lock Period Button:** A security feature to lock the VAT period, preventing further edits to underlying bills once tax returns are filed.

### The Data List (The main report table):
* Date (Invoice Date)
* Supplier Name
* Supplier TIN
* Invoice No. / OR No.
* Description
* Gross Amount
* VAT-Exempt Amount
* VATable Sales (Net of VAT)
* Input VAT Claimed

## What Information We Need to Capture (The Data Fields)
This is a purely read-only review and aggregation module. It pulls data precisely mapped to the requirements of the BIR SLP.

### Report Columns Map
| Column Name | Source | Description |
| :--- | :--- | :--- |
| Date | `bill_date` or `transaction_date`| The date on the physical receipt. |
| Supplier Name | `suppliers.registered_name` | Registered name of the vendor. |
| Supplier TIN | `suppliers.tin` | 9 to 13 digit tax number. |
| Address | `suppliers.address_line_1` | Needed for BIR dat files. |
| Document No. | `supplier_invoice_no` | Reference number from the vendor. |
| Gross Amount | Auto-computed | The total invoice amount. |
| Exempt/Zero | Transaction Lines | Amount tagged with Exempt or Zero-Rated tax codes. |
| Net Taxable | Transaction Lines | The base amount subject to 12% VAT. |
| Input VAT | Transaction Lines | The actual 12% tax amount claimed. |

## The Supabase Database Architecture (For Lovable)
We will create a comprehensive **SQL View** that unions line-item data from both `vendor_bill_lines` and `cash_purchase_lines` where the associated tax code represents a valid Input VAT category.

### View: `vw_input_vat_review`
This view aggregates tax data at the document level for easy review, grouped by the source transaction.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `transaction_id`| UUID | Required | Links to `vendor_bills.id` or `cash_purchases.id`. |
| `source_module` | Text | Required | `vendor_bill` or `cash_purchase`. |
| `invoice_date` | Date | Required | The date of the transaction. |
| `supplier_tin` | Text | Required | Joined from `suppliers.tin`. |
| `supplier_name` | Text | Required | Joined from `suppliers.registered_name`. |
| `supplier_address`| Text | Required | Joined from `suppliers.address_line_1` & city. |
| `invoice_no` | Text | Required | The supplier's OR/SI number. |
| `gross_purchases`| Numeric | Computed | Sum of all line items. |
| `exempt_purchases`| Numeric | Computed | Sum of lines where VAT Code is Exempt. |
| `zero_rated` | Numeric | Computed | Sum of lines where VAT Code is Zero-Rated. |
| `taxable_base` | Numeric | Computed | Sum of lines subject to 12% VAT. |
| `input_vat` | Numeric | Computed | The 12% tax amount. |

*Note to Developer:* Ensure the view correctly handles supplier returns (Vendor Credits), deducting them from the total Input VAT for the respective period.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
