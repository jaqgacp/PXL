# PXL ERP Blueprint: Customers

## Module Overview
The Customers module acts as the master record for all client entities purchasing goods or services from the business. In the Philippine context, precise customer data is crucial for BIR compliance, particularly for the accurate generation of Sales Invoices (SI), Official Receipts (OR), and the Summary List of Sales (SLS). Correctly tagging a customer's TIN, Tax Type (VAT, Zero-Rated, Exempt), and registered address is strictly required to ensure proper output VAT calculations and to allow the business and its customers to properly claim input tax and withholding taxes.

## What You Will See on the Screen (The Dashboard)
When you open the "Customers" page, you will see an organized directory of all client records.

### The Action Bar (Top of the page):
* **Search Bar:** Quickly locate customers by Name, Trade Name, or TIN.
* **Filter:** Dropdown to filter by "Customer Group", "Tax Type", or "Active/Inactive" status.
* **Customize View:** Allows showing/hiding of columns based on user preference.
* **Export Button:** Downloads the current customer list to Excel or CSV.
* **Print Button:** Generates a printable report of the customer directory.
* **Create New Customer Button:** A primary button to register a new customer.
* **Import Button:** Allows bulk uploading of customer records via CSV/Excel, crucial during initial data migration.

### The Data List (The main table):
Default columns displayed are:
* Customer Code
* Registered Name
* Trade Name
* TIN
* Tax Type
* Contact Person
* Status (Active/Inactive)

Row actions include "Edit" and "View" details.

## What Information We Need to Capture (The Data Fields)
When clicking "Create New Customer" or "Edit", the form is divided into the following sections.

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Customer Code | Text Input. Unique alphanumeric identifier (e.g., CUST-001). Auto-generated or manual. | Yes |
| Customer Group | Dropdown to classify customers (e.g., Retail, Wholesale, Corporate). | Optional |
| Registered Name | Text Input. Exact legal name matching their BIR Form 2303. | Yes |
| Trade Name | Text Input. "Doing Business As" name. | Optional |
| Business Style | Text Input. The business style as indicated in their BIR registration. | Optional |

### Section 2: Tax & Compliance Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| TIN | Text Input. Taxpayer Identification Number (000-000-000-00000). | Yes |
| Default Tax Type | Dropdown linked to Tax Types master. Dictates default tax treatment on sales. | Yes |
| Default EWT Code | Dropdown linked to EWT Codes master. | Optional |

### Section 3: Contact & Address Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Registered Address | Text Area. Full legal address for BIR billing purposes. | Yes |
| Delivery Address | Text Area. Address for shipping/delivery. Auto-filled from Registered Address if ticked. | Yes |
| Contact Person | Text Input. Primary person to contact for sales/collections. | Optional |
| Email | Email Input. Email address for sending electronic invoices/statements. | Optional |
| Phone Number | Text Input. Landline or mobile number. | Optional |

### Section 4: Commercial Terms
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Default Payment Terms | Dropdown linked to Payment Terms master. | Yes |
| Default Currency | Dropdown linked to Currencies master. | Yes |
| Default GL Account | Dropdown linked to Chart of Accounts (AR Account). | Yes |
| Credit Limit | Number Input. Maximum allowable outstanding balance before blocking sales. | Optional |

## The Supabase Database Architecture (For Lovable)

We will create the core `customers` table.


**Critical Database Rule for Lovable:** A composite unique constraint must be enforced on (company_id, `customer_code`) to ensure multi-tenant data integrity.

### Table: `customers`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the customer. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_code` | Text | Required, Unique | The unique customer identifier. |
| `customer_group_id` | UUID | Nullable, Foreign Key | Links to a future `customer_groups` table. |
| `registered_name` | Text | Required | Legal name. |
| `trade_name` | Text | Nullable | Trade name. |
| `business_style` | Text | Nullable | Business style. |
| `tin` | Text | Required | 13-15 digit string. |
| `default_tax_type` | UUID | Required, Foreign Key | Links to `tax_types` table. |
| `default_ewt_code_id` | UUID | Nullable, Foreign Key | Links to `ewt_codes` table. |
| `registered_address` | Text | Required | Full legal address. |
| `delivery_address` | Text | Required | Shipping address. |
| `contact_person` | Text | Nullable | Name of primary contact. |
| `email` | Text | Nullable | Contact email. |
| `phone_number` | Text | Nullable | Contact number. |
| `default_terms_id` | UUID | Nullable, Foreign Key | Links to `payment_terms` table. |
| `default_currency_id` | UUID | Required, Foreign Key | Links to `currencies` table. |
| `default_gl_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts` table. |
| `credit_limit` | Numeric | Default: 0 | Maximum credit allowed. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
