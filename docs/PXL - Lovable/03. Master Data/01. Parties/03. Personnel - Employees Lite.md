# PXL ERP Blueprint: Personnel (Employees Lite)

## Module Overview
The Personnel (Employees Lite) module maintains basic profiles of individuals working within or for the company. While not a full-fledged HRIS, this module captures essential employee details required for workflow approvals, sales commissions (Salesmen), operational tagging (Technicians/Drivers), and expense liquidation tracking. In the Philippine context, identifying internal personnel correctly connects operational actions (like cash advances or inventory receiving) to specific responsible individuals.

## What You Will See on the Screen (The Dashboard)
When you open the "Personnel" page, you will see a list of registered employees and contractors.

### The Action Bar (Top of the page):
* **Search Bar:** Search for personnel by Name or Employee ID.
* **Filter:** Dropdown options for "Department", "Role/Type" (e.g., Sales, Admin, Operations), and "Status" (Active/Inactive).
* **Customize View:** Allows showing/hiding of table columns.
* **Export Button:** Exports the list to CSV/Excel format.
* **Print Button:** Generates a printable personnel list.
* **Create New Personnel Button:** To register a new employee.
* **Import Button:** To upload multiple employee records via CSV/Excel.

### The Data List (The main table):
Default columns displayed are:
* Employee ID
* Full Name
* Department
* Job Title
* Email
* Status (Active/Inactive)

At the end of every row, there will be buttons to "Edit" or "View" details.

## What Information We Need to Capture (The Data Fields)

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Employee ID | Text Input. Unique identifier (e.g., EMP-2023-001). | Yes |
| First Name | Text Input. First name of the employee. | Yes |
| Last Name | Text Input. Last name of the employee. | Yes |
| Middle Name | Text Input. Middle name. | Optional |
| Date of Birth | Date Picker. Used for age verification or record purposes. | Optional |

### Section 2: Employment Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Job Title | Text Input. Official position. | Yes |
| Department | Dropdown of company departments. | Optional |
| Employment Type | Text Input. Regular, Probationary, Contractual, Project-based. | Yes |
| Date Hired | Date Picker. Official start date. | Optional |

| Field Name | Description | Required? |
| :--- | :--- | :--- |

### Section 4: Contact Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Email Address | Email Input. Work or personal email. | Optional |

## The Supabase Database Architecture (For Lovable)


**Critical Database Rule for Lovable:** A composite unique constraint must be enforced on (company_id, `employee_id`) to ensure multi-tenant data integrity.

### Table: `personnel`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the personnel. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `employee_id` | Text | Required, Unique | Unique employee code. |
| `first_name` | Text | Required | First name. |
| `last_name` | Text | Required | Last name. |
| `middle_name` | Text | Nullable | Middle name. |
| `date_of_birth` | Date | Nullable | Date of birth. |
| `job_title` | Text | Required | Job title or designation. |
| `department_id` | UUID | Nullable, Foreign Key | Links to a `departments` table. |
| `employment_type` | Text | Required | E.g., `regular`, `contractual`. |
| `date_hired` | Date | Nullable | Hire date. |
| `user_id` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `is_salesperson` | Boolean | Default: `false` | Tag for sales transactions. |
| `is_approver` | Boolean | Default: `false` | Tag for workflow routing. |
| `email` | Text | Nullable | Email address. |
| `mobile_number` | Text | Nullable | Mobile number. |
| `address` | Text | Nullable | Residential address. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
