# PXL ERP Blueprint: Units of Measure

## Module Overview
The Units of Measure (UOM) module standardizes how quantities are tracked across purchasing, inventory, and sales. It allows the definition of base units (e.g., Piece) and conversion factors (e.g., Box of 10 Pieces). In a Philippine business setting, managing accurate UOMs is essential for ensuring that BIR VAT calculations on invoices align correctly with physical warehouse stock quantities, preventing unit price discrepancies.

## What You Will See on the Screen (The Dashboard)
When you open the "Units of Measure" page, a list of all recognized metrics will appear.

### The Action Bar (Top of the page):
* **Search Bar:** Find UOM by abbreviation or name.
* **Filter:** Filter by "Base Unit" or "Conversion Unit", and status.
* **Export Button:** Download UOM lists.
* **Create New UOM Button:** Add a new unit definition.
* **Import Button:** Bulk load UOMs.

### The Data List (The main table):
Default columns displayed are:
* UOM Code
* Description
* Base Unit (Checkbox/Icon)
* Active Status

## What Information We Need to Capture (The Data Fields)

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| UOM Code | Text Input. Abbreviation (e.g., PCS, BOX, KG, LTR). | Yes |
| UOM Description | Text Area. Full name (e.g., Pieces, Box, Kilograms, Liters). | Yes |
| Is Base Unit? | Checkbox. If true, this is the smallest indivisible unit. | Yes |

### Section 2: Conversions (For Line Items in a Sub-table)
If it's NOT a base unit, the user must define the conversion.
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Base UOM | Dropdown selecting an existing Base UOM. | Conditional |
| Conversion Factor | Number Input. Number of Base UOMs inside this UOM (e.g., 1 Box = 12 PCS). | Conditional |

## The Supabase Database Architecture (For Lovable)


**Critical Database Rule for Lovable:** A composite unique constraint must be enforced on (company_id, `uom_code`) to ensure multi-tenant data integrity.

### Table: `units_of_measure`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the UOM. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `uom_code` | Text | Required, Unique | Abbreviation (PCS, BOX). |
| `description` | Text | Required | Full description. |
| `is_base_unit` | Boolean | Required, Default: `false` | Identifies if it's the lowest level unit. |
| `base_uom_id` | UUID | Nullable, Foreign Key | Links to `units_of_measure.id`. Required if `is_base_unit` is false. |
| `conversion_factor` | Numeric | Nullable | Quantity of base unit. Required if `is_base_unit` is false. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
