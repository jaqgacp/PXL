# PXL ERP Blueprint: Item Categories

## Module Overview
The Item Categories module defines the hierarchical grouping of inventory items and services. Structuring categories correctly is critical for meaningful financial and operational reporting. In the Philippines, categories often dictate the default accounts used for Sales, Cost of Goods Sold (COGS), and Inventory asset tracking, ensuring seamless integration between the warehouse operations and the Chart of Accounts for automated journal entries.

## What You Will See on the Screen (The Dashboard)
When you open the "Item Categories" page, you will see a tree-like or flat list of all defined categories.

### The Action Bar (Top of the page):
* **Search Bar:** Search category by Name or Code.
* **Filter:** Filter by parent category or active status.
* **Export Button:** Exports the category list.
* **Print Button:** Prints the category hierarchy.
* **Create New Category Button:** Add a new classification.
* **Import Button:** Bulk upload of category trees.

### The Data List (The main table):
Default columns displayed are:
* Category Code
* Category Name
* Parent Category
* Default Sales Account
* Default COGS Account
* Default Inventory Account

## What Information We Need to Capture (The Data Fields)

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Category Code | Text Input. Unique alphanumeric identifier (e.g., CAT-ELEC). | Yes |
| Category Name | Text Input. Descriptive name (e.g., Electronics, Raw Materials). | Yes |
| Parent Category | Dropdown linking to another category to establish a hierarchy. | Optional |
| Description | Text Area. Brief text explaining what belongs in this category. | Optional |

### Section 2: Default Accounting Links
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Sales Account | Dropdown linking to Chart of Accounts (Revenue). | Optional |
| COGS Account | Dropdown linking to Chart of Accounts (Cost of Sales). | Optional |
| Inventory Account | Dropdown linking to Chart of Accounts (Asset). | Optional |
| Adjustment Account | Dropdown for inventory discrepancies (Expense/Loss). | Optional |

## The Supabase Database Architecture (For Lovable)


**Critical Database Rule for Lovable:** A composite unique constraint must be enforced on (company_id, `category_code`) to ensure multi-tenant data integrity.

### Table: `item_categories`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the category. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `category_code` | Text | Required, Unique | Unique category code. |
| `category_name` | Text | Required | Name of the category. |
| `parent_category_id` | UUID | Nullable, Foreign Key | Self-referencing to `item_categories.id`. |
| `description` | Text | Nullable | Category description. |
| `sales_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. |
| `cogs_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. |
| `inventory_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. |
| `adj_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
