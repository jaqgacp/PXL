# PXL ERP Blueprint: Petty Cash Replenishment

## Module Overview
The Petty Cash Replenishment module aggregates disbursed and approved Petty Cash Vouchers (PCVs) to reimburse the fund custodian and restore the Petty Cash Fund to its imprest balance. The replenishment itself is a Cash Disbursement — the company issues a check or bank transfer to restore the fund. This generates a Cash Disbursement Journal Entry (Dr. Petty Cash Fund GL / Cr. Cash/Bank Account). Note: expenses were already recognized when PCVs were approved. The replenishment only restores the fund's cash balance — it does NOT re-expense the PCVs.

**Architecture Correction:** This module previously referenced `ap_vouchers.id` — a table that does not exist in this blueprint system. The replenishment is a **Cash Disbursement** transaction linked to the company's bank accounts, not an AP payable. The custodian is an employee, not a third-party vendor.

## Dashboard UI

### The Action Bar
* **Search Bar:** Find a replenishment by Number or PCF Name.
* **Filter:** Filter by Status (Draft, Approved, Posted) or Date Range.
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the current list to an Excel or CSV file.
* **Print Button:** Generates a print-ready summary report.
* **Create New Replenishment Button:** Opens the form to select PCVs and request reimbursement.

### The Data List
Default columns:
* Replenishment Number
* Date
* Petty Cash Fund
* Total Amount
* Disbursing Bank Account
* Status

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 50 records.

## GL Posting Logic (System-Generated on Post)

The GL posting fires on the Post action. This is the cash disbursement that restores the fund.

| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Petty Cash Fund GL Account | Total Replenishment Amount | | `petty_cash_funds.gl_account_id` |
| 2 | Bank / Cash Account | | Total Replenishment Amount | `pcf_replenishments.disbursing_bank_id` → `bank_accounts.gl_account_id` |

*Note: Expense accounts are NOT debited here — they were already debited when each PCV was approved. This entry only restores the cash balance.*

*On Post, all included PCVs are updated to `status = 'replenished'`.*

## Supabase Database Architecture

### Table 1: `pcf_replenishments` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, replenishment_number)` — prevents duplicate document numbers.
2. `journal_entry_id` is the mandatory GL traceability link. Populated on Post (Principle 7).
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the replenishment. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `pcf_id` | UUID | Required, Foreign Key | Links to `petty_cash_funds.id`. |
| `replenishment_number` | Text | Required | `UNIQUE(company_id, replenishment_number)`. |
| `replenishment_date` | Date | Required | Validated against open `accounting_periods`. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `disbursing_bank_id` | UUID | Required, Foreign Key | Links to `bank_accounts.id`. The account from which cash is drawn. |
| `particulars` | Text | Required | Summary description. |
| `total_amount` | Numeric(15,2) | Required | Sum of all selected PCV gross amounts. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. GL traceability link. |
| `posted_at` | Timestamp | Nullable | System-set on Post. Never editable. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `approved`, `posted`, `voided`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, replenishment_number)`

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on posted or voided replenishments
CREATE POLICY "block_update_on_posted_replenishment"
ON pcf_replenishments
FOR UPDATE
USING (status IN ('draft', 'approved'));

-- Block DELETE on all replenishments
CREATE POLICY "block_delete_all_replenishments"
ON pcf_replenishments
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_replenishments"
ON pcf_replenishments
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: tenant isolation
CREATE POLICY "tenant_isolation_replenishment_lines"
ON pcf_replenishment_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
