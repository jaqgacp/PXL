# Credit Memos

## Module Overview
The Credit Memo (CM) module is used to reduce the amount owed by a customer. This is typical in Philippine business for scenarios such as sales allowances, overbillings, or retroactive discounts. A Credit Memo directly reduces Accounts Receivable and reverses the corresponding Sales Revenue and Output VAT. It acts as a negative invoice.

### Document Flow & Data Inheritance
* **Source:** Usually tied directly to a specific **Sales Invoice**.
* **Inheritance:** When linked to an SI, the user selects the specific invoice lines to credit. The `item_id`, `unit_price`, and tax codes are inherited to ensure the reversal matches the original billing exactly.

## Dashboard UI
A centralized view for tracking reductions in customer receivables.

### The Action Bar
* **Search Bar:** Search by CM Number, Customer, or Original Invoice Number.
* **Filter:** Dropdown for Status (Draft, Approved, Applied).
* **Customize View:** Toggle visible columns.
* **Export Button:** Export list to Excel/CSV.
* **Print Button:** Print the CM for the customer.
* **Create Credit Memo Button:** Initiate a new memo.
* **Import Button:** Bulk upload CMs.

### The Data List
Default columns displayed:
* CM Date
* CM Number
* Customer Name
* Related Invoice
* Reason
* Total Amount
* Status

## Supabase Database Architecture


**Critical Database Rule for Lovable:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `credit_memos` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the CM. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `invoice_id` | UUID | Nullable, Foreign Key | Links to `sales_invoices.id`. |
| `cm_number` | Text | Required, Unique | Formatted document number (e.g., CM-2023-0001). |
| `cm_date` | Date | Required | Date of the memo. |
| `reason_code_id` | UUID | Required, Foreign Key | Links to `ref_reason_codes.id`. |
| `remarks` | Text | Nullable | Internal notes. |
| `total_amount` | Numeric | Required | Total amount credited. |
| `vat_amount` | Numeric | Required | Reversal of output VAT. |
| `status` | Text | Required | `draft`, `approved`, `applied`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
