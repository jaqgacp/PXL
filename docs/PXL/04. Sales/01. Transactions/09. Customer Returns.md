# Customer Returns

## Module Overview
The Customer Returns module handles the physical return of goods from a customer. Unlike a pure Credit Memo (which just adjusts financials), this document affects both financials and inventory. It increases warehouse stock while simultaneously creating a Credit Memo to reduce Accounts Receivable. Proper tracking is essential for inventory valuation and recognizing damaged vs. sellable returned goods.

### Document Flow & Data Inheritance
* **Source:** Derived from a **Delivery Receipt** or a **Sales Invoice**.
* **Inheritance:** Pulls the exact item, original unit price, and delivered quantity from the source document to ensure inventory is valued correctly upon return, and the financial credit matches the original billing.

## Dashboard UI
A tracking list for the warehouse and accounting teams to monitor returned items.

### The Action Bar
* **Search Bar:** Search by Return Number, Customer, or Source DR/SI.
* **Filter:** Dropdown for Return Status (Pending Inspection, Accepted, Rejected).
* **Customize View:** Adjust columns.
* **Export Button:** Export to Excel/CSV.
* **Print Button:** Print return acknowledgment slip.
* **Create Return Button:** Initiate a new return.

### The Data List
Default columns displayed:
* Return Date
* Return Number
* Customer Name
* Source Document
* Items Returned
* Return Condition
* Status

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `customer_returns` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the return. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `source_doc_type`| Text | Required | `delivery_receipt` or `sales_invoice`. |
| `source_doc_id` | UUID | Required | Polymorphic ID. |
| `return_number` | Text | Required, Unique | Formatted document number. |
| `return_date` | Date | Required | Date items were received back. |
| `reason_code_id` | UUID | Required, Foreign Key | Links to `ref_reason_codes.id`. |
| `action_taken` | Text | Required | `credit_memo`, `replacement`. |
| `status` | Text | Required | `pending`, `accepted`, `rejected`. |
| `linked_cm_id` | UUID | Nullable, Foreign Key | Links to generated `credit_memos.id`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
