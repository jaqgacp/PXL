# Percentage Tax Review

## Module Overview
The Percentage Tax Review module is designed specifically for Non-VAT registered businesses in the Philippines (typically those whose gross annual sales do not exceed the VAT threshold, subject to 3% tax under Section 116). This module aggregates Gross Sales/Receipts to facilitate the preparation of BIR Form 2551Q (Quarterly Percentage Tax Return). 

*Note: Under Philippine tax rules, percentage tax is generally based on gross sales/receipts. This is a reporting module that aggregates posted Sales Invoices and Cash Sales.*

## Dashboard UI
A straightforward financial summary dashboard for quarterly tax filing preparation.

### The Action Bar
* **Year Filter:** Dropdown to select the taxable year.
* **Quarter Filter:** Dropdown to select Q1, Q2, Q3, or Q4.
* **Export Summary:** Download the summarized data to Excel.
* **Print Summary:** Print a hardcopy for the accountant.
* **Mark as Filed:** Locks the quarter to prevent backdated entries that alter gross receipts.

### The Data List (Summary Metrics)
Top cards display:
* Total Gross Sales (Month 1)
* Total Gross Sales (Month 2)
* Total Gross Sales (Month 3)
* Total Quarterly Gross Sales
* Tax Due (e.g., 3% of Total)

Below the summary is a detailed list of transactions for the quarter.
Columns: Date, Doc Type, Doc No., Customer Name, Gross Amount.

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

This relies on a simple aggregation of gross amounts from sales documents for non-VAT entities.

### View Definition: `vw_percentage_tax_summary`
This view pulls from `sales_invoices`, `cash_sales`, and deducts `credit_memos`.

| Column Name | Data Type | Source | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `transaction_date`| Date | Source `invoice_date`, `cs_date`, etc. | For period filtering. |
| `period_month` | Integer | Extracted from Date | 1 to 12 (for monthly breakdown). |
| `period_quarter` | Integer | Extracted from Date | 1 to 4. |
| `doc_type` | Text | 'SI', 'CS', 'CM' | Document source. |
| `doc_number` | Text | Source Document Number | Reference. |
| `customer_name` | Text | `customers.registered_name` | Buyer name. |
| `gross_amount` | Numeric | Net of SI/CS minus CM | Base for percentage tax. |
| `tax_rate` | Numeric | System Config (Default 0.03) | Current percentage tax rate. |
| `computed_tax` | Numeric | `gross_amount * tax_rate` | Tax due per transaction. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

*Note: The frontend will use this view to sum the `gross_amount` per quarter to match the requirements of Form 2551Q.*
