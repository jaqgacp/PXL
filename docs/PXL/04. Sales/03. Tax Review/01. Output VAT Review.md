# Output VAT Review

## Module Overview
The Output VAT Review module is a critical compliance tool for VAT-registered businesses in the Philippines. It aggregates all Output VAT generated from Sales Invoices, Cash Sales, and Credit/Debit Memos for a specific month or quarter. This module serves as the preliminary review screen before generating BIR Form 2550M/2550Q (VAT Returns) and the Summary List of Sales (SLS) DAT file.

*Note: This is a reporting and review module. It reads posted transaction data and provides a consolidated view.*

## Dashboard UI
A financial summary dashboard focused on tax compliance.

### The Action Bar
* **Period Filter:** Dropdown to select Month and Year (e.g., "October 2023").
* **Quarter Toggle:** Switch between Monthly or Quarterly views.
* **Export Summary:** Download the summarized data to Excel.
* **Generate SLS Button:** Exports the data into the BIR Relief formatted file (DAT/CSV).
* **Mark as Filed:** Locks the period from further backdated invoice entries.

### The Data List (Summary Metrics)
Top cards display:
* Total Vatable Sales
* Total Zero-Rated Sales
* Total Exempt Sales
* Total Output VAT

Below the cards is a list of all transactions contributing to these totals for the selected period.
Columns: Date, Doc No., Customer, TIN, Gross Sales, Vatable, Zero-Rated, Exempt, Output VAT.

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

This relies on aggregating the line items from various sales documents, grouped by their assigned tax codes.

### View Definition: `vw_output_vat_summary`
This view aggregates data from `sales_invoices`, `cash_sales`, `credit_memos` (as negatives), and `debit_memos`.

| Column Name | Data Type | Source | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `transaction_date`| Date | Source `invoice_date`, `cs_date`, etc. | For period filtering. |
| `doc_number` | Text | Source Document Number | Reference. |
| `customer_name` | Text | `customers.registered_name` | Buyer name. |
| `customer_tin` | Text | `customers.tin` | Buyer TIN (crucial for SLS). |
| `vatable_sales` | Numeric | Sum of net amount where tax is 12% | Vatable base. |
| `zero_rated_sales`| Numeric | Sum of net amount where tax is 0% | Export/PEZA sales. |
| `exempt_sales` | Numeric | Sum of net amount where tax is Exempt | Non-taxable sales. |
| `output_vat` | Numeric | Computed 12% on vatable sales | Tax due. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

*To ensure strict compliance, the system will use this view to generate the exact column sequence required by the BIR Relief system for the SLS attachment.*
