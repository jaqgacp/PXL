# PXL ERP Blueprint: Sales Invoice

## Module Overview
The Sales Invoice is the primary BIR-registered document evidencing a sale of goods or services. Under Philippine tax law (NIRC Section 237), all VAT-registered and non-VAT sellers are required to issue invoices. A Sales Invoice generates an AR sub-ledger entry (debit AR, credit Revenue, credit Output VAT/Percentage Tax), posts to the General Ledger, and populates the Output VAT Review, the Sales Invoice Register, and the BIR SLS (Summary List of Sales). Every field on this document has a downstream effect on tax compliance.

## Document Flow & Data Inheritance

When a user creates a new Sales Invoice, the system must auto-fill the following fields based on the selected master data records. The user must NOT be required to manually enter any data that the system already knows.

### Auto-Fill Chain on Customer Selection
When a **Customer** is selected, the following fields auto-populate immediately:
| Field Auto-Filled | Source |
| :--- | :--- |
| Customer TIN | Customer TIN snapshot in `XXX-XXX-XXX-XXXXX` format |
| Customer Address | `customers.registered_address` |
| VAT Status of Customer | `customers.vat_status` (determines if VAT is applicable) |
| Payment Terms | `customers.payment_terms_id` |
| Currency | `customers.currency_code` |
| Default Sales Tax Code (header) | `customers.default_sales_tax_id` |
| AR Control Account | `customers.ar_account_id` |
| Subject to CWT | `customers.is_subject_to_cwt` and `customers.default_cwt_atc_code_id` |

### Auto-Fill Chain on Item Selection (per line)
When an **Item** is selected on a line, the following fields auto-populate:
| Field Auto-Filled | Source |
| :--- | :--- |
| Item Description | `items.description` |
| Unit of Measure | `items.uom_id` |
| Unit Price | `items.standard_selling_price` |
| VAT Code (line) | `items.default_sales_vat_id` → falls back to `item_categories.sales_account_id` mapping |
| Revenue GL Account (hidden, for posting) | `items.sales_account_id` → `item_categories.sales_account_id` → `gl_posting_configurations` |
| COGS GL Account (hidden, for posting) | `items.cogs_account_id` → `item_categories.cogs_account_id` → `gl_posting_configurations` |

### Section 3: Summary Totals (Read-only, computed from lines)
| Field Name | Computed From |
| :--- | :--- |
| Total Taxable Sales (VAT Base) | Sum of Net Amount on lines where `vat_classification = 'regular'`. |
| Total Zero-Rated Sales | Sum of Net Amount on lines where `vat_classification = 'zero_rated'`. |
| Total Exempt Sales | Sum of Net Amount on lines where `vat_classification = 'exempt'`. |
| Total Output VAT | 12% × Total Taxable Sales. |
| Less: CWT (if customer is withholding agent) | Displayed as informational. Not deducted from the Invoice total — collected separately via Official Receipt. |
| Grand Total | Sum of all line Total Amounts (all classifications). |

## GL Posting Logic (System-Generated on Post)

Upon posting, the system generates the following GL entries automatically. No manual Journal Entry is needed.

| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Accounts Receivable (AR Control) | Grand Total | | `customers.ar_account_id` |
| 2 | Sales Revenue | | Net Taxable Amount | `items.sales_account_id` → category → GL config |
| 3 | Zero-Rated Sales | | Net Zero-Rated Amount | `items.sales_account_id` (zero-rated item) |
| 4 | Exempt Sales | | Net Exempt Amount | `items.sales_account_id` (exempt item) |
| 5 | Output VAT Payable | | Total Output VAT | `tax_codes.gl_account_id` for the VAT code |

## Supabase Database Architecture

### Table 1: `sales_invoices` (Header)

**Critical Database Rules:**
1. Composite unique constraint on `(company_id, si_number)`.
2. `status` is an ENUM: `draft`, `approved`, `posted`, `cancelled`. Posted and Cancelled invoices cannot be edited (RLS policy mirrors Journal Entries pattern).
3. `posted_at` and `posted_by` are system-set on the Post action and are never editable.
4. A cancelled (voided) SI must retain its record. The number is never reused. A `void_reason_id` must be supplied.

| Column Name | Data Type | Rules | Description |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `si_number` | Text | Required | BIR-registered sequential SI number from Number Series. |
| `date` | Date | Required | Invoice date. Validated against `accounting_periods`. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `customer_name_snapshot` | Text | Required | Customer name at time of posting (audit trail). |
| `customer_tin_snapshot` | Text | Required | Customer TIN at time of posting (BIR audit trail). |
| `customer_address_snapshot` | Text | Required | Customer address at time of posting. |
| `payment_terms_id` | UUID | Required, Foreign Key | Links to `payment_terms.id`. |
| `due_date` | Date | Required | Computed due date. |
| `currency_code` | Text | Required, Default `PHP` | Transaction currency. |
| `reference` | Text | Nullable | Sales Order or DR reference. |
| `memo` | Text | Nullable | Prints on invoice footer. |
| `total_taxable_amount` | Numeric(15,2) | Required, Default 0 | Sum of net amounts on regular VAT lines. |
| `total_zero_rated_amount` | Numeric(15,2) | Required, Default 0 | Sum of net amounts on zero-rated lines. |
| `total_exempt_amount` | Numeric(15,2) | Required, Default 0 | Sum of net amounts on exempt lines. |
| `total_vat_amount` | Numeric(15,2) | Required, Default 0 | Total Output VAT (12% of taxable amount). |
| `total_amount` | Numeric(15,2) | Required, Default 0 | Grand total (all lines, inclusive of VAT). |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `approved`, `posted`, `cancelled`. |
| `void_reason_id` | UUID | Nullable, Foreign Key | Links to `void_reason_codes.id`. Required if `status = 'cancelled'`. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. |
| `posted_at` | Timestamp | Nullable | System-set on Post action. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, si_number)`

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on posted or cancelled Sales Invoices
CREATE POLICY "block_update_on_posted_si"
ON sales_invoices
FOR UPDATE
USING (status = 'draft' OR status = 'approved');

-- Block DELETE on all Sales Invoices (voiding sets status = 'cancelled', never deletes)
CREATE POLICY "block_delete_all_si"
ON sales_invoices
FOR DELETE
USING (false);
```
