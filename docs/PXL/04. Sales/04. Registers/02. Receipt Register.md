# Receipt Register

## Module Overview
The Receipt Register is a chronological log of all Official Receipts (OR) or Collection Receipts (CR) issued by the company. For Philippine accounting compliance, this serves as the foundation for the **Cash Receipts Journal**. It documents all cash inflows from customers, detailing the breakdown of bank deposits, accounts receivable reductions, and withheld taxes (2307).

*Note: This is an aggregated reporting view. It fetches posted data from the `receipts` module.*

## Dashboard UI
A read-only tabular view for treasury and accounting teams to reconcile bank deposits.

### The Action Bar
* **Date Range Filter:** "From" and "To" dates.
* **Bank Account Filter:** Dropdown to filter collections deposited to a specific bank.
* **Payment Mode Filter:** Isolate Cash, Check, or Bank Transfers.
* **Export Button:** Export to Excel/CSV for bank reconciliation.
* **Print Button:** Generate physical reports for the Cash Receipts Book.

### The Data List
Default columns displayed chronologically:
* Date
* OR/CR Number
* Customer Name
* Payment Mode
* Reference No. (Check/Trace No.)
* Deposited To (Bank Account)
* Cash Received (Debit Cash)
* CWT Withheld (Debit Creditable Tax)
* AR Applied (Credit Accounts Receivable)
* Status

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

This report maps to the `receipts` table, incorporating the totals from `receipt_lines`.

### View Definition: `vw_receipt_register`
| Column Name | Data Type | Source | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `receipt_date` | Date | `receipts.receipt_date` | Chronological sorting. |
| `receipt_number` | Text | `receipts.receipt_number` | Document sequence. |
| `customer_name` | Text | `customers.registered_name` | Payer name. |
| `payment_mode` | Text | `ref_payment_modes.name` | Cash/Check/Online. |
| `reference_number` | Text | `receipts.reference_number` | Check or trace number. |
| `bank_account_name`| Text | `chart_of_accounts.name` | Where cash was deposited. |
| `total_cash` | Numeric | `receipts.total_amount` | Debit to Cash in Bank. |
| `total_cwt` | Numeric | `receipts.total_cwt` | Debit to Prepaid Tax (2307). |
| `total_ar_applied` | Numeric | `total_cash + total_cwt` | Credit to Accounts Receivable. |
| `status` | Text | `receipts.status` | Important for tracking bounced checks or voided ORs. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
