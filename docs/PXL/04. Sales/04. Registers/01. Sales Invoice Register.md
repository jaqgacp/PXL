# Sales Invoice Register

## Module Overview
The Sales Invoice Register is a chronological listing of all posted Sales Invoices within a specific period. In Philippine accounting and BIR compliance, this report serves as the digital equivalent of the **Sales Journal** (one of the mandatory Books of Accounts). It provides a clear audit trail of all recognized revenue, output VAT, and accounts receivable generated through billing.

*Note: This is a reporting module. No data entry occurs here. It reads directly from the `sales_invoices` table.*

## Dashboard UI
A read-only, highly structured table designed to mimic traditional accounting ledgers.

### The Action Bar
* **Date Range Filter:** "From" and "To" dates (e.g., a specific month).
* **Customer Filter:** Optional, to isolate a specific buyer.
* **Export Button:** Export the register to Excel/CSV for month-end reconciliation.
* **Print Button:** Print in a format suitable for hardbound book compliance (if using loose-leaf).

### The Data List
Default columns displayed chronologically:
* Date
* SI Number
* Customer Name
* TIN
* Gross Amount
* Discount
* Net Sales (Base)
* Output VAT
* Total Invoice Amount
* Status (Must highlight "Voided" invoices for continuous sequence auditing)

## Data Fields (Query Parameters)
| Field Name | Component Type | Required? | Behavior |
| :--- | :--- | :--- | :--- |
| Start Date | Date Picker | Yes | Beginning of the reporting period. |
| End Date | Date Picker | Yes | End of the reporting period. |
| Include Voided | Checkbox | No | BIR requires sequence checking, so voids are often shown with zero amounts. |

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

This report maps directly to the `sales_invoices` table but flattens the totals for easy reading.

### View Definition: `vw_sales_invoice_register`
| Column Name | Data Type | Source | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `invoice_date` | Date | `sales_invoices.invoice_date` | Chronological sorting. |
| `si_number` | Text | `sales_invoices.si_number` | Document sequence. |
| `customer_name` | Text | `customers.registered_name` | Buyer name. |
| `customer_tin` | Text | `customers.tin` | Buyer TIN. |
| `gross_amount` | Numeric | `sales_invoices.gross_amount` | Total before deductions. |
| `total_discount` | Numeric | `sales_invoices.total_discount` | Trade discounts. |
| `vatable_sales` | Numeric | Aggregated from `sales_invoice_lines`| Base for 12% tax. |
| `exempt_sales` | Numeric | Aggregated from `sales_invoice_lines`| Non-taxable base. |
| `zero_rated_sales`| Numeric | Aggregated from `sales_invoice_lines`| 0% base. |
| `output_vat` | Numeric | `sales_invoices.vat_amount` | VAT recognized. |
| `net_amount` | Numeric | `sales_invoices.net_amount` | Total AR recognized. |
| `status` | Text | `sales_invoices.posting_status`| Validates document status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

*Sorting strictly by `si_number` and `invoice_date` is critical for BIR audit trails.*
