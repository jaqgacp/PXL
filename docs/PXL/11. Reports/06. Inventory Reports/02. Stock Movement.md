# Stock Movement

## 1. Module Overview
**Relevance to PH Accounting/Business Practices:**
The Stock Movement is essential for Philippine financial reporting, statutory compliance (SEC, BIR), and management decision-making. It ensures accurate representation of financial status and operational performance in accordance with Philippine Financial Reporting Standards (PFRS) and Bureau of Internal Revenue (BIR) requirements.

**Data Aggregation from GL, Subsidiary Ledgers, and Master Data:**
- **General Ledger (GL):** Extracts validated, posted journal entries to compute aggregate balances per account.
- **Subsidiary Ledgers (SL):** Pulls detailed transaction records (e.g., Accounts Receivable, Accounts Payable, Inventory, Fixed Assets) to support and reconcile with GL control accounts.
- **Master Data:** Utilizes Chart of Accounts (COA), Customer/Vendor Profiles, Cost Centers, Bank Accounts, and Branch codes to appropriately map and categorize financial data for this specific report.

## 2. Dashboard UI

- **Data Grid:** Displays the report output with hierarchical grouping (e.g., Account Types, Categories, Aging Buckets).
- **Pagination/Infinite Scroll:** For handling large report datasets seamlessly.
- **Column Sorter & Resizer:** Interactive columns for customizable viewing.
- **Drill-Down Links:** Clickable amounts that navigate directly to the underlying Subsidiary Ledger, Transaction Document, or Journal Entry.

## 4. Supabase Database Architecture

To manage saved report configurations and historical snapshots, the architecture follows a strict Header-Line structural pattern.

### Header Table: 
rpt_stock_movement_templates
Stores the saved configuration and metadata for the report.

| Column Name | Data Type | Constraints / Relations |
|---|---|---|
| id | UUID | Primary Key, Default: uuid_generate_v4() |
| template_name | VARCHAR(255) | NOT NULL, UNIQUE per user |
| description | TEXT | NULLABLE |
| is_public | BOOLEAN | Default: FALSE |
| created_by | UUID | NOT NULL, Foreign Key to auth.users |
| updated_by | UUID | NOT NULL, Foreign Key to auth.users |
| created_at | TIMESTAMPTZ | NOT NULL, Default: now() |
| updated_at | TIMESTAMPTZ | NOT NULL, Default: now() |

### Line-Item Table: 
rpt_stock_movement_filters
Stores the specific parameters and field selections for the saved template.

| Column Name | Data Type | Constraints / Relations |
|---|---|---|
| id | UUID | Primary Key, Default: uuid_generate_v4() |
| template_id | UUID | NOT NULL, Foreign Key to rpt_stock_movement_templates(id) ON DELETE CASCADE |
| filter_key | VARCHAR(100) | NOT NULL (e.g., 'date_range', 'account_group') |
| filter_value | TEXT | NOT NULL |
| filter_operator | VARCHAR(50) | NOT NULL (e.g., 'equals', 'between') |
| created_by | UUID | NOT NULL, Foreign Key to auth.users |
| updated_by | UUID | NOT NULL, Foreign Key to auth.users |
| created_at | TIMESTAMPTZ | NOT NULL, Default: now() |
| updated_at | TIMESTAMPTZ | NOT NULL, Default: now() |

**Database Constraints & Triggers:**
- **Critical Database Rule:** UNIQUE Constraint: UNIQUE(template_id, filter_key) to prevent duplicate filter definitions for a single template.
- **Audit Triggers:** updated_at is automatically modified via Supabase triggers upon any row update.
