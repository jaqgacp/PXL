## Module Overview
The 2307 Issued Review module is a compliance tracking dashboard. Under Philippine tax regulations, whenever a company deducts Expanded Withholding Tax (EWT) from a supplier payment, it is legally obligated to provide the supplier with a BIR Form 2307 (Certificate of Creditable Tax Withheld at Source). This module tracks all EWT deductions, monitors whether the corresponding 2307 certificate has been generated, and records whether it has been sent to and acknowledged by the supplier.

## What You Will See on the Screen (The Dashboard)
When you open the "2307 Issued Review" page, you will see a workflow-oriented workspace.

### The Action Bar (Top of the page):
* **Period Filter:** Dropdowns for Year and Month/Quarter.
* **Supplier Search:** Quickly find a specific vendor.
* **Status Filter:** View by generation/delivery status (Pending Generation, Generated, Sent to Supplier, Acknowledged).
* **Batch Generate Button:** Select multiple rows to generate PDFs simultaneously.
* **Email to Supplier Button:** Automatically emails the generated certificates to the registered supplier email.
* **Export Button:** Downloads the tracking list to Excel.

### The Data List (The main table):
* Quarter / Period
* Supplier Name
* Supplier TIN
* Document Reference (Vendor Bill / Payment Voucher)
* Total Tax Base
* Total EWT Withheld
* Status (Pending, Generated, Sent, Acknowledged)
* Actions (View PDF, Mark as Sent, Mark as Acknowledged)

## What Information We Need to Capture (The Data Fields)
This module acts as a tracking layer over the `ewt_summary` data. It requires specific status flags to manage the distribution of the certificates.

### Tracking Columns
When users perform actions on a row, the following data points are updated:

| Field Name | Description | UI Component | Required? | Inheritance |
| :--- | :--- | :--- | :--- | :--- |
| Status | Current state of the certificate. | Auto-computed | Yes | None |
| Date Generated | Timestamp when the PDF was created. | Auto-computed | Auto | None |
| Date Sent | Timestamp when emailed or physically dispatched. | Date Picker | Optional | None |
| Date Acknowledged| Timestamp when the supplier confirmed receipt. | Date Picker | Optional | None |
| PDF File | Link to the generated document. | File Upload/Link | Optional | None |

## The Supabase Database Architecture
We will create a specific tracking table that links to the aggregated EWT data. Since 2307s are typically issued per quarter (or per month depending on agreement), this table stores the issuance status for a supplier for a specific period.

### Table 1: `form_2307_issuances`
This table records the lifecycle of the actual certificate document.

**Critical Database Rule:**
- Ensure a composite unique constraint on `supplier_id`, `tax_year`, and `tax_quarter` to prevent duplicate issuances for the same supplier in the same period.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the record. |
| `supplier_id` | UUID | Required, Foreign Key | Links to `suppliers.id`. |
| `tax_year` | Integer | Required | The applicable year. |
| `tax_quarter` | Integer | Required | The applicable quarter (1, 2, 3, 4). |
| `total_tax_base` | Numeric | Required | Aggregated net amount for the period. |
| `total_ewt` | Numeric | Required | Aggregated withheld amount. |
| `status` | Text | Required | `pending`, `generated`, `sent`, `acknowledged`. |
| `pdf_url` | Text | Nullable | Path to the generated PDF in Supabase Storage. |
| `date_generated` | Timestamp | Nullable | When the file was created. |
| `date_sent` | Timestamp | Nullable | When it was dispatched. |
| `date_acknowledged`| Timestamp | Nullable | When receipt was confirmed. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Record creation timestamp. |
| `updated_at` | Timestamp | Auto | Record update timestamp. |

### View: `vw_2307_monitoring`
A view combining `vw_ewt_summary` and `form_2307_issuances` to easily display pending vs. completed certificates on the dashboard.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
