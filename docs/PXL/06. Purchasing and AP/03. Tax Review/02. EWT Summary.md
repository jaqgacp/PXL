## Module Overview
The Expanded Withholding Tax (EWT) Summary module is a specialized tax dashboard for Philippine compliance. Whenever the company purchases services, rents property, or buys from specific top withholding taxpayers, it is mandated to withhold a percentage of the payment (EWT) and remit it to the BIR. This module aggregates all withheld amounts from Vendor Bills and Cash Purchases for a specific month. It provides the exact data needed to file BIR Form 1601-EQ and is the foundation for generating BIR Form 2307 (Certificate of Creditable Tax Withheld at Source) for the suppliers.

## What You Will See on the Screen (The Dashboard)
When you open the "EWT Summary" page, you will see a tax audit interface organized by Alphanumeric Tax Code (ATC).

### The Action Bar (Top of the page):
* **Period Filter:** Dropdowns for Year and Month/Quarter.
* **Supplier Search:** Isolate withholdings for a specific vendor.
* **ATC Filter:** View by specific tax code (e.g., WC158 for Professional Fees, WC160 for Rent).
* **Export Button:** Downloads the data to Excel, formatted for the BIR Alphalist Data Entry (QAP).
* **Generate 2307 Button:** Batch generates PDF certificates for selected suppliers.
* **Print Button:** Generates a summary report.

### The Data List (The main report table):
* Date (Invoice Date)
* Supplier Name
* Supplier TIN
* ATC (Tax Code)
* Nature of Income Payment
* Tax Base (Net Amount before VAT)
* Tax Rate (%)
* Tax Withheld

## What Information We Need to Capture (The Data Fields)
This is a read-only review and aggregation module. It extracts line-item data mapped to the strict requirements of the BIR Alphalist.

### Report Columns Map
| Column Name | Source | Description |
| :--- | :--- | :--- |
| Date | `bill_date` | The date on the physical receipt/invoice. |
| Supplier Name | `suppliers.registered_name` | Registered name of the vendor. |
| Supplier TIN | `suppliers.tin` | 9 to 13 digit tax number. |
| ATC | `tax_codes.code` | E.g., WI158, WC160. |
| Nature of Payment| `tax_codes.description` | Description of the ATC. |
| Tax Base | Transaction Lines | The net amount of the specific line item subject to EWT. |
| Rate | `tax_codes.rate` | The percentage withheld (e.g., 1%, 2%, 5%). |
| Tax Withheld | Auto-computed | Tax Base * Rate. |

## The Supabase Database Architecture
We will create a specific **SQL View** that unions line-item data from both `vendor_bill_lines` and `cash_purchase_lines` where an `ewt_code_id` is present.

### View: `vw_ewt_summary`
This view aggregates tax data at the line-item level, grouped by the source transaction and ATC, which is required for accurate QAP (Quarterly Alphalist of Payees) generation.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `transaction_id`| UUID | Required | Links to `vendor_bills.id` or `cash_purchases.id`. |
| `source_module` | Text | Required | `vendor_bill` or `cash_purchase`. |
| `invoice_date` | Date | Required | The date of the transaction. |
| `supplier_id` | UUID | Required | For grouping by vendor. |
| `supplier_tin` | Text | Required | Joined from `suppliers.tin`. |
| `supplier_name` | Text | Required | Joined from `suppliers.registered_name`. |
| `atc_code` | Text | Required | Joined from `tax_codes.code` (e.g., WC158). |
| `nature_of_payment`| Text | Required | Joined from `tax_codes.description`. |
| `tax_rate` | Numeric | Required | Joined from `tax_codes.rate`. |
| `tax_base` | Numeric | Computed | The net amount (excluding VAT) subject to this ATC. |
| `tax_withheld` | Numeric | Computed | The calculated EWT amount. |

*Note to Developer:* An invoice may have multiple line items with different ATC codes (e.g., mixed goods and services). This view must aggregate the tax base at the ATC level per document, not just summarize the entire document.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
