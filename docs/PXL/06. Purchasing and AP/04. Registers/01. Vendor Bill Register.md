## Module Overview
The Vendor Bill Register (often identical or closely related to the Purchases Journal) is a standard accounting report. It provides a chronological listing of all approved Vendor Bills entered into the system during a specific period. In the Philippines, this register is used by auditors and accountants to verify that all supplier invoices have been correctly recorded, classified to the right expense/asset accounts, and that the total Accounts Payable matches the General Ledger.

## What You Will See on the Screen (The Dashboard)
When you open the "Vendor Bill Register" page, you will see a detailed, read-only reporting table.

### The Action Bar (Top of the page):
* **Date Range Filter:** Select the period to view (e.g., This Month, Last Month, Custom Date Range).
* **Supplier Filter:** Optionally isolate the register for a specific supplier.
* **Status Filter:** View Draft, Open, Paid, or Cancelled bills.
* **Customize View:** Toggle detailed columns (e.g., show GL distribution).
* **Export Button:** Downloads the register to Excel or CSV.
* **Print Button:** Generates a formal, print-ready PDF of the register.

### The Data List (The main report table):
* Bill Date
* Bill Number
* Supplier Name
* Supplier Invoice No.
* Terms / Due Date
* Gross Amount
* VAT Amount
* Net Amount / Payable

## What Information We Need to Capture (The Data Fields)
As a reporting module, there is no data entry. It dynamically pulls data from the `vendor_bills` header and lines.

### Report Columns Map
| Column Name | Source | Description |
| :--- | :--- | :--- |
| Date | `bill_date` | The invoice date. |
| System No. | `bill_number` | The ERP generated number. |
| Reference No. | `supplier_invoice_no`| The physical invoice number. |
| Supplier | `suppliers.registered_name`| Name of the vendor. |
| Terms | `suppliers.payment_terms`| E.g., 30 Days. |
| Due Date | `due_date` | Payment deadline. |
| Tax Base | `vendor_bill_lines` | Total amount exclusive of VAT. |
| VAT Amount | `vendor_bill_lines` | Total Input VAT. |
| EWT Amount | `total_ewt` | Total Withholding Tax deducted. |
| Total Payable | `total_amount` | The final amount credited to AP. |
| Status | `status` | Current state of the bill. |

## The Supabase Database Architecture
We will create an optimized **SQL View** that flattens the Vendor Bill data into a single, easy-to-export row per document, pre-calculating the tax summaries.

### View: `vw_vendor_bill_register`
This view aggregates the line items and joins them with the header to provide a comprehensive listing.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `bill_id` | UUID | Required | Links to `vendor_bills.id`. |
| `bill_number` | Text | Required | System reference. |
| `bill_date` | Date | Required | Invoice date. |
| `supplier_name` | Text | Required | Vendor name. |
| `supplier_invoice_no`| Text | Required | Vendor reference. |
| `due_date` | Date | Required | Deadline. |
| `tax_base` | Numeric | Computed | Sum of line net amounts. |
| `input_vat` | Numeric | Computed | Sum of line VAT amounts. |
| `ewt_deducted` | Numeric | Required | Total EWT withheld. |
| `total_payable` | Numeric | Required | Final AP amount. |
| `status` | Text | Required | Status. |
| `created_by_name` | Text | Required | Joined from `users.name`. |
| `created_at` | Timestamp | Required | Timestamp of creation. |

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
