## Module Overview
The Bank Adjustments module records Debit and Credit Memos (DM/CM) initiated by the bank that do not originate from the normal AP/AR cycles. In the Philippines, this typically includes bank charges, service fees, interest income, withholding tax on interest (20% FWT deducted by the bank on interest income per NIRC Section 27(D)(1)), and returned checks. Proper recording ensures that the Book Balance precisely matches the Bank Statement Balance during reconciliation. Every posted Bank Adjustment generates a double-entry journal entry — the bank GL account is debited or credited, and the offsetting expense/income account is posted automatically.

**Philippine Compliance Note:** Banks in the Philippines withhold 20% Final Withholding Tax (FWT) on interest income credited to corporate accounts. When a Credit Memo for interest income is encoded, the system must split the entry: the gross interest is credited to Interest Income, and 20% FWT is debited to a Creditable Withholding Tax (CWT) receivable account, with the net balance debiting the Bank account.

## Dashboard UI

### The Action Bar
* **Search Bar:** Find an adjustment by Reference Number or Description.
* **Filter:** Filter by Bank Account, Type (Debit Memo/Credit Memo), or Date.
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the list to Excel or CSV.
* **Create New Adjustment Button:** Opens the form to encode a bank memo.
* **Import Button:** Upload multiple adjustments directly from a bank CSV file.

### The Data List
Default columns:
* Reference Number
* Date
* Bank Account
* Adjustment Type (DM/CM)
* Total Amount
* Status

**Pagination Mandate:** Server-side pagination via Supabase `.range(from, to)`. Default page size: 50 records.

## GL Posting Logic (System-Generated on Post)

### Standard Debit Memo (e.g., Bank Charges)
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Bank Charges Expense | Line Amount | | `bank_adjustment_lines.gl_account_id` |
| 2 | Bank GL Account | | Line Amount | `bank_accounts.gl_account_id` for the header bank |

### Standard Credit Memo (e.g., Refund, Error Reversal)
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Bank GL Account | Line Amount | | `bank_accounts.gl_account_id` |
| 2 | Offsetting Income/Liability | | Line Amount | `bank_adjustment_lines.gl_account_id` |

### Interest Income Credit Memo (Philippine FWT Split — Mandatory)
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Bank GL Account (net of FWT) | Gross - 20% FWT | | `bank_accounts.gl_account_id` |
| 2 | CWT Receivable — Bank Interest (20% FWT) | 20% of Gross Interest | | Configured FWT receivable account |
| 3 | Interest Income | | Gross Interest Amount | `bank_adjustment_lines.gl_account_id` |

*The CWT receivable represents the company's creditable tax against its income tax liability and is reportable on BIR Form 2306 received from the bank.*

## Supabase Database Architecture

### Table 1: `bank_adjustments` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, reference_number)` — prevents duplicate DM/CM numbers per tenant.
2. `journal_entry_id` is the mandatory GL traceability link. Populated on Post (Principle 7).
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the adjustment. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `bank_account_id` | UUID | Required, Foreign Key | Links to `bank_accounts.id`. |
| `reference_number` | Text | Required | `UNIQUE(company_id, reference_number)`. Bank DM/CM tracking number. |
| `adjustment_date` | Date | Required | Validated against open `accounting_periods`. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `adjustment_type` | Text | Required | ENUM: `debit_memo`, `credit_memo`. |
| `is_interest_income` | Boolean | Required, Default: `false` | Triggers the 20% FWT withholding split on posting. |
| `fwt_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. Required if `is_interest_income = true`. The CWT Receivable account. |
| `total_amount` | Numeric(15,2) | Required | Total gross value of the adjustment. |
| `fwt_amount` | Numeric(15,2) | Required, Default 0 | 20% FWT withheld by bank on interest. Computed on save. |
| `remarks` | Text | Nullable | Explanatory text. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. GL traceability link. |
| `posted_at` | Timestamp | Nullable | System-set on Post. Never editable. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required | ENUM: `draft`, `posted`, `voided`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, reference_number)`

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on posted bank adjustments
CREATE POLICY "block_update_on_posted_bank_adj"
ON bank_adjustments
FOR UPDATE
USING (status = 'draft');

-- Block DELETE on all bank adjustments
CREATE POLICY "block_delete_all_bank_adj"
ON bank_adjustments
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_bank_adj"
ON bank_adjustments
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: block edit when parent is posted
CREATE POLICY "block_update_posted_bank_adj_lines"
ON bank_adjustment_lines
FOR UPDATE
USING (
  (SELECT status FROM bank_adjustments WHERE id = adjustment_id) = 'draft'
);

-- Tenant isolation on lines
CREATE POLICY "tenant_isolation_bank_adj_lines"
ON bank_adjustment_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
