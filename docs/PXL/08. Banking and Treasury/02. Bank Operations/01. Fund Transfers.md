## Module Overview
The Fund Transfers module records the internal movement of cash between different bank accounts owned by the same company. In Philippine corporate treasury, moving funds from a collection account to a disbursement account is standard practice. This module ensures that both the withdrawal and the deposit are recorded simultaneously as a double-entry GL posting, keeping the General Ledger balanced and creating exact matching entries for Bank Reconciliation. A Fund Transfer is not complete until its journal entry is posted â€” cash movement without a GL posting is not an accounting record.

## GL Posting Logic (System-Generated on Post)

The system generates the following journal entries upon posting. The source bank account's GL is resolved from `bank_accounts.gl_account_id`.

**For each destination line:**
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Destination Bank GL Account | Transfer Amount | | `bank_accounts.gl_account_id` for destination |
| 2 | Source Bank GL Account | | Transfer Amount | `bank_accounts.gl_account_id` for source |

**If bank fee applies (additional JE lines):**
| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 3 | Bank Charges Expense | Bank Fee Amount | | `fund_transfers.fee_gl_account_id` |
| 4 | Source Bank GL Account | | Bank Fee Amount | `bank_accounts.gl_account_id` for source |

*The net effect: Source bank account is credited for the total transfer + fees. Destination bank account(s) are debited for their respective amounts. Bank charges are expensed.*

## Supabase Database Architecture

### Table 1: `fund_transfers` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, reference_number)` â€” prevents cross-tenant duplicate document numbers.
2. `journal_entry_id` is the mandatory GL traceability link. Populated on Post (Principle 7).
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the transfer. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `reference_number` | Text | Required | `UNIQUE(company_id, reference_number)`. |
| `transfer_date` | Date | Required | Validated against open `accounting_periods`. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `source_bank_id` | UUID | Required, Foreign Key | Links to `bank_accounts.id`. |
| `total_transfer_amount` | Numeric(15,2) | Required | Sum of all destination line amounts. |
| `bank_fee_amount` | Numeric(15,2) | Required, Default 0 | Transfer charges billed by the bank. |
| `fee_gl_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. Required if `bank_fee_amount > 0`. |
| `particulars` | Text | Required | Reason for transfer. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. GL traceability link. |
| `posted_at` | Timestamp | Nullable | System-set on Post. Never editable. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required | ENUM: `draft`, `posted`, `cancelled`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, reference_number)`

## Row-Level Security â€” Immutability Policy

```sql
-- Block UPDATE on posted transfers
CREATE POLICY "block_update_on_posted_transfer"
ON fund_transfers
FOR UPDATE
USING (status = 'draft');

-- Block DELETE on all fund transfers
CREATE POLICY "block_delete_all_fund_transfers"
ON fund_transfers
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_fund_transfers"
ON fund_transfers
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: block edit when parent is posted
CREATE POLICY "block_update_posted_transfer_lines"
ON fund_transfer_lines
FOR UPDATE
USING (
  (SELECT status FROM fund_transfers WHERE id = transfer_id) = 'draft'
);

-- Tenant isolation on lines
CREATE POLICY "tenant_isolation_fund_transfer_lines"
ON fund_transfer_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
