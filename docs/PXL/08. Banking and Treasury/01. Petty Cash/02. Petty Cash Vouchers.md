## Module Overview
Petty Cash Vouchers (PCV) record small daily expenditures disbursed from a specific Petty Cash Fund. In Philippine business practice, PCVs capture the payee, particulars, and associated VAT amounts for eventual liquidation. A PCV is an accounting event — when a PCV is approved and cash is disbursed, the Petty Cash Fund's GL account is credited and the corresponding expense account is debited. The GL posting happens at the PCV approval stage, not at replenishment, ensuring the expense is recognized when the cash leaves the fund.

## GL Posting Logic (System-Generated on Approval)

The GL posting fires when the PCV status changes from `draft` to `approved`. This is the moment cash is deemed disbursed.

| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Expense Account (per line) | Net Amount | | `petty_cash_voucher_lines.gl_account_id` |
| 2 | Input VAT (if applicable) | VAT Amount | | Configured Input VAT GL account |
| 3 | Petty Cash Fund GL Account | | Total Gross Amount | `petty_cash_funds.gl_account_id` |

*The net effect: the expense is recognized immediately and the Petty Cash Fund balance is reduced. When replenishment occurs later, the fund is restored by an AP/Cash Disbursement entry — no double-posting of the expense.*

## Supabase Database Architecture

### Table 1: `petty_cash_vouchers` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, pcv_number)` — prevents cross-tenant duplicate PCV numbers.
2. `journal_entry_id` is the mandatory GL traceability link. Populated on Approval (status → `approved`).
3. `approved_at` and `approved_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the PCV. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `pcf_id` | UUID | Required, Foreign Key | Links to `petty_cash_funds.id`. |
| `pcv_number` | Text | Required | `UNIQUE(company_id, pcv_number)`. |
| `pcv_date` | Date | Required | Validated against open `accounting_periods`. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `payee_name` | Text | Required | Recipient of funds. |
| `particulars` | Text | Required | Description of expense. |
| `total_gross_amount` | Numeric(15,2) | Required | Sum of all line `gross_amount` values. |
| `total_vat_amount` | Numeric(15,2) | Required, Default 0 | Sum of all line `vat_amount` values. |
| `total_net_amount` | Numeric(15,2) | Required | `total_gross_amount - total_vat_amount`. |
| `reference_doc` | Text | Nullable | Receipt or SI number from payee. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Approval. GL traceability link. |
| `approved_at` | Timestamp | Nullable | System-set on Approval. Never editable. |
| `approved_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required, Default `draft` | ENUM: `draft`, `approved`, `replenished`, `voided`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, pcv_number)`

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on approved, replenished, or voided PCVs
CREATE POLICY "block_update_on_approved_pcv"
ON petty_cash_vouchers
FOR UPDATE
USING (status = 'draft');

-- Block DELETE on all PCVs (voided PCVs stay on record)
CREATE POLICY "block_delete_all_pcv"
ON petty_cash_vouchers
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_pcv"
ON petty_cash_vouchers
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: block edit when parent is approved or later
CREATE POLICY "block_update_approved_pcv_lines"
ON petty_cash_voucher_lines
FOR UPDATE
USING (
  (SELECT status FROM petty_cash_vouchers WHERE id = pcv_id) = 'draft'
);

-- Tenant isolation on lines
CREATE POLICY "tenant_isolation_pcv_lines"
ON petty_cash_voucher_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
