# PXL ERP Blueprint: Depreciation Profiles

## Module Overview
The Depreciation Profiles setup governs the structured methodologies used to allocate the cost of an asset over its useful life. This module specifically enforces compliance with BIR regulations (e.g., RR 12-2012) regarding allowable depreciation schedules. By defining Standard, Straight-Line, or Accelerated depreciation schedules in advance, the system ensures consistent periodic expense calculations and standardizes how asset values degrade leading up to asset retirement.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **Search Bar:** Find profiles by Code or Description.
* **Filter:** Toggle between "Active" and "Inactive".
* **Customize View:** Select visible columns.
* **Export Button:** Export the profile list to CSV/Excel.
* **Create New Profile Button:** Add a new depreciation schedule structure.

Default columns displayed:

## The Supabase Database Architecture

### Table 1: `depreciation_profiles` (Header)
Defines the overarching calculation methodologies for asset depreciation schedules.

**Critical Database Rule:** Composite Unique Constraint `UNIQUE(company_id, profile_code)`. The `profile_code` column must NOT have a global unique constraint — multiple tenants may use the same code (e.g., `SL-FULL`).

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `profile_code` | Text | Required | Unique code within the tenant. `UNIQUE(company_id, profile_code)`. |
| `description` | Text | Required | Full description of the depreciation logic. |
| `method` | Text | Required | ENUM: `straight_line`, `syd`, `double_declining`. |
| `averaging_convention`| Text | Required | ENUM: `full_month`, `half_year`, `actual_days`. |
| `is_active` | Boolean | Default: `true` | Profile status. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

### Table 2: `depreciation_profile_configs` (Line Items)
Stores specific numerical configurations required to execute complex depreciation methods.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Denormalized for RLS efficiency. |
| `profile_id` | UUID | Required, Foreign Key | Links to `depreciation_profiles.id`. |
| `parameter_key` | Text | Required | e.g., `declining_multiplier`. |
| `parameter_value` | Numeric | Required | e.g., `2.0` for double declining. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |
