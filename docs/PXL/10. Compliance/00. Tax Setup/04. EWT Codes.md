## Module Overview
Expanded Withholding Tax (EWT) Codes are critical for generating BIR Form 2307 (Certificate of Creditable Tax Withheld at Source) and compiling the MAP (Monthly Alphalist of Payees) and QAP (Quarterly Alphalist of Payees). This module links standard tax deduction percentages to specific Alphanumeric Tax Codes (ATC) mandated by the BIR for services, rentals, goods purchases, and contractor payments.

Each EWT Code record must be a fully self-contained, queryable entity with its own code, description, and rate — so that Purchase Invoice lines, Supplier records, and MAP generation can all reference and display the EWT information without requiring deep JOINs through parent tables.

## What You Will See on the Screen (The Dashboard)

### The Action Bar
* **Search Bar:** Search by EWT Code or ATC.
* **Filter:** By Tax Form (1601EQ, 1601FQ) or Status.
* **Add EWT Code Button:** Create a new withholding parameter.

### The Data List
| Column | Description |
| :--- | :--- |
| EWT Code | Internal code (e.g., `EWT-WC158-10`). |
| Description | Full description (e.g., "Professional Fees - 10%"). |
| ATC Code | Linked BIR Alphanumeric Tax Code (e.g., `WC158`). |
| Rate (%) | Withholding percentage. |
| Form Type | BIR form for remittance (e.g., `1601EQ`). |
| GL Account | EWT Payable account where the withheld tax is credited on posting. |
| Status | Active / Inactive. |

## What Information We Need to Capture (The Data Fields)

### Section 1: EWT Code Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| EWT Code | Unique internal code (e.g., `EWT-WC158-10`). | Yes |
| Description | Full text description matching BIR ATC description (e.g., "Professional Fees - 10%"). | Yes |
| ATC Reference | Lookup to `atc_codes.id` (e.g., `WC158`). | Yes |
| Rate | The percentage to withhold (e.g., `10.00`). Stored here directly for query efficiency. | Yes |
| Form Type | BIR form for monthly/quarterly remittance. EWT: `1601EQ`. FWT: `1601FQ`. | Yes |
| GL Account | The EWT Payable GL account credited on Purchase Invoice posting. Links to `chart_of_accounts.id` via `tax_codes.gl_account_id`. | Yes (via parent tax_code) |

## The Supabase Database Architecture

### Retired Legacy Wrapper

**Current database rule:** the former company-specific EWT wrapper table was retired in session 91. Supplier/customer withholding defaults now point directly at `atc_codes`; company-specific posting accounts come from GL posting configuration, and report evidence uses posted `tax_detail_entries`.
