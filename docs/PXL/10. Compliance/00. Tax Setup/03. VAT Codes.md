## Module Overview
VAT Codes specifically handle the application of Value-Added Tax. In the Philippines, VAT carries multiple classifications with distinct tax treatment and BIR reporting buckets: **Regular/Taxable (12%)**, **Zero-Rated (0%)**, and **Exempt**. This module maps these classifications to ensure correct presentation on BIR Form 2550Q, the SLSP (Summary List of Sales and Purchases), and the RELIEF DAT file.

The `vat_classification` field is not optional — it is the primary discriminator that determines which column a transaction is reported under in BIR Form 2550Q (AT = Amount Subject to Tax, AZ = Zero-Rated Amount, AE = Amount Exempt from VAT). Misconfiguring this causes incorrect VAT returns.

## The Supabase Database Architecture

**Critical Database Rule:** `vat_codes` is a standalone table — not a view. Implementing this as a view of `tax_codes` removes the ability to add VAT-specific columns (`vat_classification`, `relief_category`) and breaks FK integrity for VAT-specific lookups from Sales Invoice and Purchase Invoice lines.

### Table 1: `vat_codes`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `tax_code_id` | UUID | Required, Foreign Key | Links to master `tax_codes.id`. The parent tax code record. |
| `vat_classification` | Text | Required | **`regular`**, **`zero_rated`**, or **`exempt`**. Drives 2550Q column assignment and RELIEF DAT bucket. |
| `transaction_type` | Text | Required | `input_vat` (Purchases) or `output_vat` (Sales). |
| `relief_category` | Text | Nullable | BIR RELIEF system category code for DAT file generation. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

**Composite Unique Constraint:** `UNIQUE(tax_code_id, transaction_type)`

## BIR 2550Q Column Mapping Reference

| `vat_classification` | BIR Form 2550Q Column | RELIEF Bucket |
| :--- | :--- | :--- |
| `regular` | AT (Amount Subject to Tax) | Taxable |
| `zero_rated` | AZ (Zero-Rated Amount) | Zero-Rated |
| `exempt` | AE (Exempt Amount) | Exempt |

This mapping must be applied automatically when generating the 2550Q working paper and RELIEF DAT file. The system should never require a user to manually select the 2550Q column.
