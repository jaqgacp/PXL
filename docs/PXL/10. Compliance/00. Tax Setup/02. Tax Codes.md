## Module Overview
Tax Codes act as the master classification for all tax-related computations. Instead of hard-coding percentages on documents, users select a Tax Code (which inherits specific rates, GL mappings, and ATC linkages). This centralized approach allows the business to quickly adapt to legislative changes in Philippine tax rates.

## What You Will See on the Screen (The Dashboard)
A consolidated registry of all tax codes.

### The Action Bar (Top of the page):
* **Search Bar:** Code or Description.
* **Filter:** Tax Type (VAT, EWT, FWT, Percentage).
* **Create Code Button:** Add a new tax parameter.

### The Data List (The main table):
* Tax Code
* Description
* Tax Type
* Rate (%)
* Status

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `tax_codes`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `code` | Text | Required, Unique | E.g., `VAT12`. |
| `description` | Text | Required | E.g., `Standard 12% VAT`. |
| `tax_type` | Text | Required | `vat`, `ewt`, `fwt`, `pt`. |
| `rate` | Numeric | Required | E.g., `12.00`. |
| `gl_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
