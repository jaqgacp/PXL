Current implementation note: `src/pages/BIRFormConfigPage.tsx` is a read-only reference list backed by `ref_compliance_forms`. It does not currently edit the legacy `bir_forms` or `bir_form_mappings` tables described below. Do not preserve broad writes merely because this blueprint describes a future “Update Mappings” action.

## Module Overview
The BIR Form Configuration module maps transaction data directly to the Bureau of Internal Revenue's required tax returns (e.g., 2550M/Q for VAT, 1601-EQ for EWT, 1702Q for Income Tax). This ensures that standard system reports perfectly mimic the fields of official BIR forms, simplifying end-of-month and end-of-year tax filing.

## What You Will See on the Screen (The Dashboard)
A list of supported BIR forms and their mapping parameters.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Form No. (e.g., 2550Q).
* **Filter:** Active/Inactive Forms.
* **Future governed Update Mappings action:** Modify which approved sources feed a form line. This is not present in the current read-only page and must not be added until write authority, audit evidence, and validation are explicitly governed.

### The Data List (The main table):
* Form Number (e.g., "BIR Form 2550Q")
* Form Description (e.g., "Quarterly Value-Added Tax Return")
* Frequency (Monthly, Quarterly, Annual)
* Status (Active)

## The Supabase Database Architecture

**Critical Database Rules:** Prevent duplicates; allow required authenticated reads; deny ordinary authenticated insert/update/delete; route any future maintenance through an explicitly authorized and audited role or RPC.

### Table 1: `bir_forms` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `form_number` | Text | Required, Unique | E.g., `2550Q`. |
| `description` | Text | Required | E.g., `Quarterly VAT Return`. |
| `frequency` | Text | Required | `monthly`, `quarterly`, `annual`. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

### Table 2: `bir_form_mappings` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `form_id` | UUID | Required, Foreign Key | Links to `bir_forms.id`. |
| `line_identifier` | Text | Required | E.g., `Part_II_Item_15A`. |
| `source_type` | Text | Required | `gl_account`, `tax_code`, `calc`. |
| `source_id` | UUID | Nullable | ID of account or tax code. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
