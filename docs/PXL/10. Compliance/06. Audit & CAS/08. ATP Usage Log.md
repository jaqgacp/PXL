## Purpose

The ATP Usage Log monitors BIR Authority To Print ranges using the same immutable evidence produced by document-number allocation. It shows actual reserved, issued, and voided/gap counts rather than estimating usage from a mutable UI counter.

## Authoritative Sources

- `number_series` stores the company/branch/document configuration and optional `atp_series_start`, `atp_series_end`, and alert threshold.
- `cas_document_number_issuances` records every allocated number and its forward-only lifecycle (`reserved`, `issued`, `voided`, or `abandoned`). Unused reservations remain evidence and are never recycled.
- `vw_cas_atp_usage` combines the two sources with branch/document labels, current/next sequence, remaining authorized numbers, usage percentage, evidence counts, exhaustion, and low-range alert state.

## Enforcement

`fn_next_document_number(company, branch, code)` locks the series, selects the next authorized sequence, and writes the issuance row atomically. When an ATP range is configured:

- allocation starts no earlier than the authorized start;
- allocation beyond the authorized end is rejected before a number is returned;
- sequence/current counters cannot be moved backward;
- series identity and number formatting cannot be rewritten after evidence exists;
- extending an ATP end safely is permitted, but shrinking it below issued/current numbers is rejected.

The allocator retains its existing authenticated three-argument contract because several transaction pages still allocate immediately before inserting their header. There is no arbitrary branch-less overload and no one-unresolved-reservation rule.

## User Interface

`CASATPUsageLogPage` reads `vw_cas_atp_usage` and shows the authorized range, current sequence, reserved/issued/voided counts, remaining numbers, usage percentage, and alert/exhaustion state per branch and document type.
