## Module Overview
The Fiscal Calendar breaks down a Fiscal Year into individual periods (usually months, or 13 periods for some retail setups). These periods dictate when sub-ledgers close and when monthly financial reports (and VAT returns) are prepared.

## What You Will See on the Screen (The Dashboard)
A breakdown of periods belonging to a selected Fiscal Year.

### The Action Bar (Top of the page):
* **Year Selector:** Choose the Fiscal Year.
* **Generate Periods Button:** Auto-generate 12 months based on FY boundaries.

### The Data List (The main table):
* Period Name (e.g., "Oct-2026")
* Start Date
* End Date
* Global Status (Open/Closed)

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `fiscal_calendar_periods`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `fiscal_year_id` | UUID | Required, Foreign Key | Links to `fiscal_years.id`. |
| `sequence_no` | Integer | Required | Month number. |
| `period_name` | Text | Required | E.g., `Jan 2026`. |
| `start_date` | Date | Required | Start date of period. |
| `end_date` | Date | Required | End date of period. |
| `status` | Text | Required | `open`, `closed`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
