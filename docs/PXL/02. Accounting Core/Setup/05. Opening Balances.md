## Module Overview
Opening Balances allow companies to transition from legacy systems or manual books to the new ERP. It sets the initial Trial Balance (TB) at the cut-off date. This must tie back precisely to the audited financial statements of the preceding period.

## What You Will See on the Screen (The Dashboard)
A massive data entry grid for inputting or uploading the initial TB.

### The Action Bar (Top of the page):
* **Cut-off Date Picker:** Date of the initial balances.
* **Import TB Button:** Upload CSV of accounts and balances.
* **Validate Button:** Check if Total Debits = Total Credits.
* **Post Balances Button:** Commit to the ledger.

### The Data List (The main table):
* Account Code
* Account Name
* Debit Amount
* Credit Amount

## What Information We Need to Capture (The Data Fields)

### Section 1: Opening Balance Header
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Cut-off Date | The day before go-live. | Yes |
| Reference No | Usually "OB-001" or similar. | Yes |
| Total Debit | Auto-summed. | Yes |
| Total Credit | Auto-summed. | Yes |
| Status | Draft, Balanced, Posted. | Yes |

### Section 2: Account Lines
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Account | Lookup from Chart of Accounts. | Yes |
| Debit | Amount if Debit normal. | Optional |
| Credit | Amount if Credit normal. | Optional |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `opening_balance_headers`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `cut_off_date` | Date | Required | Balances as of this date. |
| `reference_no` | Text | Required | OB Reference. |
| `status` | Text | Required | `draft`, `balanced`, `posted`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

### Table 2: `opening_balance_lines`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `header_id` | UUID | Required, Foreign Key | Links to `opening_balance_headers.id`. |
| `account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `debit_amount` | Numeric | Default: `0` | OB Debit. |
| `credit_amount` | Numeric | Default: `0` | OB Credit. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
