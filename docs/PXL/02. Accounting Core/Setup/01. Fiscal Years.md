## Module Overview
Fiscal Years setup defines the reporting year for the company. In the Philippines, businesses generally use a standard Calendar Year (Jan 1 to Dec 31) for BIR reporting, but some SEC-registered corporations opt for a Fiscal Year ending in another month (e.g., June 30). This module sets up the annual boundaries for financial statements and retained earnings roll-forward.

## What You Will See on the Screen (The Dashboard)
A list of generated fiscal years for the company.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Year Name.
* **Filter:** Status (Open, Closed).
* **Create Fiscal Year Button:** Initialize a new year.

### The Data List (The main table):
* Year Name (e.g., FY 2026)
* Start Date
* End Date
* Is Calendar Year?
* Status (Open, Closed, Historical)

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `fiscal_years`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `year_name` | Text | Required | E.g., `FY2026`. |
| `start_date` | Date | Required | E.g., `2026-01-01`. |
| `end_date` | Date | Required | E.g., `2026-12-31`. |
| `is_calendar` | Boolean | Default: `false` | `true` or `false`. |
| `status` | Text | Required | `open`, `closed`. |
| `retained_earnings_id`| UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
