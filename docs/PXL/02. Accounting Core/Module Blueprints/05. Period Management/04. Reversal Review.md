## Module Overview
Reversing a posted transaction in accounting is a highly sensitive action. Because the General Ledger is immutable in CAS-compliant systems in the Philippines, a "reversal" does not delete the original entry; instead, it posts an exact opposite entry (Debits become Credits, Credits become Debits) to net the balance to zero. The Reversal Review module provides a strict governance layer. If a user requests to reverse a posted document, the request lands here for the Controller to review the justification before the reversing entry is authorized and committed to the GL.

### GL Posting Behavior on Approval
If approved:
1. The system creates a NEW document (e.g., JE-2026-005-REV) mirroring the lines of the original, but swapping Debits and Credits.
2. It posts this new document to `general_ledger_entries`.
3. It updates the status of the original document to `reversed` to prevent duplicate reversals.
4. Total Debits MUST still equal Total Credits on the reversing entry.
