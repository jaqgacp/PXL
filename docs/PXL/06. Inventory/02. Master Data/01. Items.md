# PXL ERP Blueprint: Items Master Data

## Module Overview
The Items module is the core catalog of all trackable goods and services in the system. It is fundamental for ensuring correct Philippine VAT and withholding tax application in sales and purchasing transactions. Every item explicitly defines its tax treatment, valuation method (FIFO/Weighted Average), and unit of measure, which directly drives accurate financial reporting and BIR compliance.

## What You Will See on the Screen (The Dashboard)
When you open the "Items" page, you will see a managed catalog of products and services.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Item Code, Name, or Barcode.
* **Filter:** Dropdown to show "Active" or "Inactive", and filter by "Category".
* **Customize View:** Choose columns to display.
* **Export Button:** Download the list to Excel or CSV.
* **Import Button:** Bulk upload items via CSV/Excel, crucial for initial system setup.
* **Create New Item Button:** Opens the item creation form.

### The Data List (The main table - must be customizable):
Columns:
* Image (Thumbnail)
* Item Code
* Item Name
* Category
* Item Type (Inventory, Non-Inventory, Service)
* Base UOM (Unit of Measure)
* Default Price
* Status (Active/Inactive badge)

## The Supabase Database Architecture (For Lovable)

### Table: `items`
This is the master catalog table. Stock quantities are NEVER stored here; they are calculated dynamically via the `inventory_movements` ledger which is tightly coupled to Sales and Purchasing.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the item. |
| `item_type` | Text | Required | `inventory`, `non_inventory`, `service`. |
| `item_code` | Text | Required, Unique | SKU or internal code. |
| `item_name` | Text | Required | The product/service name. |
| `category_id` | UUID | Nullable, Foreign Key | Links to `item_categories.id`. |
| `barcode` | Text | Nullable, Unique | Scannable barcode. |
| `default_uom_id` | UUID | Required, Foreign Key | Links to `uom.id`. |
| `costing_method` | Text | Nullable | `fifo`, `moving_average`, `standard`. |
| `default_cost` | Numeric(15,4) | Nullable | Standard cost. |
| `default_price` | Numeric(15,4) | Nullable | Standard price. |
| `reorder_level` | Numeric(15,4) | Default: 0 | Low stock warning threshold. |
| `default_warehouse_id`| UUID | Nullable, Foreign Key | Links to `warehouses.id`. |
| `allow_negative_stock`| Boolean | Default: `false`| Control flag for stock validations. |
| `default_tax_code_id` | UUID | Required, Foreign Key | Links to `tax_codes.id`. |
| `revenue_account_id` | UUID | Nullable, Foreign Key | Links to `accounts.id`. |
| `expense_account_id` | UUID | Nullable, Foreign Key | Links to `accounts.id`. |
| `cogs_account_id` | UUID | Nullable, Foreign Key | Links to `accounts.id`. |
| `inventory_account_id`| UUID | Nullable, Foreign Key | Links to `accounts.id`. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | User who created the record. |
| `updated_by` | UUID | Nullable, Foreign Key | User who last modified the record. |
| `created_at` | Timestamp | Auto | Date created. |
| `updated_at` | Timestamp | Auto | Date last updated. |


**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
