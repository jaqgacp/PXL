# PXL ERP Blueprint: Warehouses Master Data

## Module Overview
The Warehouses module tracks the physical or logical locations where inventory is stored. In the Philippine context, warehouses must be accurately mapped to specific company branches and RDOs (Revenue District Offices) because the movement of goods between branches may have tax implications, and accurate location tracking ensures reliable inventory valuation for year-end reporting.

## What You Will See on the Screen (The Dashboard)
When you navigate to "Warehouses," you will see a list of all your storage locations.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Warehouse Code or Name.
* **Filter:** Filter by "Company/Branch" or Status ("Active" vs "Inactive").
* **Customize View:** Select which data columns to show.
* **Export Button:** Download the list to Excel or CSV.
* **Import Button:** Bulk create warehouses via CSV/Excel template.
* **Create New Warehouse Button:** Form to define a new storage facility.

### The Data List (The main table - must be customizable):
Columns:
* Warehouse Code
* Warehouse Name
* Branch/Company
* Location/City
* Status (Active/Inactive)

## The Supabase Database Architecture

**Critical Database Rule:**
A composite unique constraint is required for `company_id` + `warehouse_code` to ensure that a warehouse code is unique per company/branch.

### Table: `warehouses`
This table stores the physical and logical storage spaces. Stock balances are not stored here; they are computed dynamically from the `inventory_movements` ledger grouping by `warehouse_id`.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the warehouse. |
| `warehouse_code` | Text | Required, Unique | Short identifier code. |
| `warehouse_name` | Text | Required | Descriptive name. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Maps to legal branch. |
| `is_transit` | Boolean | Default: `false`| Flags if used strictly for in-transit stock. |
| `address_line_1` | Text | Required | Primary address. |
| `address_line_2` | Text | Nullable | Secondary address (Barangay/Subdivision). |
| `city` | Text | Required | City or Municipality. |
| `province` | Text | Required | Province. |
| `contact_person` | Text | Nullable | Supervisor/Manager name. |
| `contact_number` | Text | Nullable | Phone number. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | User who created the record. |
| `updated_by` | UUID | Nullable, Foreign Key | User who last modified the record. |
| `created_at` | Timestamp | Auto | Date created. |
| `updated_at` | Timestamp | Auto | Date last updated. |


**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
