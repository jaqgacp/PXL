# PXL ERP Blueprint: Stock Transfer

## Module Overview
The Stock Transfer module manages the movement of inventory from one warehouse/branch to another. In the Philippines, transferring goods between branches (inter-branch transfers) requires specific tracking because it dictates where physical assets reside and which RDO jurisdiction the inventory falls under. Approved transfers automatically deduct stock from the source warehouse and add stock to the destination warehouse via the `inventory_movements` ledger.

## What You Will See on the Screen (The Dashboard)

### The Action Bar (Top of the page):
* **Search Bar:** Search by Transfer Number.
* **Filter:** Filter by Status (Draft, In Transit, Received, Cancelled) and Date.
* **Customize View:** Choose columns to display.
* **Export Button:** Download list to Excel or CSV.
* **Create New Transfer Button:** Opens the transfer entry form.

### The Data List (The main table - must be customizable):
Columns:
* Transfer Number
* Date
* Source Warehouse
* Destination Warehouse

* Created By

## The Supabase Database Architecture

### Table 1: `stock_transfers` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `transfer_number`| Text | Required, Unique | E.g., "ST-2026-001". |
| `transfer_date` | Date | Required | Date dispatched. |
| `source_warehouse_id`| UUID | Required, Foreign Key | Links to `warehouses.id`. |
| `dest_warehouse_id` | UUID | Required, Foreign Key | Links to `warehouses.id`. |
| `carrier_name` | Text | Nullable | Driver or shipping company. |
| `plate_number` | Text | Nullable | Delivery vehicle info. |
| `status` | Text | Required | `draft`, `in_transit`, `received`, `cancelled`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `stock_transfer_items` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `transfer_id` | UUID | Required, Foreign Key | Links to `stock_transfers.id`. |
| `item_id` | UUID | Required, Foreign Key | Links to `items.id`. |
| `transfer_qty` | Numeric(15,4) | Required | Amount being moved. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

**Crucial Logic:**
When status changes to `in_transit`, the system inserts a negative quantity in `inventory_movements` for the Source Warehouse.
When status changes to `received`, the system inserts a positive quantity in `inventory_movements` for the Destination Warehouse.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
