## Module Overview
The Asset Categories module defines the classification of Property, Plant, and Equipment (PPE) aligned with Philippine Financial Reporting Standards (PFRS/PFRS for SMEs) and Bureau of Internal Revenue (BIR) regulations. This setup dictates critical policies such as Capitalization Thresholds (ensuring items below the set PHP threshold are immediately expensed rather than capitalized) and maps directly to the Chart of Accounts for automated journal entry generation during acquisition, depreciation, and asset retirement.

## The Supabase Database Architecture

### Table 1: `asset_categories` (Header)
Stores the primary definition and threshold policies for each asset classification.

**Critical Database Rule:** Composite Unique Constraint `UNIQUE(company_id, category_code)` to prevent cross-tenant duplication. The `category_code` column must NOT have a global unique constraint — two different companies may both use the code `IT-EQP`.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `category_code` | Text | Required | Unique code within the tenant. `UNIQUE(company_id, category_code)`. |
| `category_name` | Text | Required | Descriptive name of the category. |
| `capitalization_threshold` | Numeric | Required | The PHP amount threshold for capitalization. |
| `default_useful_life_months` | Integer | Required | Standard useful life for the depreciation schedule. |
| `residual_value_percent` | Numeric | Required | Default percentage for asset retirement salvage value. |
| `is_active` | Boolean | Default: `true` | Status of the category. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

### Table 2: `asset_category_accounts` (Line Items)
Stores the specific GL account mappings tied to the category, enabling automated accounting entries.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Denormalized for RLS efficiency. |
| `category_id` | UUID | Required, Foreign Key | Links to `asset_categories.id`. |
| `account_type` | Text | Required | `asset`, `accumulated_depreciation`, `depreciation_expense`, `disposal_gain_loss`. |
| `account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `created_by` | UUID | Required, Foreign Key | Link to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Link to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |
