## Module Overview
The Disposal module enforces Asset Retirement rules when an asset reaches the end of its useful life, is sold, scrapped, or donated. It removes the asset from the active Asset Register, calculates the exact Gain or Loss on Disposal by comparing sale proceeds against the asset's current Net Book Value (NBV), and automatically posts the double-entry journal entries to the GL for PFRS and BIR tax calculation purposes. The gain or loss is computed and posted — not just calculated and discarded.

## GL Posting Logic (System-Generated on Post)

Upon posting, the system generates the following journal entries. All GL accounts are resolved from `asset_category_accounts` for the disposed asset's category.

| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Accumulated Depreciation | Accumulated Dep. Balance | | `asset_category_accounts` where `account_type = 'accumulated_depreciation'` |
| 2 | Cash / AR (if sold) | Proceeds Amount | | `customers.ar_account_id` or bank account if cash |
| 3 | Asset Account | | Original Acquisition Cost | `asset_category_accounts` where `account_type = 'asset'` |
| 4a | Loss on Disposal (if loss) | `Proceeds < NBV` → debit difference | | `asset_category_accounts` where `account_type = 'disposal_gain_loss'` |
| 4b | Gain on Disposal (if gain) | | `Proceeds > NBV` → credit difference | `asset_category_accounts` where `account_type = 'disposal_gain_loss'` |

*On posting, the asset's status in `asset_register` is updated to `disposed` and the record is locked.*

## The Supabase Database Architecture

### Table 1: `asset_disposals` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, disposal_no)` — prevents cross-tenant number duplication.
2. `journal_entry_id` is the mandatory GL traceability link. Populated on Post.
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `disposal_no` | Text | Required | `UNIQUE(company_id, disposal_no)`. |
| `disposal_date` | Date | Required | Validated against open `accounting_periods`. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `disposal_type` | Text | Required | ENUM: `sale`, `scrap`, `donation`, `trade_in`. |
| `customer_id` | UUID | Nullable, Foreign Key | Links to `customers.id`. Required if `disposal_type = 'sale'`. |
| `or_ci_reference` | Text | Nullable | OR/CI number for the sale. Required if `disposal_type = 'sale'`. |
| `total_proceeds` | Numeric(15,2) | Required, Default 0 | Sum of all line item proceeds. |
| `total_gain_loss` | Numeric(15,2) | Required, Default 0 | Net gain (positive) or loss (negative) across all lines. |
| `remarks` | Text | Nullable | Context for the disposal. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. GL traceability link. |
| `posted_at` | Timestamp | Nullable | System-set on Post action. Never editable. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required | ENUM: `draft`, `approved`, `posted`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, disposal_no)`

| Column Name | Data Type | Rules | What it stores |

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on posted disposals
CREATE POLICY "block_update_on_posted_disposal"
ON asset_disposals
FOR UPDATE
USING (status = 'draft' OR status = 'approved');

-- Block DELETE on all disposals
CREATE POLICY "block_delete_all_disposals"
ON asset_disposals
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_disposals"
ON asset_disposals
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: block edits when parent is posted
CREATE POLICY "block_update_posted_disposal_lines"
ON asset_disposal_lines
FOR UPDATE
USING (
  (SELECT status FROM asset_disposals WHERE id = disposal_id) IN ('draft', 'approved')
);

-- Tenant isolation on lines
CREATE POLICY "tenant_isolation_disposal_lines"
ON asset_disposal_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
