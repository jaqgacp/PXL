## Module Overview
The Impairment module ensures compliance with Philippine Accounting Standards (PAS 36) by recording reductions in the recoverable amount of fixed assets. If an asset is damaged, becomes technologically obsolete, or otherwise loses value faster than its regular depreciation schedule, this module calculates the Impairment Loss. The loss is then posted to the GL as a double-entry journal entry (Dr. Impairment Loss Expense / Cr. Accumulated Impairment Loss), which adjusts the asset's Net Book Value downward and recalibrates the base for all future depreciation computations. The impairment amount is a computed, posted figure — not a number that sits in a table disconnected from the General Ledger.

## GL Posting Logic (System-Generated on Post)

Upon posting, the system generates the following journal entries per impaired asset line:

| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Impairment Loss Expense | Impairment Loss Amount | | `asset_category_accounts` where `account_type = 'impairment_loss_expense'` |
| 2 | Accumulated Impairment Loss | | Impairment Loss Amount | `asset_category_accounts` where `account_type = 'accumulated_impairment'` |

*On posting:*
- *`asset_register.net_book_value` is reduced by the impairment loss amount.*
- *`asset_register` records `last_impairment_date` and `total_impairment_recognized` for future depreciation recalibration.*
- *Future depreciation is recalculated on the reduced carrying amount over the remaining useful life.*

## The Supabase Database Architecture

### Table 1: `asset_impairments` (Header)

**Critical Database Rules:**
1. `UNIQUE(company_id, impairment_no)` — prevents cross-tenant number duplication.
2. `journal_entry_id` is the mandatory GL traceability link. Populated on Post.
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `branch_id` | UUID | Nullable, Foreign Key | Links to `branches.id`. |
| `impairment_no` | Text | Required | `UNIQUE(company_id, impairment_no)`. |
| `assessment_date` | Date | Required | Date the impairment was assessed. Validated against open periods. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `reason` | Text | Required | ENUM: `physical_damage`, `obsolescence`, `market_decline`, `regulatory_change`. |
| `remarks` | Text | Nullable | Detailed PAS 36 justification. |
| `total_impairment_loss` | Numeric(15,2) | Required, Default 0 | Sum of all line impairment losses. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. GL traceability link. |
| `posted_at` | Timestamp | Nullable | System-set on Post action. Never editable. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required | ENUM: `draft`, `approved`, `posted`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, impairment_no)`

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on posted impairments
CREATE POLICY "block_update_on_posted_impairment"
ON asset_impairments
FOR UPDATE
USING (status = 'draft' OR status = 'approved');

-- Block DELETE on all impairments (PAS 36 records are permanent evidence)
CREATE POLICY "block_delete_all_impairments"
ON asset_impairments
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_impairments"
ON asset_impairments
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: block edits when parent is posted
CREATE POLICY "block_update_posted_impairment_lines"
ON asset_impairment_lines
FOR UPDATE
USING (
  (SELECT status FROM asset_impairments WHERE id = impairment_id) IN ('draft', 'approved')
);

-- Tenant isolation on lines
CREATE POLICY "tenant_isolation_impairment_lines"
ON asset_impairment_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
