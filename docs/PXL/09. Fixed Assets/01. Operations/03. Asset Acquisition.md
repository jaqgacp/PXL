## Module Overview
The Asset Acquisition module manages the procurement and capitalization of fixed assets. It ensures that any purchased item proposed for capitalization passes the defined Capitalization Thresholds — items below the threshold are automatically redirected as direct expenses, not assets. Once approved, this process links the asset to the Chart of Accounts, sets the foundation for its depreciation schedule, officially inputs the item into the Asset Register, and generates the mandatory double-entry GL posting (Dr. Asset Account / Cr. Accounts Payable or Cash). All GL account mappings are inherited from `asset_categories` — no manual account selection is required.

## Capitalization Threshold Failure Path

If `unit_cost <= asset_categories.capitalization_threshold`:
- The system **blocks** saving the line.
- Alert displayed: *"This item's unit cost of ₱[X] does not meet the capitalization threshold of ₱[Y] for category [Z]. Please remove this line and process the purchase through Purchase Invoice as a direct expense."*
- The acquisition document cannot be posted with sub-threshold lines present.

## GL Posting Logic (System-Generated on Post)

Upon posting, the system automatically generates a double-entry Journal Entry. No manual JE is needed.

**GL Accounts are inherited from `asset_category_accounts` per line item's category.**

| # | Account | Debit | Credit | Source |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Asset Account (per category) | Total Cost per line | | `asset_category_accounts` where `account_type = 'asset'` for the line's `category_id` |
| 2 | Accounts Payable | | Grand Total of all lines | `suppliers.ap_account_id` |

*If payment is by cash/check rather than on credit, replace the AP credit with the bank/cash GL account.*

## The Supabase Database Architecture

### Table 1: `asset_acquisitions` (Header)
Captures the overarching transaction details of the asset purchase.

**Critical Database Rules:**
1. `UNIQUE(company_id, acquisition_no)` — prevents cross-tenant number duplication.
2. `journal_entry_id` is populated on Post. It is the mandatory link that makes the acquisition traceable in the General Ledger (Principle 7).
3. `posted_at` and `posted_by` are system-set. Never editable.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `acquisition_no` | Text | Required | `UNIQUE(company_id, acquisition_no)`. |
| `acquisition_date` | Date | Required | Date the assets were capitalized. Validated against open periods. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. Set on save. |
| `vendor_id` | UUID | Required, Foreign Key | Links to `suppliers.id`. |
| `po_reference` | Text | Nullable | Links to a PO if applicable. |
| `invoice_reference` | Text | Required | The supplier OR/SI number. |
| `grand_total` | Numeric(15,2) | Required, Default 0 | Sum of all line `total_cost` values. |
| `journal_entry_id` | UUID | Nullable, Foreign Key | Links to `journal_entries.id`. Populated on Post. The GL traceability link. |
| `posted_at` | Timestamp | Nullable | System-set on Post. Never editable. |
| `posted_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `status` | Text | Required | ENUM: `draft`, `approved`, `posted`, `rejected`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, acquisition_no)`

### Table 2: `asset_acquisition_lines` (Line Items)
Details the specific assets being capitalized. On posting, each approved line is automatically pushed to create a record in `asset_register`.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Denormalized for RLS. |
| `acquisition_id` | UUID | Required, Foreign Key | Links to `asset_acquisitions.id` (ON DELETE CASCADE). |
| `category_id` | UUID | Required, Foreign Key | Links to `asset_categories.id`. Drives GL account lookup and threshold validation. |
| `asset_description` | Text | Required | Name/description of the item being capitalized. |
| `quantity` | Integer | Required, CHECK (quantity > 0) | Number of units purchased. |
| `unit_cost` | Numeric(15,4) | Required | Cost per unit. Must be > `asset_categories.capitalization_threshold`. |
| `total_cost` | Numeric(15,2) | Required | `quantity × unit_cost`. |
| `asset_gl_account_id` | UUID | Required, Foreign Key | Auto-resolved from `asset_category_accounts` where `account_type = 'asset'`. |
| `asset_register_id` | UUID | Nullable, Foreign Key | Links to `asset_register.id`. Populated when this line is pushed to the Asset Register on Post. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

## Row-Level Security — Immutability Policy

```sql
-- Block UPDATE on posted or rejected acquisitions
CREATE POLICY "block_update_on_posted_acquisition"
ON asset_acquisitions
FOR UPDATE
USING (status = 'draft' OR status = 'approved');

-- Block DELETE on all acquisitions (voiding changes status, never deletes)
CREATE POLICY "block_delete_all_acquisitions"
ON asset_acquisitions
FOR DELETE
USING (false);

-- Tenant isolation
CREATE POLICY "tenant_isolation_acquisitions"
ON asset_acquisitions
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);

-- Line items: block edits on posted acquisitions
CREATE POLICY "block_update_posted_acquisition_lines"
ON asset_acquisition_lines
FOR UPDATE
USING (
  (SELECT status FROM asset_acquisitions WHERE id = acquisition_id) IN ('draft', 'approved')
);

-- Tenant isolation on lines
CREATE POLICY "tenant_isolation_acquisition_lines"
ON asset_acquisition_lines
FOR ALL
USING (company_id = (auth.jwt() ->> 'company_id')::UUID);
```
