# PXL ERP Blueprint: Summary List of Purchases (SLP)

## Module Overview
The Summary List of Purchases (SLP) is a strict, legally mandated report required by the Bureau of Internal Revenue (BIR) in the Philippines. It must be submitted as an attachment to the Quarterly VAT Return (BIR Form 2550Q). The SLP requires data to be formatted in a highly specific `.dat` file structure generated through the BIR Relief system. This module compiles the necessary data from the Input VAT Review and formats it precisely so the user can bypass manual entry and export the required `.dat` file directly from the ERP.

## What You Will See on the Screen (The Dashboard)
When you open the "SLP" page, you will see a highly structured, compliance-focused dashboard.

### The Action Bar (Top of the page):
* **Taxable Year Filter:** Dropdown for the year.
* **Quarter Filter:** Dropdown for Q1, Q2, Q3, or Q4.
* **Month Filter:** View specific months within the quarter.
* **Validate Button:** Runs a system check for missing TINs, invalid addresses, or formatting errors before export.
* **Export DAT Button:** The primary function. Generates the `.dat` file required by the BIR.
* **Export Excel Button:** Downloads a readable version for the accountant's working papers.

## What Information We Need to Capture (The Data Fields)
This module acts as an extraction and formatting tool over the `vw_input_vat_review` data. No data is entered here; it relies 100% on the accuracy of the Vendor Bills and Cash Purchases.

### BIR Required Data Mapping
| Field Name | Description | Source Mapping |
| :--- | :--- | :--- |
| TIN | 9-digit TIN without dashes. | Cleaned `suppliers.tin` |
| Branch Code | 3-to-5 digit branch code. | Parsed from `suppliers.tin` |
| Registered Name| Supplier's official name. | `suppliers.registered_name` |
| Last/First Name| Only if individual. | Parsed from `suppliers` if applicable |
| Address | Full address. | `suppliers.address_line_1` + `address_line_2` |
| Gross Purchases| Total invoice value. | Computed gross from transactions |
| Exempt | Non-vatable purchases. | Lines tagged with Exempt VAT code |
| Zero-Rated | Purchases rated at 0%. | Lines tagged with Zero-Rated VAT code |
| Taxable Goods | Regular vatable items. | Lines tagged with Regular VAT (Goods) |
| Taxable Services| Vatable services. | Lines tagged with Regular VAT (Services) |
| Capital Goods | Assets > 1M PHP. | Lines tagged with Capital Goods VAT code |
| Input Tax | Total 12% claimed. | Computed tax amount |

## The Supabase Database Architecture
We do not create a new data table for this. We create a highly specialized **SQL View** that formats the data exactly as the BIR dat file requires.

### View: `vw_bir_slp_export`
This view aggregates data by Supplier per Month, classifying the purchases into the specific BIR buckets based on the `tax_codes` assigned to the line items.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `taxable_month` | Text | Required | MM/YYYY format. |
| `supplier_tin` | Text | Required | Formatted without dashes. |
| `branch_code` | Text | Required | e.g., "00000". |
| `registered_name`| Text | Required | Cleaned string. |
| `address` | Text | Required | Concatenated address. |
| `gross_purchases`| Numeric | Required | Total amount. |
| `exempt_purchases`| Numeric | Required | Total exempt. |
| `zero_rated` | Numeric | Required | Total zero-rated. |
| `taxable_goods` | Numeric | Required | Base for goods. |
| `taxable_services`| Numeric | Required | Base for services. |
| `capital_goods` | Numeric | Required | Base for capital goods. |
| `input_tax` | Numeric | Required | Total VAT. |
| `created_at` | Timestamp | Required | Maximum timestamp of underlying documents. |

*Note to Developer:* The export functionality on the frontend will need to concatenate these columns into the exact comma-separated text string required by the BIR (e.g., `P,123456789,000,SUPPLIER NAME,...`).

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
