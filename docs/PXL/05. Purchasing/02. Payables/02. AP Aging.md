# PXL ERP Blueprint: AP Aging

## Module Overview
The Accounts Payable (AP) Aging report is a critical financial management tool used to monitor outstanding obligations to suppliers. In the Philippines, managing cash flow requires strict visibility over which Vendor Bills are due, past due, or upcoming. This report groups unpaid balances into aging "buckets" (e.g., Current, 1-30 Days, 31-60 Days) based on the invoice due date. It allows management to prioritize check preparations and negotiate payment extensions when necessary.

## What You Will See on the Screen (The Dashboard)
When you open the "AP Aging" page, you will see a comprehensive reporting matrix.

### The Action Bar (Top of the page):
* **As Of Date:** A date picker to view the aging report as of a specific historical date (defaults to today).
* **Supplier Group Filter:** Filter by specific supplier categories (e.g., Trade vs. Non-Trade payables).
* **Summary vs. Detailed Toggle:** Switch between a summary view (one row per supplier) and a detailed view (lists individual bills under each supplier).
* **Export Button:** Downloads the report to Excel or CSV for management presentations.
* **Print Button:** Generates a clean PDF version of the aging report.

### The Data List (The main report table - Summary View):
* Supplier Name
* Total Outstanding Balance
* Current (Not yet due)
* 1 - 30 Days Past Due
* 31 - 60 Days Past Due
* 61 - 90 Days Past Due
* > 90 Days Past Due

## What Information We Need to Capture (The Data Fields)
As a reporting module, this screen requires no data entry. It actively calculates days past due based on open Vendor Bills and unapplied Vendor Credits.

### Report Columns Map
| Column Name | Source / Calculation | Description |
| :--- | :--- | :--- |
| Supplier | `suppliers.registered_name`| The name of the vendor. |
| Total Outstanding| `vendor_bills.balance_due` | Sum of all unpaid balances for the supplier. |
| Current | Computed | Balance where `due_date` > "As Of Date". |
| 1 - 30 Days | Computed | Balance where `due_date` is 1 to 30 days before "As Of Date". |
| 31 - 60 Days | Computed | Balance where `due_date` is 31 to 60 days before "As Of Date". |
| 61 - 90 Days | Computed | Balance where `due_date` is 61 to 90 days before "As Of Date". |
| > 90 Days | Computed | Balance where `due_date` is more than 90 days before "As Of Date". |

## The Supabase Database Architecture (For Lovable)
This module does not require a standalone table. We will create a robust **SQL View** that aggregates the necessary data from the `vendor_bills` table, calculating the aging buckets dynamically.

### View: `vw_ap_aging_summary`
This view calculates the aging buckets for all open bills relative to the current date. (For "As Of" historical reporting, a parameterized RPC/function is recommended instead).

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `supplier_id` | UUID | Foreign Key | Links to `suppliers.id`. |
| `supplier_name` | Text | Required | The vendor's name. |
| `total_balance` | Numeric | Computed | Sum of `balance_due`. |
| `current_amount` | Numeric | Computed | Sum of balances not yet due. |
| `aging_1_30` | Numeric | Computed | Sum of balances 1-30 days past due. |
| `aging_31_60` | Numeric | Computed | Sum of balances 31-60 days past due. |
| `aging_61_90` | Numeric | Computed | Sum of balances 61-90 days past due. |
| `aging_over_90` | Numeric | Computed | Sum of balances >90 days past due. |

*Note to Developer:* Ensure the calculation considers the `status` of the bill (`open` or `partially_paid`) and strictly references the `balance_due` column rather than the `total_amount`.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.
