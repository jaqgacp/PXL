# PXL ERP Blueprint: Vendor Bills

## Module Overview
The Vendor Bills module represents the official recognition of Accounts Payable (AP). In the Philippines, this is triggered when the supplier provides a Sales Invoice or a Statement of Account (for services). This module is critical for tax compliance because it establishes the Input VAT claim and identifies any Expanded Withholding Tax (EWT) to be deducted. It forms the final part of the 3-way match (PO -> RR -> Vendor Bill) before payment is processed, ensuring the company only pays for properly received and invoiced items.

## What You Will See on the Screen (The Dashboard)
When you open the "Vendor Bills" page, you will see a clean workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Quickly find a Bill by Bill Number, Supplier Name, or Supplier Invoice No.
* **Filter:** Dropdowns to filter by Date Range, Status (Draft, Open, Partially Paid, Paid, Cancelled).
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the list to Excel or CSV.
* **Print Button:** Generates a print-ready view of the Bill list.
* **Create New Bill Button:** Prominent button to record a new supplier invoice.
* **Import Button:** Bulk upload Bills from an Excel/CSV file.

### The Data List (The main table):
* Bill Date
* Bill Number
* Supplier Name
* Supplier Invoice No.
* Due Date
* Total Amount
* Balance Due
* Status (Draft, Open, Partially Paid, Paid, Cancelled)

## What Information We Need to Capture (The Data Fields)
When creating or editing a Vendor Bill, the form is divided into Header and Line Items.

**Data Inheritance:** A Vendor Bill is usually created by selecting one or more unbilled Receiving Reports (RRs) or directly from a Purchase Order (for services without physical receipt). The Supplier, Items, Quantities, and Prices are auto-filled from the source documents. The user adds the tax details (VAT, EWT) based on the physical invoice.

| :--- | :--- | :--- | :--- | :--- |

## The Supabase Database Architecture (For Lovable)

**Critical Database Rule for Lovable:**
- Ensure a composite unique constraint on `company_id` and `bill_number` if a `company_id` is used, otherwise `bill_number` must be unique.

**Critical UX/GL Rule:** The system MUST auto-fetch the Item's default Revenue, Expense, COGS, and Inventory GL accounts in the background to auto-generate the Journal Entry. Encoders MUST NEVER manually select a GL account for standard items.

### Table 1: `vendor_bills` (Header Table)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the Bill. |
| `bill_number` | Text | Required, Unique | Auto-generated number (e.g., "VB-2023-0001"). |
| `bill_date` | Date | Required | Invoice date. |
| `supplier_id` | UUID | Required, Foreign Key | Links to `suppliers.id`. |
| `supplier_invoice_no`| Text | Required | Supplier's physical invoice number. |
| `due_date` | Date | Required | Payment deadline. |
| `ap_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `status` | Text | Required | `draft`, `open`, `partially_paid`, `paid`, `cancelled`. |
| `total_amount` | Numeric | Required | Total bill amount including VAT. |
| `total_ewt` | Numeric | Required | Total withholding tax to be deducted. |
| `balance_due` | Numeric | Required | Unpaid amount. |
| `remarks` | Text | Nullable | Extra details. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Record creation timestamp. |
| `updated_at` | Timestamp | Auto | Record update timestamp. |

### Table 2: `vendor_bill_lines` (Line Items Table)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the line item. |
| `bill_id` | UUID | Required, Foreign Key | Links to `vendor_bills.id`. |
| `rr_line_id` | UUID | Nullable, Foreign Key | Links to `receiving_report_lines.id` for 3-way match. |
| `item_id` | UUID | Required, Foreign Key | Links to `items.id`. |
| `description` | Text | Nullable | Line detail. |
| `quantity` | Numeric | Required | Billed quantity. |
| `unit_price` | Numeric | Required | Cost per unit. |
| `gross_amount` | Numeric | Required | Qty * Price. |
| `vat_code_id` | UUID | Required, Foreign Key | Links to `tax_codes.id` (Input VAT). |
| `ewt_code_id` | UUID | Nullable, Foreign Key | Links to `tax_codes.id` (Withholding Tax). |
| `net_amount` | Numeric | Required | Final calculated line amount. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Record creation timestamp. |
| `updated_at` | Timestamp | Auto | Record update timestamp. |
