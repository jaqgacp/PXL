# PXL ERP Blueprint: Cash Count Sheet

## Module Overview
The Cash Count Sheet standardizes the documentation of physical cash counts for Petty Cash Funds or daily cash register tills. Under strict Philippine internal audit guidelines, regular or surprise cash counts verify that the physical cash on hand plus unreplenished vouchers exactly equal the imprest fund balance, identifying shortages or overages immediately.

## Dashboard UI
When you open the "Cash Count Sheet" page, you will see an organized log of all cash counts.

### The Action Bar
* **Search Bar:** Find a count sheet by Reference Number or Custodian.
* **Filter:** Filter by Date Range or Fund Name.
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the list to Excel or CSV.
* **Print Button:** Generates a physical sign-off sheet for auditors.
* **Create New Count Button:** Opens the denomination counting form.
* **Import Button:** Not applicable for physical counts.

### The Data List
The main table displays historical cash counts. Default columns:
* Reference Number
* Date
* Fund / Register
* Counter Name
* Total Counted
* Variance
* Status

## Data Fields

### Section 1: Cash Count Header
| Field Name | Description | Required? | UI Component | Data Inheritance |
| :--- | :--- | :--- | :--- | :--- |
| Reference Number | Unique count identifier. | Yes | Text Input (Auto-generated) | None |
| Count Date | Date and time of the count. | Yes | Date/Time Picker | Defaults to current |
| Petty Cash Fund | The fund being audited. | Yes | Dropdown linked to Petty Cash Funds | Fetched from PCF Setup |
| Counter Name | Auditor or employee counting. | Yes | Dropdown linked to Employees | Fetched from Employee Masterfile |
| Expected Balance | The imprest balance minus pending PCVs. | Yes | Read-only Computed Field | Auto-calculated from System |
| Total Counted | Sum of all physical denominations. | Yes | Read-only Computed Field | Sum of Line Items |
| Variance | Difference between expected and actual. | Yes | Read-only Computed Field | Expected - Total Counted |
| Remarks | Explanations for variance. | Optional | Text Area | None |

### Section 2: Denominations (Line Items)
| Field Name | Description | Required? | UI Component | Data Inheritance |
| :--- | :--- | :--- | :--- | :--- |
| Denomination | Bill or coin value (e.g., 1000, 500, 1). | Yes | Dropdown linked to Pre-set PHP values | Pre-configured |
| Quantity | Number of pieces counted. | Yes | Numeric Input | None |
| Total Amount | Denomination multiplied by Quantity. | Yes | Read-only Computed Field | Denomination * Quantity |

## Supabase Database Architecture

**Critical Database Rule:**
- `cash_counts` must have a composite unique constraint on `company_id` and `reference_number`.

### Table 1: `cash_counts` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the cash count. |
| `company_id` | UUID | Required, Foreign Key | Links to companies table. |
| `pcf_id` | UUID | Required, Foreign Key | Links to `petty_cash_funds.id`. |
| `reference_number` | Text | Required | Official count number. |
| `count_date` | Timestamp | Required | Exact date and time. |
| `counted_by_id` | UUID | Required, Foreign Key | Links to `users.id` (Auditor). |
| `expected_balance` | Numeric | Required | System balance at time of count. |
| `total_counted` | Numeric | Required | Sum of physical cash. |
| `variance_amount` | Numeric | Required | Expected minus Actual. |
| `remarks` | Text | Nullable | Explanations for overage/shortage. |
| `status` | Text | Required | `draft`, `completed`, `reviewed`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `cash_count_lines` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the line item. |
| `cash_count_id` | UUID | Required, Foreign Key | Links to `cash_counts.id`. |
| `denomination` | Numeric | Required | PHP bill or coin value. |
| `quantity` | Integer | Required | Number of pieces. |
| `total_amount` | Numeric | Required | Denomination x Quantity. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
