# PXL ERP Blueprint: Petty Cash Fund Setup

## Module Overview
The Petty Cash Fund (PCF) Setup module is designed to track the establishment, custody, and parameters of petty cash funds across different branches or departments. In Philippine accounting, maintaining an imprest petty cash system is crucial for internal controls over small daily expenditures. This module ensures that every fund is assigned to a specific custodian, linked to the correct General Ledger (GL) account, and monitored for authorized fund limits.

## Dashboard UI
When you open the "Petty Cash Fund Setup" page, you will see a clean and organized workspace.

### The Action Bar
* **Search Bar:** Allows you to easily find a fund by Name or Custodian.
* **Filter:** A dropdown to show "Active" or "Inactive" funds.
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the current list to Excel or CSV.
* **Create New Fund Button:** Opens the form to register a new Petty Cash Fund.
* **Import Button:** Bulk upload PCF records using a standardized CSV template.

### The Data List
The main table displays all registered petty cash funds. Default columns:
* Fund Code
* Fund Name
* Branch / Department
* Custodian Name
* Fund Amount (PHP)
* GL Account
* Status (Active/Inactive)

## Data Fields

### Section 1: Fund Details
| Field Name | Description | Required? | UI Component | Data Inheritance |
| :--- | :--- | :--- | :--- | :--- |
| Fund Code | Unique identifier for the fund. | Yes | Text Input (Auto-generated option) | None |
| Fund Name | Descriptive name (e.g., "Main Office PCF"). | Yes | Text Input | None |
| Branch / Department | The location or department holding the fund. | Yes | Dropdown linked to Branches | Fetched from Organization Setup |
| Custodian | The employee responsible for the fund. | Yes | Dropdown linked to Employees | Fetched from Employee Masterfile |
| Fund Amount | The approved fixed imprest amount. | Yes | Numeric Input | None |
| GL Account | The specific asset account in the chart of accounts. | Yes | Dropdown linked to Chart of Accounts | Fetched from Chart of Accounts |
| Remarks | Any additional notes. | Optional | Text Area | None |

## Supabase Database Architecture

**Critical Database Rule:**
- `petty_cash_funds` must have a composite unique constraint on `company_id` and `fund_code`.

### Table: `petty_cash_funds`
This table stores the master records of all petty cash funds.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the PCF. |
| `company_id` | UUID | Required, Foreign Key | Links to companies table. |
| `fund_code` | Text | Required | System-generated or manual unique code. |
| `fund_name` | Text | Required | Descriptive name of the fund. |
| `branch_id` | UUID | Required, Foreign Key | Links to branches/departments table. |
| `custodian_id` | UUID | Required, Foreign Key | Links to employees/users table. |
| `fund_amount` | Numeric | Required | The fixed imprest balance. |
| `gl_account_id` | UUID | Required, Foreign Key | Links to chart of accounts. |
| `remarks` | Text | Nullable | Internal notes. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | User who created the record. |
| `updated_by` | UUID | Nullable, Foreign Key | User who last updated the record. |
| `created_at` | Timestamp | Auto | Record creation timestamp. |
| `updated_at` | Timestamp | Auto | Record update timestamp. |
