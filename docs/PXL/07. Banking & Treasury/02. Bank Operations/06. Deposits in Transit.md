# PXL ERP Blueprint: Deposits in Transit

## Module Overview
The Deposits in Transit (DIT) module is a vital treasury tracking dashboard that interfaces directly with the Accounts Receivable (AR) ledger. It tracks collections (cash, checks, or online transfers) that have been recorded as received by the company in the AR module, but have not yet reflected on the official bank statement. This ensures proper monitoring of undeposited funds and delays in clearing.

## Dashboard UI
When you open the "Deposits in Transit" page, you will see a real-time list of floating deposits.

### The Action Bar
* **Search Bar:** Find a deposit by OR Number, Customer Name, or Check Number.
* **Filter:** Filter by Bank Account, Date Received, or Collection Type.
* **Customize View:** Choose which columns to show or hide.
* **Export Button:** Downloads the list to Excel or CSV for audit schedules.

### The Data List
The main table displays uncleared collections. Default columns:
* OR / Reference Number
* Date Received
* Customer
* Expected Bank Account
* Collection Type (Cash/Check/Transfer)
* Amount
* Days Floating

## Data Fields

Like Outstanding Checks, this is a dynamic reporting module driven by the AR Ledger and Bank Reconciliation.

### Section 1: Deposit Details (Read-Only Interface)
| Field Name | Description | Required? | UI Component | Data Inheritance |
| :--- | :--- | :--- | :--- | :--- |
| OR / Reference No. | Official Receipt or tracking ID. | Yes | Read-only Text | Fetched from AR Collections |
| Receive Date | Date the payment was logged. | Yes | Read-only Text | Fetched from AR Collections |
| Customer Name | Client who made the payment. | Yes | Read-only Text | Fetched from AR Collections / Masterfile |
| Expected Bank | Where the funds were deposited. | Yes | Read-only Text | Fetched from AR Collections |
| Collection Type | Cash, Check, or Online. | Yes | Read-only Text | Fetched from AR Collections |
| Amount | Deposited value. | Yes | Read-only Numeric | Fetched from AR Collections |
| Days Floating | Number of days since receipt. | Yes | Read-only Computed Field | Current Date - Receive Date |

## Supabase Database Architecture

### Table: `vw_deposits_in_transit` (Database View)
This module uses a **SQL View** to query the existing AR Collections ledger and identifies all receipts that have not yet been marked as cleared via the Bank Reconciliation module.

| Column Name | Data Type | Rules | What it stores (Source) |
| :--- | :--- | :--- | :--- |
| `ar_collection_id` | UUID | Primary Key | `ar_collections.id` |
| `reference_number` | Text | Required | `ar_collections.or_number` |
| `receive_date` | Date | Required | `ar_collections.collection_date` |
| `customer_name` | Text | Required | Fetched via `ar_collections.customer_id` |
| `bank_account_id` | UUID | Required | `ar_collections.deposited_to_bank_id` |
| `collection_type` | Text | Required | `ar_collections.payment_method` |
| `amount` | Numeric | Required | `ar_collections.amount` |
| `days_floating` | Integer | Computed | `CURRENT_DATE - ar_collections.collection_date` |
| `created_by` | UUID | Required | `ar_collections.created_by` |
| `updated_by` | UUID | Nullable | `ar_collections.updated_by` |
| `created_at` | Timestamp | Required | `ar_collections.created_at` |
| `updated_at` | Timestamp | Required | `ar_collections.updated_at` |

*Note: The view strictly includes records where `ar_collections.is_cleared = false` (or not present as `is_cleared = true` in `bank_reconciliation_lines`). This guarantees strict interface compliance with the AR ledger and Treasury processes.*
