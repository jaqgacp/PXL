# PXL Transaction Matrix

Last synced: 2026-07-02

Purpose: this is the living source of truth for PXL transactions. Any code change that adds, removes, modifies, fixes, posts, reverses, voids, reports, or validates a transaction must update this file in the same session.

Status key:

- `Implemented`: code exists and transaction is usable, subject to audit findings.
- `Partial`: code exists but accounting, tax, validation, traceability, or UX is incomplete.
- `Planned`: future module or placeholder.

Audit rule: if a transaction cannot drill from source document -> journal entry -> GL -> trial balance -> financial statements -> compliance report, record the gap in `PXL_END_TO_END_AUDIT_FINDINGS.md`.
