## Module Overview
The Accounts Receivable (AR) Aging module is a vital reporting tool that categorizes unpaid customer invoices based on the length of time they have been outstanding. In Philippine business practices, this report is essential for cash flow forecasting, prioritizing collection efforts, and calculating provisions for bad debts (ECL - Expected Credit Losses) required by PFRS/PFRS for SMEs.

*Note: This is a reporting module. It does not process new transactions but aggregates data from posted Sales Invoices and Receipts.*

## Dashboard UI
A high-level summary view that allows users to drill down into specific aging buckets.

### The Action Bar
* **As Of Date:** A date picker to run the aging report as of a specific point in time (defaults to today).
* **Customer Filter:** Optional dropdown to view aging for a specific customer.
* **Export Button:** Download the aging report to Excel/CSV for management meetings.
* **Print Button:** Generate a PDF summary.

### The Data List
The grid displays summarized balances per customer, broken down into standard aging buckets.

Default columns displayed:
* Customer Name
* Current (Not yet due)
* 1 to 30 Days Past Due
* 31 to 60 Days Past Due
* 61 to 90 Days Past Due
* Over 90 Days Past Due
* Total Outstanding Balance

*Clicking on a customer row expands to show the specific Sales Invoices making up those balances.*

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

The AR Aging report relies on complex date math (Current Date vs `due_date`). It is best implemented as a PostgreSQL View or RPC function that queries the `sales_invoices` table and subtracts applied `receipts`.

### View Definition: `vw_ar_aging_summary`
This view calculates the outstanding balance per invoice and assigns it to a bucket.

| Column Name | Data Type | Source / Logic | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `customer_id` | UUID | `sales_invoices.customer_id` | Grouping key. |
| `customer_name` | Text | `customers.registered_name` | Display name. |
| `current_bal` | Numeric | Sum of Net where `due_date >= As Of Date` | Not yet due. |
| `days_1_30` | Numeric | Sum of Net where days past due is 1-30 | Overdue bucket 1. |
| `days_31_60`| Numeric | Sum of Net where days past due is 31-60 | Overdue bucket 2. |
| `days_61_90`| Numeric | Sum of Net where days past due is 61-90 | Overdue bucket 3. |
| `over_90` | Numeric | Sum of Net where days past due > 90 | Overdue bucket 4. |
| `total_ar` | Numeric | Sum of all buckets | Total outstanding. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

*Note: The frontend will query this view, passing the `As Of Date` as a parameter if using an RPC, or the view will default to `CURRENT_DATE`.*
