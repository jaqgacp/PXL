## Module Overview
The Delivery Receipt (DR) is vital in Philippine logistics and inventory management. It serves as legal proof that physical goods were received by the customer. While it triggers an inventory deduction (Cost of Goods Sold recognition depending on setup), it does *not* recognize VAT Output or Sales Revenue. Under BIR regulations, the Sales Invoice is the primary document for revenue. The DR is an operational document bridging the warehouse and the customer.

### Document Flow & Data Inheritance
* **Source:** Most commonly generated from an approved **Sales Order**.
* **Inheritance:** When inherited from a Sales Order, the DR auto-fills the `customer_id` and the `expected_delivery_date`. The line items are populated based on the unfulfilled quantities from the `sales_order_lines`. A single Sales Order can spawn multiple Delivery Receipts (partial deliveries).

## Dashboard UI
A monitoring space for warehouse and logistics teams to track outgoing shipments.

### The Action Bar
* **Search Bar:** Search by DR Number, SO Number, or Customer Name.
* **Filter:** Dropdown for Status (Draft, In Transit, Delivered, Cancelled).
* **Customize View:** Adjust columns.
* **Export Button:** Export list to Excel/CSV.
* **Print Button:** Generate physical DR copies to accompany the driver.
* **Create DR Button:** Create a DR from scratch or by pulling an SO.
* **Import Button:** Bulk import DR details from 3PL (Third Party Logistics) systems.

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `delivery_receipts` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the DR. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `sales_order_id` | UUID | Nullable, Foreign Key | Links to `sales_orders.id` (Inheritance). |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `dr_number` | Text | Required, Unique | Formatted document number (e.g., DR-2023-0001). |
| `dr_date` | Date | Required | Date of actual delivery. |
| `shipping_method` | Text | Required | Internal, Courier, Pickup. |
| `tracking_number` | Text | Nullable | 3PL Waybill or tracking. |
| `driver_name` | Text | Nullable | Name of the delivery personnel. |
| `delivery_address`| Text | Required | Target address. |
| `status` | Text | Required | `draft`, `in_transit`, `delivered`, `cancelled`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
