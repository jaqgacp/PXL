## Module Overview
The Debit Memo (DM) module is used to increase the amount owed by a customer outside of the standard Sales Invoice process. Common scenarios in the Philippines include charging back bank fees for bounced checks, late payment penalties, or correcting underbilled invoices. It increases Accounts Receivable.

### Document Flow & Data Inheritance
* **Source:** Can be standalone or tied to a **Sales Invoice** (for underbilling) or a **Receipt** (for a bounced check).
* **Inheritance:** When linked, it inherits the customer details and original document references to ensure a clear audit trail.

## Dashboard UI
A tracking list for all supplementary charges issued to customers.

### The Action Bar
* **Search Bar:** Search by DM Number or Customer.
* **Filter:** Dropdown for Status (Draft, Approved, Paid).
* **Customize View:** Adjust columns.
* **Export Button:** Export list to Excel/CSV.
* **Print Button:** Print DM for customer notification.
* **Create Debit Memo Button:** Initiate a new charge.
* **Import Button:** Bulk create DMs.

### The Data List
Default columns displayed:
* DM Date
* DM Number
* Customer Name
* Reason
* Total Amount
* Status

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `debit_memos` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the DM. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `source_doc_type`| Text | Nullable | `invoice` or `receipt`. |
| `source_doc_id` | UUID | Nullable | Polymorphic link to source doc. |
| `dm_number` | Text | Required, Unique | Formatted document number. |
| `dm_date` | Date | Required | Date of the memo. |
| `reason_code_id` | UUID | Required, Foreign Key | Links to `ref_reason_codes.id`. |
| `remarks` | Text | Nullable | Internal notes. |
| `total_amount` | Numeric | Required | Total amount charged. |
| `vat_amount` | Numeric | Required | Output VAT (if applicable). |
| `status` | Text | Required | `draft`, `approved`, `paid`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
