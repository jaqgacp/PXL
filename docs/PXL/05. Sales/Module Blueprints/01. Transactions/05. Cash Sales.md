## Module Overview
The Cash Sales module is designed for over-the-counter transactions where goods are delivered and payment is collected simultaneously. In Philippine accounting, this document inherently combines the functions of a Delivery Receipt, a Sales Invoice, and an Official Receipt. It immediately recognizes Revenue, Output VAT, Cash (or equivalent), and reduces Inventory without passing through Accounts Receivable.

### Document Flow & Data Inheritance
* **Source:** Typically standalone, no predecessor document.
* **Inheritance:** Does not inherit from Quotations or Orders as it is an immediate transaction.

## Dashboard UI
A fast, point-of-sale style view for managing daily cash transactions.

### The Action Bar
* **Search Bar:** Search by Cash Sale Number or Customer Name.
* **Filter:** Dropdown for Date Range or Payment Mode (Cash, Card, GCash).
* **Customize View:** Hide or show columns.
* **Export Button:** Export day's sales to Excel.
* **Print Button:** Reprint a Cash Sales receipt.
* **New Cash Sale Button:** Open the POS-style entry form.

### The Data List
Default columns displayed:
* Date
* Cash Sale No.
* Customer (Often "Walk-in")
* Payment Mode
* Total Amount
* VAT Amount
* Created By

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `cash_sales` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the cash sale. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `customer_id` | UUID | Nullable, Foreign Key | Links to `customers.id` if registered. |
| `customer_name` | Text | Required | Actual name or "Walk-in". |
| `cs_number` | Text | Required, Unique | Formatted document number (e.g., CS-2023-0001). |
| `cs_date` | Timestamp | Required | Date and time of sale. |
| `payment_mode_id` | UUID | Required, Foreign Key | Links to `ref_payment_modes.id`. |
| `payment_reference`| Text | Nullable | Card/Check/GCash Ref. |
| `gross_amount` | Numeric | Required | Total before VAT/Discounts. |
| `total_discount` | Numeric | Default 0 | Total discounts. |
| `vat_amount` | Numeric | Required | Output VAT. |
| `net_amount` | Numeric | Required | Final amount collected. |
| `posting_status` | Text | Required | `posted`, `voided`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
