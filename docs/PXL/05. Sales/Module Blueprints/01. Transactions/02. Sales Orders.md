## Module Overview
The Sales Order (SO) module serves as the formal commitment between the business and the customer to deliver goods or perform services at agreed prices and conditions. In the Philippine context, while the SO does not recognize revenue or VAT output, it is a crucial internal control document. It allocates inventory, triggers procurement if out of stock, and authorizes the warehouse to prepare the Delivery Receipt. 

### Document Flow & Data Inheritance
* **Source:** Can be created manually or generated from an approved **Quotation**.
* **Inheritance:** When inherited from a Quotation, the `customer_id`, `currency_id`, and all line items (`item_id`, `quantity`, `unit_price`) are automatically populated. The system maps the `quotation_id` to the SO header to establish a traceable document flow.

## Dashboard UI
A centralized view for tracking all confirmed orders and their fulfillment status.

### The Action Bar
* **Search Bar:** Search by SO Number, Customer, or Reference Number.
* **Filter:** Dropdown for Fulfillment Status (Open, Partially Fulfilled, Fully Fulfilled, Cancelled).
* **Customize View:** Select which columns are visible.
* **Export Button:** Download list to Excel/CSV.
* **Print Button:** Generate a Sales Order summary.
* **Create Sales Order Button:** Draft a new SO.
* **Import Button:** Bulk upload SOs from an external platform (e.g., e-commerce integration).

### The Data List
Default columns displayed:
* SO Date
* SO Number
* Customer Name
* Expected Delivery Date
* Total Amount
* Fulfillment Status
* Approval Status

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

### Table 1: `sales_orders` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the sales order. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `quotation_id` | UUID | Nullable, Foreign Key | Links to `sales_quotations.id` (Inheritance). |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `so_number` | Text | Required, Unique | Formatted document number (e.g., SO-2023-0001). |
| `so_date` | Date | Required | Date the order was confirmed. |
| `expected_delivery_date` | Date | Required | Target date for fulfillment. |
| `currency_id` | UUID | Required, Foreign Key | Links to `ref_currencies.id`. |
| `reference_number` | Text | Nullable | Customer PO number. |
| `remarks` | Text | Nullable | Internal notes. |
| `total_amount` | Numeric | Required | Total value of the order. |
| `approval_status`| Text | Required | `pending`, `approved`, `rejected`. |
| `fulfillment_status`| Text | Required | `open`, `partial`, `fulfilled`, `cancelled`. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
