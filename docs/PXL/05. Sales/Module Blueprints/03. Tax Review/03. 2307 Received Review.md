## Module Overview
The 2307 Received Review module tracks the physical or digital receipt of BIR Form 2307 (Certificate of Creditable Tax Withheld at Source) from customers. When recording Receipts (collections), users often deduct CWT. However, that CWT cannot be legally claimed as a deduction against the company's Income Tax Payable until the actual signed 2307 form is provided by the customer. This module bridges the gap between accounting entries and tax compliance documentation.

### Document Flow & Data Inheritance
* **Source:** Links to posted **Receipts** where a `cwt_amount` > 0 was recorded.
* **Inheritance:** Lists all collections with withheld taxes and tracks whether the corresponding form has been attached/collected.

## Dashboard UI
A tracking list for the accounting/tax team to hunt down missing tax certificates from clients.

### The Action Bar
* **Search Bar:** Search by Customer Name or Receipt Number.
* **Filter:** Dropdown for Status (Pending 2307, Received, Claimed).
* **Date Filter:** Filter by Quarter/Year.
* **Export Button:** Export list of missing 2307s to send to the collections team.
* **Upload 2307 Button:** Bulk upload scanned forms.

### The Data List
Default columns displayed:
* Receipt Date
* Receipt Number
* Customer Name
* CWT Amount Withheld
* Expected ATC Code (Alphanumeric Tax Code)
* 2307 Status (Red for Missing, Green for Received)
* Date Received

## Data Fields (Form Tracking Entry)
When a user clicks "Mark as Received" on a pending item, a modal appears.

| Field Name | Component Type | Required? | Behavior |
| :--- | :--- | :--- | :--- |
| Date Received | Date Picker | Yes | When the form was acquired. |
| ATC Code | Dropdown | Yes | E.g., WC158 (Goods), WC160 (Services). |
| Period Covered | Text/Dropdown | Yes | Quarter the customer reported it. |
| Upload File | File Picker | Optional | PDF scan of the signed 2307. |
| Remarks | Text Area | Optional | Any discrepancies in amount. |

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

We track the status of these forms directly linked to the receipt lines where the CWT was recognized.

### Table 1: `form_2307_tracking`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for tracking record. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `receipt_line_id`| UUID | Required, Unique, FK | Links to `receipt_lines.id` where CWT was booked. |
| `customer_id` | UUID | Required, Foreign Key | Links to `customers.id`. |
| `cwt_amount_booked`| Numeric| Required | Amount deducted in the system. |
| `status` | Text | Required | `pending`, `received`, `claimed`. |
| `date_received` | Date | Nullable | When the form was acquired. |
| `atc_code_id` | UUID | Nullable, Foreign Key | Links to `ref_atc_codes.id`. |
| `period_covered` | Text | Nullable | E.g., "Q3-2023". |
| `file_url` | Text | Nullable | Storage bucket link for the PDF. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
