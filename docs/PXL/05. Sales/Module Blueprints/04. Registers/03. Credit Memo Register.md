## Module Overview
The Credit Memo Register is a chronological listing of all approved Credit Memos. In traditional Philippine bookkeeping, this report acts as the **Sales Returns and Allowances Journal**. It provides an audit trail for any reductions made to a customer's Accounts Receivable that did not involve a cash payment, as well as the corresponding reversal of Output VAT.

*Note: This is a read-only reporting module pulling data from the `credit_memos` table.*

## Dashboard UI
A structured ledger view used for month-end financial reviews.

### The Action Bar
* **Date Range Filter:** "From" and "To" dates.
* **Reason Code Filter:** Dropdown to isolate specific types of credits (e.g., only "Sales Returns" vs "Overbillings").
* **Export Button:** Export to Excel/CSV.
* **Print Button:** Generate physical reports for auditing.

### The Data List
Default columns displayed chronologically:
* Date
* CM Number
* Source Invoice (if applicable)
* Customer Name
* Reason Code
* Net Amount Reversed (Debit Sales/Allowances)
* Output VAT Reversed (Debit Output VAT)
* Total CM Amount (Credit AR)

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

This view aggregates data from the `credit_memos` header and lines.

### View Definition: `vw_credit_memo_register`
| Column Name | Data Type | Source | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `cm_date` | Date | `credit_memos.cm_date` | Chronological sorting. |
| `cm_number` | Text | `credit_memos.cm_number` | Document sequence. |
| `invoice_number` | Text | `sales_invoices.si_number` | The invoice being adjusted. |
| `customer_name` | Text | `customers.registered_name` | Account being credited. |
| `reason_name` | Text | `ref_reason_codes.name` | Context of the adjustment. |
| `net_reversal` | Numeric | `credit_memos.total_amount - vat` | Reduction in sales revenue. |
| `vat_reversal` | Numeric | `credit_memos.vat_amount` | Reduction in output VAT. |
| `total_amount` | Numeric | `credit_memos.total_amount` | Total reduction in AR. |
| `status` | Text | `credit_memos.status` | E.g., Approved, Applied. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
