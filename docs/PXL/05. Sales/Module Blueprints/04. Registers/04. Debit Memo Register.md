## Module Overview
The Debit Memo Register is a chronological listing of all approved Debit Memos. It serves as an audit trail for any upward adjustments to a customer's Accounts Receivable outside of the normal Sales Invoice process (e.g., charging bank penalties for bounced checks, late payment fees, or correcting underbillings).

*Note: This is an aggregated reporting module pulling from the `debit_memos` table.*

## Dashboard UI
A read-only tabular view to track supplementary charges added to customer accounts.

### The Action Bar
* **Date Range Filter:** "From" and "To" dates.
* **Reason Code Filter:** Dropdown to isolate specific charge types (e.g., "Bounced Check Fees").
* **Export Button:** Export to Excel/CSV.
* **Print Button:** Generate physical reports for auditing.

### The Data List
Default columns displayed chronologically:
* Date
* DM Number
* Source Document (e.g., Reversing OR Number for bounced checks)
* Customer Name
* Reason Code
* Net Amount Charged
* VAT Amount (if applicable)
* Total DM Amount (Debit AR)

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

This view aggregates data from the `debit_memos` header and lines.

### View Definition: `vw_debit_memo_register`
| Column Name | Data Type | Source | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `dm_date` | Date | `debit_memos.dm_date` | Chronological sorting. |
| `dm_number` | Text | `debit_memos.dm_number` | Document sequence. |
| `source_ref` | Text | Fetched via polymorphic ID | Context of charge (e.g. OR/SI num).|
| `customer_name` | Text | `customers.registered_name` | Account being charged. |
| `reason_name` | Text | `ref_reason_codes.name` | Justification of charge. |
| `net_charge` | Numeric | `debit_memos.total_amount - vat` | Increase in specific revenue/expense offset. |
| `vat_charge` | Numeric | `debit_memos.vat_amount` | Added output VAT if vatable. |
| `total_amount` | Numeric | `debit_memos.total_amount` | Total increase in AR. |
| `status` | Text | `debit_memos.status` | E.g., Approved, Paid. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
