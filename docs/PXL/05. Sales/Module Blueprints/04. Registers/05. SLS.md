## Module Overview
The Summary List of Sales (SLS) is a strict, highly regulated compliance report mandated by the Bureau of Internal Revenue (BIR) for VAT-registered taxpayers. It must be generated monthly/quarterly and submitted electronically via the BIR Relief system as a specially formatted `.DAT` or `.CSV` file. The SLS aggregates all sales transactions to regular buyers, detailing the TIN, Registered Name, and the breakdown of Vatable, Exempt, and Zero-Rated sales, plus the Output Tax.

*Note: This is a compliance export module. The format must exactly match the BIR Relief schema.*

## Supabase Database Architecture


**Critical Database Rule:**
- Every transactional table MUST include a `company_id` to ensure tenant isolation.
- Composite unique constraints MUST be established (e.g., `company_id` + document number) to prevent duplicate documents within the same company.

The SLS requires aggregating `sales_invoices`, `cash_sales`, and deducting `credit_memos` *per customer*, *per period*, grouping by the Buyer's TIN. 

### View Definition: `vw_bir_sls_export`
This view applies complex grouping logic to meet BIR standards. If a buyer has multiple invoices in the month, their sales must be summed into a single row per TIN.

| Column Name | Data Type | Source / Logic | What it stores |
| :--- | :--- | :--- | :--- |
| `company_id` | UUID | Required | Links to `companies.id` / for tenant filtering. |
| `taxable_month` | Text | Param based | Format: MM/YYYY |
| `buyer_tin` | Text | `customers.tin` | Must be exactly 9 or 13 digits stripped of hyphens. |
| `registered_name`| Text | `customers.registered_name` | Name of the buyer. |
| `address` | Text | `customers.address_line_1` | Registered address. |
| `gross_sales` | Numeric | Sum(Vatable + Exempt + Zero) | Total base sales. |
| `exempt_sales` | Numeric | Sum(Net where Tax is Exempt)| Exempt sales component. |
| `zero_rated_sales`| Numeric | Sum(Net where Tax is 0%) | Zero-rated sales component. |
| `taxable_sales` | Numeric | Sum(Net where Tax is 12%) | Vatable sales component. |
| `output_tax` | Numeric | Sum(VAT Amount) | Total 12% output VAT. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

*Critical Constraint:* Transactions without a valid Customer TIN are often rejected by the BIR Relief tool. The system should flag these errors in the preview before export.
