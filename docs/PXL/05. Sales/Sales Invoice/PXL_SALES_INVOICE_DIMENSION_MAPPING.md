# PXL Sales Invoice Dimension Mapping

Status: Completeness Audit v1
Last updated: 2026-07-15
Canonical matrix: `docs/PXL/04. Transaction Framework/PXL_TRANSACTION_FIELD_SOURCE_MATRIX.md`

This file summarizes dimension policy. The field-level gate and validation status for each dimension is controlled by `docs/PXL/04. Transaction Framework/PXL_TRANSACTION_FIELD_SOURCE_MATRIX.md`.

## 1. Principle

Every dimension shown in the Sales Invoice view, GL Impact, reports, audit trail, or future BIR/compliance output must be entered, inherited, or generated by documented rules.

If a dimension is not configured or not source-backed, hide it from primary business tables and show a concise empty state where appropriate.

## 2. Current Source-Backed Dimensions

| Dimension | Level | Current source | Posting support | Status |
| --- | --- | --- | --- | --- |
| Branch | Header | `sales_invoices.branch_id` | `journal_entry_lines.branch_id` | Implemented |
| Department | Header default and line override | `sales_invoices.department_id`, `sales_invoice_lines.department_id`, Department Master | `journal_entry_lines.department_id` | Implemented |
| Cost Center | Header default and line override | `sales_invoices.cost_center_id`, `sales_invoice_lines.cost_center_id`, Cost Center Master | `journal_entry_lines.cost_center_id` | Implemented |
| Warehouse | Header default and inventory-line value | `sales_invoices.warehouse_id`, `sales_invoice_lines.warehouse_id`, Warehouse Master | Inventory ledger; not a GL dimension column | Implemented for inventory lines |
| Salesperson | Header default and line override | `sales_invoices.salesperson_id`, `sales_invoice_lines.salesperson_id`, Employee Master | Reporting context only | Implemented |
| Account Owner | Header | `sales_invoices.account_owner_id`, Employee Master | Reporting context only | Implemented |

## 3. Supported Master Data Capture Rules

| Dimension | Capture behavior | Notes |
| --- | --- | --- |
| Department | Optional header selector; blank line values inherit from header; lines may override | Posted revenue and COGS/inventory journal lines receive line value or header fallback. |
| Cost Center | Optional header selector; blank line values inherit from header; lines may override | Posted revenue and COGS/inventory journal lines receive line value or header fallback. |
| Warehouse | Optional header default; inherited only by inventory item lines; service/non-inventory lines do not automatically inherit | Required before approval/posting for inventory item lines. |
| Salesperson | Optional header selector; blank line values inherit from header; lines may override | Reporting context; not a GL dimension column. |
| Account Owner | Optional header selector from Employee Master | Relationship ownership; future Customer Master inheritance may default this field. |

## 5. Recommended Level

| Dimension | Recommended level | Reason |
| --- | --- | --- |
| Branch | Header, inherited to lines | Legal/reporting branch normally applies to whole invoice |
| Location | Header default, line override | Sales or service location may vary by line |
| Department | Header default, line override | Revenue and responsibility reporting may vary by line |
| Cost Center | Header default, line override | Cost responsibility may vary by line |
| Project | Header default, line override | Project billing can be line-specific |
| Warehouse | Line required for inventory items | Inventory issue location is line-specific |
| Functional Entity | Header default, line override | Legal/functional reporting can be line-specific |
| Salesperson | Header default, optional line override | Split commissions may be line-specific |
| Account Owner | Header | Relationship ownership is customer-level |

## 6. Posting Propagation

Current journal lines support only:

- Branch
- Department
- Cost Center

Warehouse, salesperson, and account owner are stored as transaction/reporting context but are not journal-line dimensions. Project, location, and functional entity cannot be posted to the GL until governed master data and journal/reporting support exist.

## 7. Current UI Rule

The Sales Invoice form and view display only configured/source-backed dimensions. Project, Location, Functional Entity, Business Unit, Campaign, and Lead Source remain hidden until their governed master-data sources and posting/reporting behavior are implemented. Service lines must not receive an automatic warehouse value.
