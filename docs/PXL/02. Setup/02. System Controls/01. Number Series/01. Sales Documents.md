# PXL ERP Blueprint: System Controls — Number Series

## Module Overview
The Number Series module is the centralized engine for all auto-generated document numbers in PXL ERP. It manages numbering sequences across four categories: **Sales**, **Purchasing**, **Accounting**, and **Compliance**. 

**Critical Philippine Compliance Rule:** The BIR strictly requires that Sales Invoices (SI) and Official Receipts (OR) use only the series number ranges approved under the company's Authority to Print (ATP) or CAS Permit. Furthermore, each Branch must have its own independent series to prevent duplication across locations. The system enforces this by requiring a unique number series record per Document Type per Branch.

## What You Will See on the Screen (The Dashboard)
When you open the "Number Series" page, you will see a clean and unified workspace.

### The Action Bar (Top of the page):
* **Category Tabs:** Four prominent tabs at the top: **[ Sales ] [ Purchasing ] [ Accounting ] [ Compliance ]**. Clicking a tab filters the table below.
* **Search Bar:** Find a series by Prefix or Branch.
* **Company & Branch Filters:** Dropdowns to narrow down sequences by specific entities.
* **Import Button:** Allows bulk creation of number series records by uploading a pre-formatted CSV or Excel file. Useful for batch setup across multiple branches during system go-live.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for review or documentation.
* **Print Button:** Generates a print-ready view of all configured series for hardcopy reference.
* **Create New Series Button:** A prominent button to add a new number series.

### The Data List (The main table - must be customizable):
The columns dynamically adjust depending on the active tab, but generally display:
* Document Type
* Company & Branch
* Prefix
* Current / Next Number (e.g., SI-000045)
* Reset Frequency
* ATP Range / Alert (Visible only on the Sales Tab for BIR documents)
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that number series.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Series" or "Edit," a unified form will open. The fields displayed will conditionally adjust if the selected document type is BIR-registered.

### Section 1: Series Scope & Assignment
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. | Yes |
| Branch | Dropdown linked to active branches. Enforced uniqueness per branch. | Yes |
| Category | Dropdown: Sales, Purchasing, Accounting, Compliance. | Yes |

### Section 2: Number Format Configuration
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Prefix | The base text prepended to the number (e.g., "SI-", "JV-", "2307-"). | Optional |

| Number Length (Padding) | Total digits in the numeric portion (e.g., 6 produces "000001"). | Yes |
| Starting Number | The first number in the sequence. Defaults to 1. | Yes |
| Next Available Number | The next number the system will assign. Auto-increments after each confirmed transaction. | Yes |

| :--- | :--- | :--- |

## The Supabase Database Architecture (For Lovable)
This module uses a shared database structure for all document sequences across the ERP.

### Table 1: `ref_document_types` (Reference Table)
Pre-seeded lookup table of all document types in the ERP.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `category` | Text | Required | `sales`, `purchasing`, `accounting`, `compliance`. |
| `document_code` | Text | Required, Unique | Internal code (e.g., "SI", "PO", "JV", "2307"). |
| `document_name` | Text | Required | Full name (e.g., "Sales Invoice", "Purchase Order"). |
| `is_bir_registered` | Boolean | Default: `false` | True for SI and OR requiring ATP tracking. |
| `sort_order` | Integer | Required | Display order. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `number_series` (The Main Table)
**Critical Database Rule for Lovable:** Apply a composite UNIQUE constraint on (`company_id`, `branch_id`, `document_type_id`) to prevent duplicate series.

**Application Logic:** On document save, increment `next_number` atomically (using Supabase RPC/Edge Functions) to prevent duplicate numbers during concurrent transactions. If `reset_frequency` is `yearly`, reset `next_number` to `starting_number` upon the first transaction of a new year.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the number series. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `document_type_id` | UUID | Required, Foreign Key | Links to `ref_document_types.id`. |
| `prefix` | Text | Nullable | Base text prepended to the number. |
| `has_dynamic_year` | Boolean | Default: `false` | Injects current year between prefix and number. |
| `number_length` | Integer | Required, Default: 6 | Zero-padding length. |
| `starting_number` | Integer | Required, Default: 1 | The first number of the sequence. |
| `next_number` | Integer | Required, Default: 1 | The next number the system will use. |
| `reset_frequency` | Text | Required, Default: `never` | `never`, `yearly`, `monthly`. |
| `last_reset_date` | Date | Nullable | Date the counter was last reset. |
| `atp_series_start` | Integer | Nullable | BIR-approved starting serial. |
| `atp_series_end` | Integer | Nullable | BIR-approved ending serial. |
| `atp_alert_threshold` | Integer | Nullable | Alert threshold for low serials. |
| `allow_manual_override` | Boolean | Default: `false` | Allows manual number entry. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
