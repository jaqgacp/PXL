# PXL ERP Blueprint: Number Series — Accounting Documents

## Module Overview
This page manages the auto-numbering sequences for all Accounting-related documents in PXL ERP. The most critical document type in this category is the **Journal Voucher (JV)**, which is the record used for all manual and system-generated general ledger entries. A consistent and well-structured JV numbering convention is essential for audit trails, period-close procedures, and reconciliation reviews.

The Number Series for Accounting Documents covers the following transaction types:
* Journal Entries (Journal Vouchers)
* Recurring Journal Templates

> **Reference:** This module shares the same `number_series` Supabase table defined in the **Sales Documents** blueprint. Please refer to that document for the complete database schema. This page documents only the Accounting-specific configuration and business rules.

**Key Design Note for Journal Entries:** The most common Philippine accounting convention for Journal Vouchers uses a **Yearly Reset** format that embeds the year in the sequence, such as **JV-2026-000001**. This makes every JV instantly identifiable by fiscal year and is the recommended default configuration for this document type.

## What You Will See on the Screen (The Dashboard)
When you open the "Number Series — Accounting Documents" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a series by Document Type, Prefix, or Branch.
* **Company Filter:** A dropdown to filter series by parent company.
* **Branch Filter:** A dropdown to filter by branch.
* **Document Type Filter:** A dropdown to show the series for a specific document type.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of accounting number series records by uploading a pre-formatted CSV or Excel file. Useful for batch setup across multiple branches during system go-live.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for review or documentation.
* **Print Button:** Generates a print-ready view of all configured accounting series for hardcopy reference.
* **Create New Series Button:** A prominent button to add a new number series record.

### The Data List (The main table - must be customizable):
You will see a list of all configured accounting document series. The default columns displayed are:
* Document Type
* Company
* Branch
* Prefix
* Current / Next Number (e.g., JV-2026-000045)
* Reset Frequency
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that number series.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Series" or "Edit," a form will open. Here is the exact data the system will save.

### Section 1: Series Scope & Assignment
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. The entity this number series belongs to. | Yes |
| Branch | Dropdown linked to active branches. Journal Entries can be company-wide (Head Office) or branch-specific depending on the accounting structure. | Yes |
| Document Type | Dropdown: Journal Entry, Recurring Journal Template. | Yes |

### Section 2: Number Format Configuration
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Prefix | The text prepended to the number. For Journal Entries, a common convention is "JV-" or "JV-2026-" (where the year is embedded in the prefix and updated on reset). | Optional |
| Suffix | The text appended after the number. | Optional |
| Number Length (Padding) | The total number of digits in the numeric portion, zero-padded (e.g., 6 produces "000001"). | Yes |
| Starting Number | The first number in the sequence. Defaults to 1. | Yes |
| Next Available Number | The next number the system will assign. Auto-increments after each confirmed journal entry. | Yes |

### Section 3: Reset & Control Settings
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Reset Frequency | Dropdown: Never (continuous), **Reset Yearly** (recommended default for JVs — resets to Starting Number every January 1 and embeds the new year in the prefix), Reset Monthly. | Yes |
| Allow Manual Override | Toggle (Yes / No). If YES, users with sufficient permissions can manually type a journal entry number (used for encoding prior-period or imported entries). | Yes |

### Section 4: Format Preview
This is a display-only section that shows a live preview of the generated document number based on the configuration above.

| Field Name | Description |
| :--- | :--- |
| Format Preview | Auto-rendered display (e.g., **JV-2026-000001**). Updates in real-time as the user configures the fields above. |

## The Supabase Database Architecture (For Lovable)

### Shared Table: `number_series`
The Accounting Documents number series uses the same `number_series` table defined in the **Sales Documents** blueprint. No additional tables are required for this module.

**Special Application Logic for Yearly Reset:** When `reset_frequency` is `yearly`, the system must check on the first transaction of each new calendar year (or fiscal year, based on company settings) whether `last_reset_date` is in a prior year. If so, it must reset `next_number` to `starting_number` and update `last_reset_date`. If the prefix contains a year placeholder (e.g., `JV-{YYYY}-`), the system must also update the prefix string with the new year before generating the first number of the new period.

The records pre-seeded for Accounting in the `ref_document_types` table are:

| `document_code` | `document_name` | `category` | `is_bir_registered` |
| :--- | :--- | :--- | :--- |
| `JV` | Journal Entry | `accounting` | `false` |
| `RJT` | Recurring Journal Template | `accounting` | `false` |
