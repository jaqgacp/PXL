# PXL ERP Blueprint: Journal Approval Matrix

> **ARCHITECTURE NOTE:** This module does NOT have its own database table.
>
> Journal approval workflows are managed by the **Unified Approval Workflow** system defined in `01. Unified Approval Workflow.md`. Use the `approval_workflows` table with `module_type = 'journal'`.

## Module Overview
The Journal Approval Matrix governs the posting of manual Journal Vouchers (JVs). Adjusting entries, accruals, and intercompany transfers must be reviewed by a Chief Accountant or Controller before they hit the General Ledger. This prevents material misstatements and satisfies external audit requirements.

---

## Applicable Document Types

| Document Type Value | What It Controls |
| :--- | :--- |
| `journal_entry` | All manual journal entries |
| `reversing_entry` | System-generated reversing entries (may still require review) |

---

## Common Trigger Conditions for Journal Entries

| Trigger | When to Use |
| :--- | :--- |
| `always` | All manual JEs require Chief Accountant review before posting |
| `amount_exceeds` + threshold | Only large manual adjustments need sign-off |

---

## Database Implementation

```sql
SELECT * FROM approval_workflows
WHERE company_id = :company_id
  AND module_type = 'journal'
  AND is_active = true;
```

See `01. Unified Approval Workflow.md` for full schema.
