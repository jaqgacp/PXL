# PXL ERP Blueprint: Feature Settings — Fixed Assets Settings

## Module Overview
The Fixed Assets Settings page defines the global rules that govern how the Fixed Assets module classifies, depreciates, and disposes of assets across the company. These settings establish the financial and tax thresholds that determine what qualifies as a capital asset versus an outright expense, and how depreciation is computed and posted to the General Ledger.

**Critical Philippine Tax Compliance Note:** Under BIR Revenue Regulations and the National Internal Revenue Code (NIRC), expenditures must be **capitalized** (recorded as an asset and depreciated) rather than expensed outright if they meet the capitalization threshold. The current BIR-recognized threshold is **PHP 10,000** per item; however, for practical purposes, many Philippine companies adopt a higher internal policy threshold of **PHP 50,000** aligned with accounting materiality. PXL ERP allows the company to set this threshold. The **Straight-Line Method (SLM)** is the most commonly used and BIR-accepted depreciation method for tax purposes, and it is set as the system default. The system also supports the **Double Declining Balance (DDB)** and **Sum of the Years' Digits (SYD)** methods for book purposes.

## What You Will See on the Screen (The Dashboard)
When you open the "Fixed Assets Settings" page, you will see a single, organized settings form — not a list. This page has one configuration record per company.

### The Action Bar (Top of the page):
* **Company Selector:** A dropdown to select which company's fixed assets settings you are configuring.
* **Save Settings Button:** A prominent button to save all changes on the page.
* **Reset to Defaults Button:** A secondary button to revert all settings to the system defaults.

## What Information We Need to Capture (The Data Fields)
The settings form is divided into logical sections.

### Section 1: Capitalization Policy
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. These settings apply to the selected entity. | Yes |
| Capitalization Threshold (PHP) | The minimum acquisition cost (in Philippine Peso) for an item to be recorded as a fixed asset and depreciated. Items below this amount are expensed immediately. The recommended default is **PHP 10,000** (BIR minimum) or **PHP 50,000** (common policy threshold). | Yes |
| Default Expense Account (Below Threshold) | Dropdown linked to the Chart of Accounts. The GL account where items that fall below the threshold are charged (e.g., "6010-000 Office Supplies Expense" or "6020-000 Repairs & Maintenance"). | Yes |

### Section 2: Depreciation Defaults
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Default Depreciation Method | Dropdown: **Straight-Line Method (SLM)** (Recommended — BIR accepted for tax returns), Double Declining Balance (DDB), Sum of the Years' Digits (SYD). This default applies to all new asset categories unless overridden at the asset category level. | Yes |
| Default Salvage Value % | The default residual/salvage value expressed as a percentage of the asset's acquisition cost (e.g., 10% means an asset costing PHP 100,000 has a PHP 10,000 salvage value). This default is applied to new assets but can be overridden per asset category or individual asset. | Yes |
| Depreciation Posting Frequency | Dropdown: Monthly (Recommended — posts a depreciation journal entry on the last day of each month), Quarterly, Annually. Monthly posting produces the most accurate interim financial statements. | Yes |
| Prorate Depreciation in Month of Acquisition | Toggle (Yes / No). If YES, an asset acquired mid-month is depreciated for only the remaining days of that month (pro-rata). If NO, a full month of depreciation is charged in the month of acquisition regardless of the acquisition date. | Yes |
| Default Accumulated Depreciation Account | Dropdown linked to the Chart of Accounts. The contra-asset GL account where accumulated depreciation balances are held (e.g., "1520-000 Accumulated Depreciation"). | Yes |
| Default Depreciation Expense Account | Dropdown linked to the Chart of Accounts. The expense GL account where periodic depreciation charges are posted (e.g., "6500-000 Depreciation Expense"). | Yes |

### Section 3: Disposal & Impairment Controls
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Require Approval Before Disposal | Toggle (Yes / No). If YES, an asset disposal or write-off transaction must go through the Approval Matrix workflow before it is posted to the GL. | Yes |
| Default Gain on Sale Account | Dropdown linked to the Chart of Accounts. The GL account for gains recognized on asset disposal (e.g., "7100-000 Gain on Sale of Fixed Assets"). | Yes |
| Default Loss on Sale Account | Dropdown linked to the Chart of Accounts. The GL account for losses recognized on asset disposal (e.g., "8100-000 Loss on Sale of Fixed Assets"). | Yes |
| Enable Impairment Tracking | Toggle (Yes / No). If YES, the system activates the Impairment module, allowing users to record and reverse impairment losses in compliance with PAS 36. | Yes |

### Section 4: Asset Tagging & Tracking
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Auto-Generate Asset Tag Number | Toggle (Yes / No). If YES, the system automatically assigns a unique Asset Tag Number to each new asset using the Number Series configuration. If NO, the user manually enters the asset tag. | Yes |
| Enable Branch-Level Asset Tracking | Toggle (Yes / No). If YES, each asset is assigned to a specific branch and department for location-based reporting and management. | Yes |
| Require Physical Verification | Toggle (Yes / No). If YES, the Fixed Asset Register flags assets that have not been physically verified within a configured period (e.g., annually), triggering a verification workflow. | Yes |

## The Supabase Database Architecture
This module requires one settings table. It stores one record per company.

### Table 1: `fixed_assets_settings` (The Main Table)
**Critical Database Rule:** Apply a UNIQUE constraint on `company_id` to ensure only one settings record exists per company.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key, Unique | Links to `companies.id`. |
| `capitalization_threshold` | Numeric(18,2) | Required, Default: `10000.00` | Minimum cost to capitalize an asset (in PHP). |
| `below_threshold_gl_id` | UUID | Nullable, Foreign Key | Links to `gl_accounts.id`. Expense account for items below threshold. |
| `default_depreciation_method` | Text | Required, Default: `straight_line` | `straight_line`, `double_declining`, `sum_of_years_digits`. |
| `default_salvage_value_pct` | Numeric(5,2) | Required, Default: `10.00` | Default salvage value as a percentage of cost. |
| `depreciation_posting_frequency` | Text | Required, Default: `monthly` | `monthly`, `quarterly`, `annually`. |
| `prorate_acquisition_month` | Boolean | Default: `true` | Whether to prorate depreciation in the month of acquisition. |
| `default_accum_depreciation_gl_id` | UUID | Nullable, Foreign Key | Links to `gl_accounts.id`. Accumulated depreciation contra-account. |
| `default_depreciation_expense_gl_id` | UUID | Nullable, Foreign Key | Links to `gl_accounts.id`. Depreciation expense account. |
| `require_disposal_approval` | Boolean | Default: `true` | Whether asset disposals require approval before posting. |
| `default_gain_on_sale_gl_id` | UUID | Nullable, Foreign Key | Links to `gl_accounts.id`. Gain on disposal account. |
| `default_loss_on_sale_gl_id` | UUID | Nullable, Foreign Key | Links to `gl_accounts.id`. Loss on disposal account. |
| `enable_impairment_tracking` | Boolean | Default: `false` | Whether the impairment module is active. |
| `auto_generate_asset_tag` | Boolean | Default: `true` | Whether asset tag numbers are auto-generated. |
| `enable_branch_tracking` | Boolean | Default: `true` | Whether assets are tracked per branch and department. |
| `require_physical_verification` | Boolean | Default: `true` | Whether periodic physical verification is enforced. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
