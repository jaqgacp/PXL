# PXL ERP Blueprint: Company Bank Accounts

## Module Overview
The Company Bank Accounts module stores the official bank accounts of each company and branch registered in the system. These accounts serve as the foundation for all banking and treasury transactions — including payment vouchers, receipts, fund transfers, petty cash replenishments, and bank reconciliation. Each bank account is linked to a specific General Ledger (GL) account in the Chart of Accounts to ensure that all transactions automatically post to the correct cash or bank GL account. This setup is essential for accurate cash position reporting and check printing.

## What You Will See on the Screen (The Dashboard)
When you open the "Company Bank Accounts" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a bank account by Bank Name, Account Number, or Account Name.
* **Company Filter:** A dropdown to filter bank accounts by parent company.
* **Branch Filter:** A dropdown to filter by branch.
* **Currency Filter:** A dropdown to filter by currency (e.g., PHP, USD).
* **Filter:** A dropdown to show either "Active" or "Inactive" bank accounts.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of bank accounts by uploading a pre-formatted CSV or Excel file. Useful during initial system setup or when migrating a new client's data.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file (account numbers are masked in the export for security).
* **Print Button:** Generates a print-ready view of registered bank accounts for treasury documentation.
* **Create New Bank Account Button:** A prominent button to add a new bank account.

### The Data List (The main table - must be customizable):
You will see a list of all your registered bank accounts. The default columns displayed are:
* Account Name (Short)
* Bank Name
* Account Number (masked, e.g., ****5678)
* Currency
* GL Account Linked
* Default Settings (Badges showing if it's the Default Payment or Default Collection account)
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that bank account.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Bank Account" or "Edit," a form will open. The form is divided into logical sections to make data entry easy. Here is the exact data the system will save.

### Section 1: Account Identity & Defaults
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. The entity that owns this bank account. | Yes |
| Branch | Dropdown linked to active branches. Identifies which branch manages this account. | Yes |

| Default Payment Account | Toggle switch. If YES, the system auto-selects this account when creating new Payment Vouchers. | Yes |

## The Supabase Database Architecture (For Lovable)
This module requires one main table. It links to `companies`, `branches`, and the `gl_accounts` table.

### Table 1: `company_bank_accounts` (The Main Table)
This table will store all the bank account records.

**Critical Database Rule for Lovable:** If `is_default_payment` is set to TRUE for an account, the system should ideally trigger logic to ensure all other accounts under the same company/branch are set to FALSE for that specific toggle.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the bank account. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `account_short_name` | Text | Required | Friendly label used across the system. |
| `account_type` | Text | Required | `checking`, `savings`, `time_deposit`, `fcd`. |
| `bank_name` | Text | Required | Name of the bank. |
| `bank_branch` | Text | Nullable | Specific bank branch. |
| `account_number` | Text | Required | Full account number. |
| `brstn_swift_code` | Text | Nullable | Routing code for electronic transfers. |
| `account_currency` | Text | Required | Currency code (e.g., "PHP"). |
| `gl_account_id` | UUID | Required, Foreign Key | Links to `gl_accounts.id` (Chart of Accounts). |
| `check_template_id` | Text | Nullable | Enum/ID for printer alignment format. |
| `is_default_payment` | Boolean | Default: `false` | Marks if this is the main disbursement account. |
| `is_default_collection`| Boolean | Default: `false` | Marks if this is the main receiving account. |
| `opening_balance` | Numeric(18,2) | Nullable | Book balance as of go-live date. |
| `opening_balance_date` | Date | Nullable | As-of date for the opening balance. |
| `signatory_1_name` | Text | Nullable | Full name of primary signatory. |
| `signatory_1_position` | Text | Nullable | Job title of primary signatory. |
| `signatory_2_name` | Text | Nullable | Full name of secondary signatory. |
| `signatory_2_position` | Text | Nullable | Job title of secondary signatory. |
| `remarks` | Text | Nullable | Additional notes about this account. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
