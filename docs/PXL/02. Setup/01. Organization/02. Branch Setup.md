# PXL ERP Blueprint: Branch Setup

## Module Overview
The Branch Setup allows you to define and manage all the physical or operational branches of your business entity. Each branch is a child of a parent company and can have its own address, RDO assignment, and contact details. Branch data is used across the entire system to filter and segregate transactions, financial statements, and compliance reports per location. In a multi-branch Philippine business, each branch may have its own BIR registration (Form 2303), a unique 5-digit TIN branch code suffix, and potentially its own local tax incentive status (like PEZA).

## What You Will See on the Screen (The Dashboard)
When you open the "Branch Setup" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a branch by typing its Branch Code or Branch Name.
* **Company Filter:** A dropdown to filter branches by their parent company (important for multi-company setups).
* **Filter:** A dropdown to show either "Active" or "Inactive" branches.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of branches by uploading a pre-formatted CSV or Excel file. Useful for onboarding companies with many branches at once.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for reporting or offline review.
* **Print Button:** Generates a print-ready view of the branch list for hardcopy documentation.
* **Create New Branch Button:** A prominent button to add a new branch.

### The Data List (The main table - must be customizable):
You will see a list of all your created branches. The default columns displayed are:
* Branch Code
* Branch Name
* Company (Parent)
* Branch Type
* TIN Branch Code
* City / Municipality
* RDO Code
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that branch.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Branch" or "Edit," a form will open. The form is divided into logical sections to make data entry easy. Here is the exact data the system will save.

### Section 1: Branch Identity
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. This is the parent entity that owns this branch. | Yes |
| Branch Code | A short identifier for the branch (e.g., "HO", "BR-001", "CEB"). Auto-uppercase. | Yes |
| Branch Name | The full descriptive name of the branch (e.g., "Head Office", "Cebu Branch"). | Yes |
| Branch Type | Dropdown: Head Office, Branch, Satellite Office, Warehouse, Project Site. | Yes |
| TIN Branch Code | The 5-digit suffix appended to the company TIN to identify this branch (e.g., 00000 for Head Office, 00001 for Branch 1). | Yes |

### Section 2: BIR, Tax & LGU Registration Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| RDO Code | The Revenue District Office where this branch is registered. Dropdown linked to `ref_rdo_codes`. | Yes |
| Tax Registration Override | Dropdown: Inherit from Company, PEZA, BOI, BMBE. (Allows this specific branch to have a different tax rule, e.g., VAT-Exempt if PEZA). | Yes (Default: Inherit) |
| BIR Registration Date | The date this branch was registered with BIR (from its own Form 2303). | Optional |
| LGU Permit Number | The Mayor's Permit or Business Permit Number for this specific branch. | Optional |
| LGU Reg. Date | The date the Mayor's Permit was issued. | Optional |
| CAS/PTU Number | CAS Acknowledgement Certificate or Permit to Use (PTU) number specific to this branch. | Optional |
| CAS Date Issued | The date the branch-specific CAS/PTU was granted by the BIR. | Optional |

### Section 3: Branch Address
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Address Line 1 | Unit/Building/Lot/Block/Street Name. | Yes |
| Address Line 2 | Subdivision/Village and Barangay. | Yes |
| City / Municipality | The City or Municipality. | Yes |
| Province | The Province. | Yes |
| ZIP Code | The 4-digit postal code. | Yes |

### Section 4: Contact Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Branch Email | Official email address for this branch. | Optional |
| Phone Number | Landline contact number of the branch. | Optional |
| Mobile Number | Mobile contact number of the branch. | Optional |
| Branch Manager | Name of the person managing or responsible for this branch. | Optional |

## The Supabase Database Architecture (For Lovable)
This module requires one main table. It links directly to the `companies` and `ref_rdo_codes` tables established in prior setup steps.

### Table 1: `branches` (The Main Table)
This table will store all the branch records for each company.

**Critical Database Rule for Lovable:** Ensure a composite unique constraint is placed on `company_id` and `branch_code`. Two different companies can have a branch called "HO", but one company cannot have two branches called "HO".

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the branch. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_code` | Text | Required | Short identifier (e.g., "BR-001"). |
| `branch_name` | Text | Required | Full branch name. |
| `branch_type` | Text | Required | `head_office`, `branch`, `satellite_office`, `warehouse`, `project_site`. |
| `tin_branch_code` | Text | Required | 5-digit TIN branch suffix (e.g., "00000", "00001"). |
| `rdo_id` | UUID | Required, Foreign Key | Links to `ref_rdo_codes.id`. |
| `tax_registration_override`| Text | Required | Defaults to `inherit`. Options: `peza`, `boi`, `bmbe`. |
| `bir_reg_date` | Date | Nullable | BIR registration date for this branch. |
| `lgu_permit_number` | Text | Nullable | Mayor's Permit number. |
| `lgu_reg_date` | Date | Nullable | Mayor's Permit date. |
| `cas_permit_no` | Text | Nullable | Branch-specific CAS/PTU number. |
| `cas_date_issued` | Date | Nullable | Date branch CAS/PTU was issued. |
| `address_line_1` | Text | Required | Street, Building. |
| `address_line_2` | Text | Required | Barangay, Subdivision. |
| `city` | Text | Required | City/Municipality. |
| `province` | Text | Required | Province. |
| `zip_code` | Text | Required | 4-digit postal code. |
| `email` | Text | Nullable | Branch email. |
| `phone_number` | Text | Nullable | Landline. |
| `mobile_number` | Text | Nullable | Mobile. |
| `branch_manager` | Text | Nullable | Name of the branch manager. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
