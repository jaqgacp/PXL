# PXL ERP Blueprint: Cost Centers

## Module Overview
Cost Centers allow the business to track and monitor income and expenses at a granular level beyond the standard department or branch structure. Each cost center represents a distinct segment of the business — such as a project, product line, geographical area, or operational unit — that requires its own financial accountability. Cost center codes are tagged on journal entries, purchase orders, vendor bills, and sales transactions to enable detailed profitability and expense analysis in management reports. The system supports hierarchical cost centers (parent-child) for rollup reporting.

## What You Will See on the Screen (The Dashboard)
When you open the "Cost Centers" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a cost center by typing its Code or Name.
* **Company Filter:** A dropdown to filter cost centers by parent company.
* **Type Filter:** A dropdown to filter by cost center type (e.g., Revenue Center, Cost Center, Profit Center).
* **Filter:** A dropdown to show either "Active" or "Inactive" cost centers.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of cost centers by uploading a pre-formatted CSV or Excel file. Useful for importing project or segment structures during setup.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for reporting or offline review.
* **Print Button:** Generates a print-ready view of the cost center list for hardcopy documentation.
* **Create New Cost Center Button:** A prominent button to add a new cost center.

### The Data List (The main table - must be customizable):
You will see a list of all your created cost centers. The default columns displayed are:
* Cost Center Code
* Cost Center Name
* Company (Parent)
* Type
* Manager
* Validity Period
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that cost center.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Cost Center" or "Edit," a form will open. The form is divided into logical sections to make data entry easy. Here is the exact data the system will save.

### Section 1: Cost Center Identity
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. The parent entity that owns this cost center. | Yes |
| Cost Center Code | A short, unique identifier (e.g., "CC-001", "PROJ-MNL"). Auto-uppercase. | Yes |
| Cost Center Name | The full descriptive name (e.g., "Manila Operations", "Project Alpha"). | Yes |
| Cost Center Type | Dropdown: Revenue Center, Cost Center, Profit Center, Investment Center. | Yes |
| Parent Cost Center | Dropdown of existing cost centers under the same company. Used for hierarchical grouping. | Optional |

### Section 2: Organizational Links & Controls
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Branch | Dropdown linked to active branches. Associates this cost center with a specific branch. | Optional |
| Department | Dropdown linked to active departments. Associates this cost center with a department. | Optional |
| Manager | Dropdown linking to the System Users. Crucial for routing approvals (e.g., Project Manager approves POs). | Optional |
| Valid From | The start date when expenses/income can be booked to this cost center (Crucial for Projects). | Optional |
| Valid To | The end date. System will block new transactions to this cost center past this date. | Optional |
| Description | A brief description of what this cost center tracks and its purpose. | Optional |

## The Supabase Database Architecture
This module requires one main table. It supports self-referencing for parent-child hierarchy and links to `companies`, `branches`, `departments`, and `users`.

### Table 1: `cost_centers` (The Main Table)
This table will store all the cost center records for the organization.

**Critical Database Rule:** 
1. Ensure a composite unique constraint is placed on `company_id` and `cost_center_code`. 
2. The `parent_cost_center_id` must have a foreign key constraint referencing `id` in the same `cost_centers` table.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the cost center. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_id` | UUID | Nullable, Foreign Key | Links to `branches.id`. |
| `department_id` | UUID | Nullable, Foreign Key | Links to `departments.id`. |
| `manager_user_id` | UUID | Nullable, Foreign Key | Links to `users.id` for approval workflows. |
| `cost_center_code` | Text | Required | Short identifier (e.g., "CC-001"). |
| `cost_center_name` | Text | Required | Full name of the cost center. |
| `cost_center_type` | Text | Required | `revenue_center`, `cost_center`, `profit_center`, `investment_center`. |
| `parent_cost_center_id` | UUID | Nullable, Foreign Key | Self-references `cost_centers.id` for hierarchy. |
| `valid_from` | Date | Nullable | Start date of cost center validity. |
| `valid_to` | Date | Nullable | End date of cost center validity. |
| `description` | Text | Nullable | Brief description of what this cost center tracks. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
