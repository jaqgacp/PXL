# PXL ERP Blueprint: Book of Account / PTU CAS

## Module Overview
The Book of Account / PTU CAS module tracks the official BIR registration of a company's books of accounts. Under Philippine tax law, every registered taxpayer is required to register their books of accounts. There are three types of registration: **Manual**, **Loose-Leaf** (requires a Permit to Use), and **Computerized Accounting System (CAS)** (requires an Acknowledgement Certificate). This module records the specific books covered, the volume numbers, the permit details, and securely stores the scanned BIR certificates. This data is critical to validate system-generated books and ensure audit readiness.

## What You Will See on the Screen (The Dashboard)
When you open the "Book of Account / PTU CAS" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a registration record by Company, Branch, or Permit Number.
* **Company Filter:** A dropdown to filter records by parent company.
* **Branch Filter:** A dropdown to filter by branch.
* **Registration Type Filter:** A dropdown to filter by type: Manual, Loose-Leaf, or CAS.
* **Filter:** A dropdown to show either "Active" or "Inactive" registration records.
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of book registrations by uploading a pre-formatted CSV or Excel file. Useful for importing historical PTU/CAS data for clients with multiple branches.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for audit preparation or BIR compliance records.
* **Print Button:** Generates a print-ready summary of all book registrations for hardcopy documentation.
* **Create New Registration Button:** A prominent button to add a new book registration record.

### The Data List (The main table - must be customizable):
You will see a list of all your book registration records. The default columns displayed are:
* Company
* Branch
* Registration Type
* Permit / AC Number
* Effective Date
* Status (A green badge for Active, gray for Inactive)
* Attachment (A paperclip icon to view the scanned BIR certificate)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that registration.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Registration" or "Edit," a form will open. The form is divided into logical sections to make data entry easy. Here is the exact data the system will save.

### Section 1: Registration Identity
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. The entity whose books are being registered. | Yes |
| Branch | Dropdown linked to active branches. Select the specific branch this registration applies to. | Yes |
| Registration Type | Dropdown: Manual Books of Account, Loose-Leaf Books of Account, Computerized Accounting System (CAS). | Yes |
| Effective Date | The exact date this registration or permit takes effect (Go-Live Date). | Yes |

### Section 2: Permit / Certificate Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Permit / AC Number | The Permit to Use (PTU) number for Loose-Leaf, or the Acknowledgement Certificate (AC) number for CAS. | Conditional (Yes for CAS/Loose-leaf) |
| Date Issued | The date the BIR issued the Permit, AC, or stamped the manual books. | Yes |
| RDO Code | The Revenue District Office that issued the permit. Dropdown linked to `ref_rdo_codes`. | Yes |
| BIR Form Used | The BIR form filed (e.g., BIR Form 1900). | Optional |
| Certificate Attachment | File upload button (PDF/Image) to store the scanned copy of the BIR-approved certificate or stamped first page. | Optional (Highly Recommended) |

### Section 3: Books Registered (The Detail Lines)
This section is a dynamic table/checklist where the user adds the specific books covered under this registration.

| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Book Type | Dropdown of standard books (General Journal, General Ledger, Cash Receipts Book, Cash Disbursements Book, Sales Journal, Purchase Journal, etc.). | Yes |
| Volume Number | The specific volume number registered (Crucial for Manual Books). | Conditional |
| Number of Pages | The total number of pages stamped by the BIR (Crucial for Manual Books). | Conditional |

### Section 4: Remarks
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Remarks | Any additional notes about this registration. | Optional |

## The Supabase Database Architecture
This module requires three tables: a reference table for book types, a main header table, and a detail line-item table.

### Table 1: `ref_book_types` (Reference Table)
Pre-seeded lookup table of all available books of account.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `book_code` | Text | Required, Unique | Internal code (e.g., "GJ", "GL"). |
| `book_name` | Text | Required | Full name (e.g., "General Journal"). |
| `sort_order` | Integer | Required | Controls checklist display order. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 2: `book_registrations` (The Main Header Table)
This is the main header table that stores the registration details for each permit.

**Critical Database Rule:** Add a composite unique constraint on `company_id`, `branch_id`, and `permit_ac_number` (where `permit_ac_number` is not null) to prevent duplicate permits.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the registration record. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `branch_id` | UUID | Required, Foreign Key | Links to `branches.id`. |
| `registration_type` | Text | Required | `manual`, `loose_leaf`, `cas`. |
| `effective_date` | Date | Required | The go-live or effective date of the books. |
| `permit_ac_number` | Text | Nullable | PTU or AC number from BIR. |
| `date_issued` | Date | Required | Date BIR issued or stamped the permit. |
| `rdo_id` | UUID | Required, Foreign Key | Links to `ref_rdo_codes.id`. |
| `bir_form_used` | Text | Nullable | The BIR form filed (e.g., 1900). |
| `certificate_url` | Text | Nullable | Supabase Storage URL for the scanned document. |
| `remarks` | Text | Nullable | Additional notes. |
| `is_active` | Boolean | Default: `true` | System status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |

### Table 3: `book_registration_items` (The Detail Table)
This is the detail table that stores the specific list of books included in the registration, along with volume details.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for this line item. |
| `book_registration_id` | UUID | Required, Foreign Key | Links to `book_registrations.id` (ON DELETE CASCADE). |
| `book_type_id` | UUID | Required, Foreign Key | Links to `ref_book_types.id`. |
| `volume_number` | Text | Nullable | Volume number (e.g., "1", "002"). |
| `number_of_pages` | Integer | Nullable | Page count stamped by BIR. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. User who created this record. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. User who last edited this record. |
| `created_at` | Timestamp | Auto | Date record was created. |
| `updated_at` | Timestamp | Auto | Date record was last edited. |
