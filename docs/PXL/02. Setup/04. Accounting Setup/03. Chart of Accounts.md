# PXL ERP Blueprint: Chart of Accounts

## Module Overview
The Chart of Accounts (COA) is the master list of all General Ledger accounts used for financial recording. To meet BIR requirements, businesses must maintain a structured COA that aligns with the accounts used in their filed ITRs and Audited Financial Statements (AFS).

## What You Will See on the Screen (The Dashboard)
A tree-like structure displaying account hierarchies (Groups, Sub-groups, and Postable Accounts).

### The Action Bar (Top of the page):
* **Search Bar:** Search by Account Code or Name.
* **Import Button:** Import COA via CSV.
* **Export Button:** Export COA.
* **Create Account Button:** Add a new GL account.

### The Data List (The main table):
* Account Code
* Account Name
* Account Type (Asset, Liability, Equity, Revenue, Expense)
* Normal Balance (Debit/Credit)
* Is Postable? (Yes/No)
* Status

## What Information We Need to Capture (The Data Fields)

### Section 1: Account Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Account Code | Unique identifier (e.g., "1010-000"). | Yes |
| Account Name | Name (e.g., "Cash in Bank - BDO"). | Yes |
| Parent Account | Group account this falls under. | Optional |
| Account Type | Main classification. | Yes |
| Normal Balance | Debit or Credit. | Yes |
| Is Postable | Can receive transactions directly? | Yes |
| Currency | Allowed currency (blank for all). | Optional |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `chart_of_accounts`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `account_code` | Text | Required | The numeric code (e.g., `1010-000`). Unique per company — see composite constraint below. |
| `account_name` | Text | Required | E.g., `Cash on Hand`. |
| `parent_id` | UUID | Nullable, Foreign Key | Self-reference to group account. |
| `account_type` | Text | Required | `asset`, `liability`, `equity`, `revenue`, `expense`. |
| `normal_balance` | Text | Required | `debit`, `credit`. |
| `is_postable` | Boolean | Default: `false` | If `true`, entries can hit this. |
| `currency_code` | Text | Nullable | Specific currency restriction. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

**Composite Unique Constraint:** `UNIQUE(company_id, account_code)`
*Note: The unique constraint is scoped to company_id. Multiple companies may use the same account codes (e.g., every company will have a `1010-000 Cash on Hand`). A global unique constraint on `account_code` alone will fail on the second company's setup.*
