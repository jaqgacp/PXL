# PXL ERP Blueprint: Financial Statement Fields

## Module Overview
Financial Statement Fields setup allows the mapping of Chart of Accounts items into structured reporting lines for the Balance Sheet, Income Statement, and Cash Flow statement. In the Philippines, mapping correctly to SEC and BIR required formats is necessary for standard compliance.

## What You Will See on the Screen (The Dashboard)
A layout designer for financial reports.

### The Action Bar (Top of the page):
* **Report Selector:** Balance Sheet, Income Statement.
* **Export Button:** Export mapping.
* **Add Section Button:** Create a new report header (e.g., "Current Assets").

### The Data List (The main table):
* Report Line Name (e.g., "Cash and Cash Equivalents")
* Mapped Accounts (e.g., 1010-000, 1020-000)
* Print Order

---

## What Information We Need to Capture (The Data Fields)

### Section 1: FS Layout Header
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Report Type | BS, IS, CF. | Yes |
| Section Name | E.g., "Current Assets". | Yes |
| Sequence | Ordering number. | Yes |
| Parent Section | If nested under another header. | Optional |

### Section 2: Account Mapping
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Account | GL Account from COA. | Yes |
| Sign Reversal | Print negative balances as positive (e.g., accumulated depreciation). | Yes |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `fs_report_sections` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `report_type` | Text | Required | `balance_sheet`, `income_statement`. |
| `section_name` | Text | Required | Name of line item on FS. |
| `parent_id` | UUID | Nullable, Foreign Key | Hierarchy control. |
| `sequence_no` | Integer | Required | Display order. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

### Table 2: `fs_account_mappings` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `section_id` | UUID | Required, Foreign Key | Links to `fs_report_sections.id`. |
| `account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `reverse_sign` | Boolean | Default: `false` | If true, flip Dr/Cr sign. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
