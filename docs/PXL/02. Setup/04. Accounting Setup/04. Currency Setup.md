# PXL ERP Blueprint: Currency Setup

## Module Overview
The Currency Setup module defines the base currency of the company and manages foreign currencies used in transactions. In the Philippines, the base currency is generally PHP, but import/export businesses transact in USD, EUR, etc. This module stores daily or monthly exchange rates for accurate forex gain/loss computation.

## What You Will See on the Screen (The Dashboard)
A list of activated currencies and a panel for exchange rates.

### The Action Bar (Top of the page):
* **Search Bar:** Currency Code.
* **Add Currency Button:** Enable a new currency.
* **Update Rates Button:** Input new exchange rates.

### The Data List (The main table):
* Currency Code (PHP, USD)
* Currency Name
* Symbol
* Is Base Currency?
* Current Exchange Rate (to Base)

---

## What Information We Need to Capture (The Data Fields)

### Section 1: Currency Header
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Currency Code | ISO 3-letter code (e.g., USD). | Yes |
| Name | US Dollar. | Yes |
| Symbol | $ or PHP. | Yes |
| Is Base Currency | Toggle. Only one base allowed. | Yes |

### Section 2: Exchange Rates (Line Items)
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Date | Date the rate is valid. | Yes |
| Rate | Conversion rate to Base Currency. | Yes |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `currencies` (Header)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `currency_code` | Text | Required, Unique | E.g., `USD`. |
| `name` | Text | Required | E.g., `US Dollar`. |
| `symbol` | Text | Required | E.g., `$`. |
| `is_base` | Boolean | Default: `false` | Is it the local reporting currency? |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

### Table 2: `exchange_rates` (Line Items)
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `currency_id` | UUID | Required, Foreign Key | Links to `currencies.id`. |
| `effective_date` | Date | Required | Date rate applies. |
| `exchange_rate` | Numeric | Required | Multiplier to get base value. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
