# PXL ERP Blueprint: Master Data Rules

## Module Overview
Master Data Rules define mandatory fields and validation logic for core records (Customers, Vendors, Items). For Philippine tax compliance, ensuring that a Vendor has a valid Philippine TIN in `XXX-XXX-XXX-XXXXX` format and ATC code before they can be used in a transaction is essential. This module allows admins to enforce these data completeness rules.

## What You Will See on the Screen (The Dashboard)
A list of customizable rules applied to Master Data.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Entity Type.
* **Filter:** Active/Inactive.
* **Export Button:** Export rules.
* **Create Rule Button:** Add a new validation requirement.

### The Data List (The main table):
* Entity Type
* Field Name
* Rule Type (Required, Regex Format, Unique)
* Error Message

---

## What Information We Need to Capture (The Data Fields)

### Section 1: Validation Rule
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Entity Type | Customer, Vendor, Item, etc. | Yes |
| Field Name | The specific column being checked (e.g., "tin_number"). | Yes |
| Rule Type | Required, Format Match, Must Be Unique. | Yes |
| Regex Pattern | Used if Rule Type is Format Match (e.g., `^\d{3}-\d{3}-\d{3}-\d{4}$` for TIN). | Optional |
| Error Message | Text shown to the user when validation fails. | Yes |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `master_data_validation_rules`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `entity_type` | Text | Required | E.g., `vendor`. |
| `field_name` | Text | Required | E.g., `tin`. |
| `rule_type` | Text | Required | `required`, `format`, `unique`. |
| `regex_pattern` | Text | Nullable | Regular expression for matching. |
| `error_message` | Text | Required | E.g., "TIN must be 13 digits." |
| `is_active` | Boolean | Default: `true` | Status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
