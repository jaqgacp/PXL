# PXL ERP Blueprint: Transaction Rules

## Module Overview
Transaction Rules validate business logic during data entry on operational documents (e.g., ensuring a Sales Order Delivery Date is not in the past, or that PO amounts don't exceed budget). In the local context, preventing invalid transaction dates is critical for proper VAT reporting.

## What You Will See on the Screen (The Dashboard)
The dashboard lists rules applied during the saving of transaction documents.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Document Type.
* **Filter:** Active/Inactive.
* **Create Rule Button:** Add a new transaction validation rule.

### The Data List (The main table):
* Document Type
* Rule Description
* Rule Condition (Formula/Logic)
* Action (Warning, Block)

---

## What Information We Need to Capture (The Data Fields)

### Section 1: Transaction Logic Rule
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Document Type | Sales Order, PO, etc. | Yes |
| Rule Description | E.g., "Delivery Date cannot be in the past". | Yes |
| Target Field | The field being validated. | Yes |
| Logic Operator | Greater than, Less than, Not Empty. | Yes |
| Comparison Value| The value to compare against (e.g., "today()"). | Yes |
| Enforcement | Warning (User can proceed), Block (User cannot save). | Yes |

---

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `transaction_validation_rules`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `document_type` | Text | Required | E.g., `sales_order`. |
| `rule_description`| Text | Required | Friendly name. |
| `target_field` | Text | Required | E.g., `delivery_date`. |
| `logic_operator` | Text | Required | `>`, `<`, `=`, `!=`. |
| `compare_value` | Text | Required | E.g., `current_date`. |
| `enforcement` | Text | Required | `warning`, `block`. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
