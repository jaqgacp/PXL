# PXL ERP Blueprint: ATC Codes

## Module Overview
Alphanumeric Tax Codes (ATC) are BIR-mandated identifiers indicating the exact nature of the tax base and rate. It is the core reporting element for all Alphalists. Keeping this table updated ensures that generated DAT files are accepted by the BIR Validation tool without errors.

## What You Will See on the Screen (The Dashboard)
A comprehensive master list of all BIR ATCs.

### The Action Bar (Top of the page):
* **Search Bar:** Search by ATC or Description.
* **Import ATCs Button:** Upload latest BIR list.

### The Data List (The main table):
* ATC Code
* Description
* Tax Rate (%)
* Tax Type (EWT, FWT, PT)

## What Information We Need to Capture (The Data Fields)

### Section 1: ATC Details
| Field Name | Description | Required? |
| :--- | :--- | :--- |

| Description | E.g., "Professionals - 10%". | Yes |
| Rate | 10.00. | Yes |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `atc_codes`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `code` | Text | Required, Unique | E.g., `WC158`. |
| `description` | Text | Required | BIR Official description. |
| `rate` | Numeric | Required | Corresponding rate. |
| `tax_category` | Text | Required | `ewt`, `fwt`, `it`, `pt`. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
