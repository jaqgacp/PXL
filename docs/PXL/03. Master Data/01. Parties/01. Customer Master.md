# PXL ERP Blueprint: Customer Master

## Module Overview
The Customer Master is the foundational AR sub-ledger record. Every Sales Invoice, Official Receipt, Credit Memo, and Debit Memo must be linked to a customer record. In the Philippine context, the customer record stores BIR-critical information: TIN, VAT registration status, and withholding agent status (which determines whether the customer will issue a BIR Form 2307 to the company). Incomplete customer records are the most common cause of rejected BIR alphalist submissions.

## Dashboard UI

### The Action Bar
* **Search Bar:** Search by Customer Code, Name, or TIN.

## Supabase Database Architecture

### Table 1: `customers`

**Critical Database Rules:**
1. Composite unique constraint on `(company_id, customer_code)` — tenant-scoped uniqueness.
2. `tin` is validated for format at application layer. No global unique constraint on TIN alone (different companies may serve the same customer, and government/large corporates may have the same TIN across many supplier databases).
3. `is_subject_to_cwt = true` with a `default_cwt_atc_code_id` drives customer CWT defaults for sales/receipt workflows.

| Column Name | Data Type | Rules | Description |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `customer_code` | Text | Required | Unique per company. Auto-generated or manual. |
| `customer_name` | Text | Required | Registered name per BIR Form 2303. |
| `customer_type` | Text | Required | `individual`, `corporation`, `government`, `non_resident`. |
| `tin` | Text | Nullable | Philippine TIN (format: `NNN-NNN-NNN-NNNNN`). |
| `tin_branch_code` | Text | Nullable, Default `00000` | 5-digit BIR branch identifier. |
| `vat_status` | Text | Required | `vat_registered`, `non_vat`, `exempt`. |
| `is_subject_to_cwt` | Boolean | Required, Default `false` | If true, customer is expected to withhold CWT on payments. |
| `default_cwt_atc_code_id` | UUID | Nullable, Foreign Key | Links to `atc_codes.id`. Required when a default CWT ATC should auto-fill. |
| `registered_address` | Text | Required | Full BIR-registered address. |
| `city` | Text | Required | City or municipality. |
| `province` | Text | Required | Province. |
| `zip_code` | Text | Required | Postal code. |
| `rdo_id` | UUID | Nullable, Foreign Key | Links to `ref_rdo_codes.id`. |
| `contact_person` | Text | Nullable | Primary contact name. |
| `contact_number` | Text | Nullable | Phone number. |
| `email_address` | Text | Nullable | Email for e-invoicing. |
| `ar_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. AR control account. |
| `payment_terms_id` | UUID | Required, Foreign Key | Links to `payment_terms.id`. |
| `credit_limit` | Numeric(15,2) | Required, Default 0 | 0 = no limit enforced. |
| `currency_code` | Text | Required, Default `PHP` | Default transaction currency. |
| `default_sales_tax_id` | UUID | Nullable, Foreign Key | Links to `vat_codes.id`. Auto-fills on Sales Invoice. |
| `is_active` | Boolean | Required, Default `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, customer_code)`
