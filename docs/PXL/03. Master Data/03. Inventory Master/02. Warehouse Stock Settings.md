# PXL ERP Blueprint: Warehouse Stock Settings

## Module Overview
The Warehouse Stock Settings module handles location-specific inventory parameters. While the "Items" module holds global minimum/maximum stock levels, businesses in the Philippines often need different reorder points per region or branch due to logistical lead times (e.g., shipping to Visayas/Mindanao branches takes longer than restocking a Metro Manila warehouse). This module allows customized stock limits per warehouse to trigger intelligent purchase requests.

## What You Will See on the Screen (The Dashboard)
When accessing this page, the user selects a specific Warehouse first, then sees the item settings for that location.

### The Action Bar (Top of the page):
* **Warehouse Selector:** Dropdown to choose the active warehouse view.
* **Search Bar:** Search items by Name or Code.
* **Export Button:** Export settings for the selected warehouse.
* **Update Settings Button:** A button to open the bulk edit grid or import changes via CSV.

### The Data List (The main table):
Default columns displayed (filtered by selected warehouse):
* Item Code
* Item Name
* Current Stock
* Minimum Level
* Maximum Level
* Reorder Quantity

## What Information We Need to Capture (The Data Fields)

For a specific item in a specific warehouse:
### Section 1: Item-Warehouse Settings
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Item | Auto-populated or selectable from Items list. | Yes |
| Warehouse | Auto-populated based on current view. | Yes |
| Minimum Stock Level | Number Input. Alerts when stock in THIS warehouse falls below this. | Yes |
| Maximum Stock Level | Number Input. Max allowed stock for THIS warehouse. | Optional |
| Default Reorder Quantity | Number Input. The standard amount to order when minimum is breached. | Optional |

## The Supabase Database Architecture


**Critical Database Rule:** A composite unique constraint must be enforced on (company_id, code) to ensure multi-tenant data integrity.

### Table: `warehouse_stock_settings`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the setting. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `item_id` | UUID | Required, Foreign Key | Links to `items.id`. |
| `warehouse_id` | UUID | Required, Foreign Key | Links to `warehouses.id`. |
| `min_stock_level` | Numeric | Default: 0 | Location specific min stock. |
| `max_stock_level` | Numeric | Nullable | Location specific max stock. |
| `reorder_quantity` | Numeric | Default: 0 | Recommended reorder amount. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |

*(Note: Ensure a UNIQUE constraint on `(item_id, warehouse_id)` to prevent duplicate settings for the same item in the same warehouse).*
