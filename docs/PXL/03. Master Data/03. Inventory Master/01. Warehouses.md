## Module Overview
The Warehouses module defines the physical or logical locations where inventory is stored. In the Philippines, businesses with multiple branches or storage facilities require strict tracking to prevent pilferage and to accurately cost inventory per location. Properly defined warehouses are also crucial when requesting BIR permits to use offsite storage facilities or when declaring branch inventory in municipal tax assessments.

## What You Will See on the Screen (The Dashboard)
When opening the "Warehouses" page, a list of all storage locations will be presented.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Warehouse Code or Name.
* **Filter:** Filter by "Type" (e.g., Main, Branch, Virtual) and Status.
* **Customize View:** Select visible columns.
* **Export Button:** Export warehouse list.
* **Print Button:** Print list.
* **Create New Warehouse Button:** Add a location.

### The Data List (The main table):
Default columns:
* Warehouse Code
* Warehouse Name
* Type
* Address
* Status

## What Information We Need to Capture (The Data Fields)

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Warehouse Code | Text Input. Unique identifier (e.g., WH-MNL-01). | Yes |
| Warehouse Name | Text Input. Descriptive name (e.g., Manila Main Storage). | Yes |
| Type | Dropdown: Physical, Virtual (e.g., In-Transit), Branch. | Yes |
| Linked Branch/Company | Dropdown to link to a specific `companies` setup record. | Optional |

### Section 2: Address Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Address | Text Area. Full physical address of the warehouse. | Yes |
| City | Text Input. City or Municipality. | Yes |
| Contact Person | Text Input. Person managing the warehouse. | Optional |
| Contact Number | Text Input. Phone or mobile number. | Optional |

## The Supabase Database Architecture


**Critical Database Rule:** A composite unique constraint must be enforced on (company_id, `warehouse_code`) to ensure multi-tenant data integrity.

### Table: `warehouses`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the warehouse. |
| `warehouse_code` | Text | Required, Unique | Unique code. |
| `warehouse_name` | Text | Required | Warehouse name. |
| `warehouse_type` | Text | Required | `physical`, `virtual`, `branch`. |
| `company_id` | UUID | Nullable, Foreign Key | Links to `companies.id`. |
| `address` | Text | Required | Full address. |
| `city` | Text | Required | City name. |
| `contact_person` | Text | Nullable | Manager name. |
| `contact_number` | Text | Nullable | Contact number. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
