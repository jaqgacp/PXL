# PXL ERP Blueprint: Services

## Module Overview
The Services module acts as the master record for all intangible offerings a company provides to its clients (e.g., Consulting, Repair, Delivery, Subscription fees). Unlike inventory items, services do not have physical stock to track. In Philippine accounting, distinguishing services is vital for proper revenue recognition and ensuring VAT is calculated correctly on Official Receipts (OR) rather than Sales Invoices (SI).

## What You Will See on the Screen (The Dashboard)
Opening the "Services" page shows a catalog of registered service offerings.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Service Code or Name.
* **Filter:** Filter by Category or Status (Active/Inactive).
* **Export Button:** Export the list to CSV/Excel.
* **Create New Service Button:** Add a new service offering.
* **Import Button:** Bulk upload for service lists.

### The Data List (The main table):
Default columns:
* Service Code
* Service Name
* Category
* Billing UOM (e.g., Hours, Sessions)
* Standard Rate
* Status

## What Information We Need to Capture (The Data Fields)

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Service Code | Text Input. Unique identifier (e.g., SRV-CON). | Yes |
| Service Name | Text Input. Name of the service. | Yes |
| Description | Text Area. Details about what the service includes. | Optional |
| Category | Dropdown from `item_categories` (inherits Sales Accounts). | Yes |

### Section 2: Billing & Pricing
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Default UOM | Dropdown linked to Units of Measure master. | Yes |
| Default Price | Number Input. Default price charged for the service. | Yes |
| Default Cost | Number Input. Default cost for the service. | Optional |
| Default Tax Code | Dropdown linked to Tax Types master. | Yes |
| Default EWT Code | Dropdown linked to EWT Codes master. | Optional |

### Section 2.5: Automated Account Determination (GL Mapping)
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Inventory Account | Dropdown linked to Chart of Accounts (Asset) - N/A for Services but required by architecture. | Optional |
| COGS Account | Dropdown linked to Chart of Accounts (Cost of Sales). | Yes |
| Revenue Account | Dropdown linked to Chart of Accounts (Income). | Yes |
| Expense Account | Dropdown linked to Chart of Accounts (Expense). | Yes |

## The Supabase Database Architecture


**Critical Database Rule:** A composite unique constraint must be enforced on (company_id, `service_code`) to ensure multi-tenant data integrity.

### Table: `services`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the service. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `service_code` | Text | Required, Unique | Unique code. |
| `service_name` | Text | Required | Service name. |
| `description` | Text | Nullable | Service details. |
| `category_id` | UUID | Required, Foreign Key | Links to `item_categories.id`. |
| `default_uom_id` | UUID | Required, Foreign Key | Links to `units_of_measure.id`. |
| `default_price` | Numeric | Default: 0 | Default billing rate. |
| `default_cost` | Numeric | Default: 0 | Default cost. |
| `default_tax_code_id` | UUID | Required, Foreign Key | Links to `tax_types.id`. |
| `default_ewt_code_id` | UUID | Nullable, Foreign Key | Links to `ewt_codes.id`. |
| `inventory_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. |
| `cogs_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `revenue_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `expense_account_id` | UUID | Required, Foreign Key | Links to `chart_of_accounts.id`. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
