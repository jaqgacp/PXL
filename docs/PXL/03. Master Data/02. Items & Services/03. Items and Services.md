## Module Overview
The Items and Services master record is the central product catalog of the PXL ERP. It is the source of truth for every line item entered on Sales Invoices, Purchase Invoices, and Inventory transactions. Critically, this record carries the default GL accounts, default tax codes, and unit prices that auto-fill transactional documents — eliminating manual data entry errors. The item record is the primary trigger for the GL auto-posting chain: Item → Item Category → GL Posting Configuration.

**Auto-Fill Chain (highest to lowest precedence):**
1. Item-level GL accounts (stored directly on this record) override everything.
2. Item Category-level GL accounts (from `item_categories`) apply if item-level is blank.
3. Module-level GL Posting Configuration applies if both item and category are blank.
4. If no mapping is found at any level → posting is blocked and user is alerted.

### Section 2: Pricing
| Field Name | UI Component | Required? | Behavior |
| :--- | :--- | :--- | :--- |
| Standard Selling Price | Numeric | Yes | Default unit price auto-filled on Sales Invoice lines. User may override on the document. |
| Standard Cost | Numeric | Yes | Default unit cost auto-filled on Purchase Invoice lines. User may override on the document. |

| Field Name | UI Component | Required? | Behavior |
| :--- | :--- | :--- | :--- |

| Default Purchase VAT Code | Dropdown | Yes | The Input VAT code applied when this item is purchased. Links to `vat_codes.id` where `transaction_type = 'input_vat'`. **Auto-fills on every Purchase Invoice line when this item is selected.** |

### Section 4: Accounting GL Defaults (Auto-Fill Source — Highest Priority)
**These accounts, when populated, take priority over category-level and module-level GL mappings.**

| Field Name | UI Component | Required? | Behavior |
| :--- | :--- | :--- | :--- |

| Purchase Expense Account | Dropdown (COA) | Conditional | Required if `item_type = 'service'` or `'non_inventory'`. The expense account debited on purchase. Links to `chart_of_accounts.id`. |

## Supabase Database Architecture

### Table 1: `items`

**Critical Database Rules:**
1. Composite unique constraint on `(company_id, item_code)`.
2. `default_sales_vat_id` and `default_purchase_vat_id` are required. An item without a tax code assignment cannot be added to a Sales Invoice or Purchase Invoice — this ensures no transaction slips through without correct VAT treatment.
3. The four GL account columns (`sales_account_id`, `cogs_account_id`, `inventory_account_id`, `purchase_expense_account_id`) are recommended, not optional, because they are the highest-priority source in the GL auto-posting chain. The system must warn (not block) if they are empty when the item is saved.

| Column Name | Data Type | Rules | Description |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System identifier. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `item_code` | Text | Required | Unique per company. |
| `description` | Text | Required | Item name as printed on invoices. |
| `description_long` | Text | Nullable | Extended description. |
| `item_type` | Text | Required | `inventory_item`, `service`, `non_inventory`. |
| `category_id` | UUID | Required, Foreign Key | Links to `item_categories.id`. |
| `uom_id` | UUID | Required, Foreign Key | Links to `units_of_measure.id`. Default unit of measure. |
| `barcode` | Text | Nullable | Optional barcode/SKU. |
| `standard_selling_price` | Numeric(15,2) | Required, Default 0 | Default unit price on Sales Invoice. |
| `standard_cost` | Numeric(15,2) | Required, Default 0 | Default unit cost on Purchase Invoice. |
| `price_is_vat_inclusive` | Boolean | Required, Default `false` | If `true`, selling price is VAT-inclusive; back-compute VAT on invoice. |
| `default_sales_vat_id` | UUID | Required, Foreign Key | Links to `vat_codes.id` (`output_vat`). Auto-fills on Sales Invoice line. |
| `default_purchase_vat_id` | UUID | Required, Foreign Key | Links to `vat_codes.id` (`input_vat`). Auto-fills on Purchase Invoice line. |
| `sales_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. Revenue account (highest priority in GL chain). |
| `cogs_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. Cost of sales account. |
| `inventory_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. Inventory asset account (for inventory items only). |
| `purchase_expense_account_id` | UUID | Nullable, Foreign Key | Links to `chart_of_accounts.id`. Purchase expense account (for services and non-inventory). |
| `costing_method` | Text | Nullable | `fifo`, `weighted_average`, `specific_identification`. Required if `item_type = 'inventory_item'`. |
| `min_stock_level` | Numeric(15,4) | Nullable | Low-stock alert threshold. |
| `reorder_point` | Numeric(15,4) | Nullable | Reorder suggestion threshold. |
| `is_active` | Boolean | Required, Default `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Last update timestamp. |

**Composite Unique Constraint:** `UNIQUE(company_id, item_code)`
