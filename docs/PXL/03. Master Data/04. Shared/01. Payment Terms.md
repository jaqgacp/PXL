# PXL ERP Blueprint: Payment Terms

## Module Overview
The Payment Terms module standardizes the credit policies extended to customers and granted by suppliers. In Philippine business practice, clearly defining payment terms (e.g., COD, Net 30, 50% Downpayment) is vital for accurate cash flow projection, generating timely Statement of Accounts (SOA), and triggering alerts for past-due receivables which impact collection efforts and bad debt provisioning.

## What You Will See on the Screen (The Dashboard)
Opening the "Payment Terms" page shows a list of standard terms configured in the system.

### The Action Bar (Top of the page):
* **Search Bar:** Search by Term Name or Code.
* **Filter:** Filter by active status.
* **Export Button:** Export list.
* **Create New Term Button:** Add a new payment policy.

### The Data List (The main table):
Default columns:
* Term Code
* Description
* Days to Due
* Requires Downpayment?
* Status

---

## What Information We Need to Capture (The Data Fields)

### Section 1: Basic Information
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Term Code | Text Input. Identifier (e.g., NET30, COD, DP50). | Yes |
| Term Name | Text Input. Descriptive name (e.g., Net 30 Days, Cash on Delivery). | Yes |

### Section 2: Term Rules
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Days to Due | Number Input. Number of days until full payment is due from invoice date (0 for COD). | Yes |
| Is Downpayment Required? | Checkbox. If true, activates DP fields. | Yes |
| Downpayment Percentage | Number Input. Percentage required upfront (if applicable). | Conditional |

## The Supabase Database Architecture


**Critical Database Rule:** A composite unique constraint must be enforced on (company_id, `term_code`) to ensure multi-tenant data integrity.

### Table: `payment_terms`

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID for the payment term. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. |
| `term_code` | Text | Required, Unique | Short code. |
| `term_name` | Text | Required | Full description. |
| `days_to_due` | Integer | Required | Days from invoice date. |
| `require_downpayment` | Boolean | Default: `false` | DP requirement flag. |
| `dp_percentage` | Numeric | Nullable | Required if DP is needed. |
| `is_active` | Boolean | Default: `true` | Active status. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Last edit date. |
