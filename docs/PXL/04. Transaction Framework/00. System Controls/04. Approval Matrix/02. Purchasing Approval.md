## Module Overview
The Purchasing Approval Matrix controls expenditure by routing Purchase Requisitions and Purchase Orders to designated managers before any commitment to a vendor is finalized. Enforcing limits on CAPEX and OPEX is crucial for budget control and preventing unauthorized disbursements.

---

## Applicable Document Types

| Document Type Value | What It Controls |
| :--- | :--- |
| `purchase_requisition` | Internal requests to buy goods or services |
| `purchase_order` | Formal commitment to a vendor |
| `debit_note` | Debit memos raised against suppliers |

---

## Common Trigger Conditions for Purchasing

| Trigger | When to Use |
| :--- | :--- |
| `amount_exceeds` + threshold | Route large POs to management or board approval |
| `always` | Require approval on every PO regardless of value |

## Database Implementation

```sql
SELECT * FROM approval_workflows
WHERE company_id = :company_id
  AND module_type = 'purchasing'
  AND is_active = true;
```

See `01. Unified Approval Workflow.md` for full schema of `approval_workflows`, `approval_workflow_steps`, and `approval_instances`.
