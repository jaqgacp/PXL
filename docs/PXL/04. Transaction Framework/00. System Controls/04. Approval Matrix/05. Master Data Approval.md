## Module Overview
The Master Data Approval Matrix enforces controls over the creation and modification of sensitive master records — Customers, Vendors, Item Masters, Chart of Accounts. Creating a new Vendor requires KYC and TIN verification; modifying an Item's standard cost requires management oversight. This prevents unauthorized master data tampering.

---

## Applicable Document Types

| Document Type Value | What It Controls |
| :--- | :--- |
| `customer` | New customer creation or credit limit changes |
| `supplier` | New supplier creation |
| `item` | New item creation or cost price changes |
| `coa_account` | New GL account creation |
| `employee` | New employee records (HR-linked) |

---

## Common Trigger Conditions for Master Data

| Trigger | When to Use |
| :--- | :--- |
| `always` | All new supplier records must be approved (KYC compliance) |
| `always` | All credit limit increases on customers require approval |

## Database Implementation

```sql
SELECT * FROM approval_workflows
WHERE company_id = :company_id
  AND module_type = 'master_data'
  AND is_active = true;
```
