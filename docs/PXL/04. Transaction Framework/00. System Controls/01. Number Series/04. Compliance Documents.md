## Module Overview
This page manages the auto-numbering sequences for BIR Compliance certificates generated by PXL ERP. Unlike transactional documents (invoices, vouchers), compliance certificates such as BIR Form 2307 (Certificate of Creditable Tax Withheld at Source) and BIR Form 2306 (Certificate of Final Tax Withheld at Source) are official BIR documents issued to payees. They must be serially numbered for internal tracking, reconciliation with the BIR Alphalist (QAP/SAWT), and audit readiness.

The Number Series for Compliance Documents covers the following certificate types:
* BIR Form 2307 — Certificate of Creditable Withholding Tax (EWT)
* BIR Form 2306 — Certificate of Final Withholding Tax (FWT)

> **Reference:** This module shares the same `number_series` Supabase table defined in the **Sales Documents** blueprint. Please refer to that document for the complete database schema. This page documents only the Compliance-specific configuration and business rules.

**Key Design Note:** BIR 2307 and 2306 certificates are issued per payment transaction or per calendar quarter depending on the withholding arrangement. A common convention is to number them per year and reset annually (e.g., **2307-2026-000001**), making the certificate year immediately identifiable and simplifying QAP reconciliation.

## What You Will See on the Screen (The Dashboard)
When you open the "Number Series — Compliance Documents" page, you will see a clean and organized workspace.

### The Action Bar (Top of the page):
* **Search Bar:** Allows you to find a series by Document Type, Prefix, or Branch.
* **Company Filter:** A dropdown to filter series by parent company.
* **Branch Filter:** A dropdown to filter by branch.
* **Document Type Filter:** A dropdown to show the series for a specific certificate type (2307 or 2306).
* **Customize View:** A button that lets you choose which columns to show or hide on your screen.
* **Import Button:** Allows bulk creation of compliance number series records by uploading a pre-formatted CSV or Excel file. Useful for batch setup across multiple branches during system go-live.
* **Export Button:** Downloads the current filtered list to an Excel or CSV file for review or QAP reconciliation documentation.
* **Print Button:** Generates a print-ready view of all configured compliance series for hardcopy reference.
* **Create New Series Button:** A prominent button to add a new number series record.

### The Data List (The main table - must be customizable):
You will see a list of all configured compliance document series. The default columns displayed are:
* Document Type
* Company
* Branch
* Prefix
* Current / Next Number (e.g., 2307-2026-000012)
* Reset Frequency
* Status (A green badge for Active, gray for Inactive)

At the end of every row, there will be buttons to "Edit" or "View" the complete details of that number series.

## What Information We Need to Capture (The Data Fields)
When you click "Create New Series" or "Edit," a form will open. Here is the exact data the system will save.

### Section 1: Series Scope & Assignment
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Company | Dropdown linked to active companies. The withholding agent entity that issues these certificates. | Yes |
| Branch | Dropdown linked to active branches. Each branch that independently processes payments and issues 2307/2306 certificates should have its own series to avoid numbering conflicts on QAP submissions. | Yes |
| Document Type | Dropdown: BIR Form 2307 (EWT Certificate), BIR Form 2306 (FWT Certificate). | Yes |

### Section 2: Number Format Configuration
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Prefix | The text prepended to the number. Recommended formats: "2307-2026-" for EWT certificates, "2306-2026-" for FWT certificates (year embedded in prefix, updated on yearly reset). | Optional |
| Suffix | The text appended after the number. | Optional |
| Number Length (Padding) | The total number of digits in the numeric portion, zero-padded (e.g., 6 produces "000001"). | Yes |
| Starting Number | The first number in the sequence. Defaults to 1. | Yes |
| Next Available Number | The next number the system will assign when a new certificate is issued. Auto-increments each time a 2307 or 2306 is generated from a payment transaction. | Yes |

### Section 3: Reset & Control Settings
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Reset Frequency | Dropdown: Never (continuous), **Reset Yearly** (recommended — resets counter at the start of each calendar year, aligning with the annual QAP filing cycle), Reset Monthly. | Yes |
| Allow Manual Override | Toggle (Yes / No). If YES, users with sufficient permissions can manually assign a certificate number (useful for correcting out-of-sequence certificates or encoding manually prepared certificates). | Yes |

### Section 4: Format Preview
This is a display-only section that shows a live preview of the generated certificate number based on the configuration above.

| Field Name | Description |
| :--- | :--- |
| Format Preview | Auto-rendered display (e.g., **2307-2026-000001** or **2306-2026-000001**). Updates in real-time as the user configures the fields above. |

## The Supabase Database Architecture

### Shared Table: `number_series`
The Compliance Documents number series uses the same `number_series` table defined in the **Sales Documents** blueprint. No additional tables are required for this module.

The records pre-seeded for Compliance in the `ref_document_types` table are:

| `document_code` | `document_name` | `category` | `is_bir_registered` |
| :--- | :--- | :--- | :--- |
| `2307` | BIR Form 2307 — EWT Certificate | `compliance` | `false` |
| `2306` | BIR Form 2306 — FWT Certificate | `compliance` | `false` |

**Application Logic Note:** When a Payment Voucher or Vendor Bill with withholding tax is confirmed and posted, the system must automatically generate the next 2307 certificate number from this series and attach it to the transaction record. Similarly, when a Final Withholding Tax transaction is confirmed, the next 2306 number is auto-assigned. This linkage ensures that the QAP and SAWT reports in the Compliance module can accurately list certificate numbers without any manual re-entry.
