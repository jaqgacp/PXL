## Module Overview
Reversal Controls dictate how already-posted documents (which have hit the GL) are reversed. To comply with Philippine BIR regulations, posted financial transactions cannot be hard-deleted. Instead, an offsetting Reversal Journal must be generated. This module defines the rules for these reversing entries.

## What You Will See on the Screen (The Dashboard)
The dashboard sets up the behavior of reversing entries for different source documents.

### The Action Bar (Top of the page):
* **Search Bar:** Document Type.
* **Export Button:** Export.
* **Configure Rule Button:** Set a reversal behavior.

### The Data List (The main table):
* Document Type
* Allowed to Reverse?
* Reversal Method (Negative Debit/Credit, Inverse Debit/Credit)
* Reversal Reason Required?

## What Information We Need to Capture (The Data Fields)

### Section 1: Reversal Rules
| Field Name | Description | Required? |
| :--- | :--- | :--- |
| Document Type | E.g., Posted Sales Invoice, Posted JV. | Yes |
| Allow Reversal | Toggle on/off. | Yes |
| Reversal Method | Inverse (Swap Dr/Cr) or Negative (Same accounts, negative values). | Yes |
| Require Reason | Must the user pick a reason code? | Yes |

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `reversal_controls`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `document_type` | Text | Required, Unique | E.g., `posted_invoice`. |
| `allow_reversal` | Boolean | Default: `true` | Is reversal allowed? |
| `reversal_method` | Text | Required | `inverse`, `negative`. |
| `require_reason` | Boolean | Default: `true` | Must give reason. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation timestamp. |
| `updated_at` | Timestamp | Auto | Update timestamp. |
