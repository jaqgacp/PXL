## Module Overview
Posting Validation Rules perform a final check immediately before a document generates accounting entries. For Philippine compliance, this ensures the GL is perfectly balanced (Debits = Credits) and that mandatory tax codes are present on VAT-able transactions before committing to the ledger.

## What You Will See on the Screen (The Dashboard)
A set of non-negotiable accounting checks.

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `posting_validation_rules`
| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `rule_name` | Text | Required | Name of the rule. |
| `check_type` | Text | Required | `dr_cr_balance`, `tax_code_exists`, `period_open`. |
| `is_active` | Boolean | Default: `true` | Status toggle. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |
