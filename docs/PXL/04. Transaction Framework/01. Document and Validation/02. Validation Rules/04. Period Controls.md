## Module Overview
Period Controls define which accounting periods (months) are Open, Closed, or Restricted. In local accounting, once VAT returns are filed (e.g., 20th or 25th of the following month), the prior month must be Hard Closed to prevent any backdated entries that would disrupt filed tax numbers.

## The Supabase Database Architecture

**Critical Database Rule:** Ensure appropriate unique constraints are applied to prevent duplicate entries.

### Table 1: `period_controls`

**IMPORTANT — Advisory Layer Only:** This table provides UI-visible status badges per module per period. It does NOT independently block transactions. Transaction blocking is exclusively handled by `accounting_periods.is_closed` (Layer 1) and `fiscal_locks.lock_date` (Layer 2). `period_controls.status` is Layer 3 — used for dashboard display and reporting only.

| Column Name | Data Type | Rules | What it stores |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary Key | System ID. |
| `company_id` | UUID | Required, Foreign Key | Links to `companies.id`. Tenant isolation anchor. |
| `fiscal_period_id` | UUID | Required, Foreign Key | Links to `accounting_periods.id`. |
| `module_name` | Text | Required | `ar`, `ap`, `gl`, `inventory`, `fixed_assets`, `bank`. |
| `status` | Text | Required, Default `open` | Advisory status: `open`, `pending_review`, `closed`. For display purposes only. |
| `restricted_role_id` | UUID | Nullable, Foreign Key | Links to `roles.id`. For display annotation only — not enforced at DB layer. |
| `created_by` | UUID | Required, Foreign Key | Links to `users.id`. |
| `updated_by` | UUID | Nullable, Foreign Key | Links to `users.id`. |
| `created_at` | Timestamp | Auto | Creation date. |
| `updated_at` | Timestamp | Auto | Update date. |

**Composite Unique Constraint:** `UNIQUE(company_id, fiscal_period_id, module_name)`
